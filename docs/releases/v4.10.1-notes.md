# v4.10.1 — Desktop database reopening and usability polish

## Summary

**v4.10.1** makes the read-only desktop shell practical for daily use:

- Persist the last successfully selected database path under the OS app config directory
- Safe restore on startup (never create a missing DB)
- Change Database / Forget Database actions
- Selected database name in the header
- Root Cargo version bumped to `4.10.1`; Markdown colophon synced

Implementation spec: [v4.10.1-desktop-database-reopening-and-usability-polish.md](../specifications/v4.10.1-desktop-database-reopening-and-usability-polish.md)

## Packaging note

- GitHub Release assets remain **CLI 3OS only**
- Desktop remains **developer preview / source-only / no bundle** (`bundle.active = false`)

## Persistence

- Backend-owned `{app_config_dir}/desktop-settings.json`
- Written only after successful `select_database` validation
- Invalid saved paths are cleared on restore attempt; app still starts unselected
- Forget clears settings only — **never deletes the SQLite file**
- Separate from CLI `db use` / `caglla.toml`

## Boundary

- Read-only UI only (no trip / day / itinerary write forms)
- No recent-DB multi-item list
- No desktop bundle distribution
- No root Cargo workspace conversion
- CLI plain-text / JSON contracts unchanged except for the version colophon in Markdown export
