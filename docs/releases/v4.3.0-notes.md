# v4.3.0 — Reservation / Summary display refinement

**v4.3.0** refines Travel Book `trip export-md` output for Reservation display, implementing the approved [v4.3.0 design](../specifications/v4.3.0-reservation-summary-display-refinement-design.md). Summary placement is unchanged; chapter layout from v4.2.0 is unchanged.

## Highlights

- **Reservations — flat chronological** — No type group headings (`### Flight` etc.); reservations follow trip itinerary order (day → sort_order)
- **Reservation headings** — `**Day N / {itinerary}**` with `— {provider}` only when provider adds information beyond the itinerary label
- **Travel Book labels** — `確認番号:` / `期間:` / `備考:` (Japanese field labels for Japanese prose)
- **Human-readable periods** — Hotel check-in/out and other periods formatted as `2026-04-26 16:40 〜 2026-04-29 10:00` instead of raw ISO
- **Days overview** — In Trip overview, after metrics, a `### Days overview` bullet list of Day summaries (days with summary only); Daily schedule day intros unchanged

## Unchanged

- Travel Book chapters and entity boundaries (no Expense / Receipt / Difference in export-md)
- trip export JSON schema and SQLite schema
- Okinawa ledger invariants: Itinerary 58 / Expense 49 / ¥561,780 / Receipt 6

## CLI

```bash
caglla trip export-md <trip-id> [--output path.md]
```

No new flags; Reservations chapter formatting and Trip overview Days overview list.
