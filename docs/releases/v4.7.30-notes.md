# v4.7.30 — P-6j destructive / structural apply operations policy

## Summary

**v4.7.30** is a **documentation-only policy** after P-6i (v4.7.28–v4.7.29): classifies Proposal Fragment apply operations (additive / in-place / destructive / structural), defines initial `delete_itinerary` and `reorder_itinerary` policies, compares rollback/undo options, and proposes a phased adoption path before any implementation.

- **Updated:** indexes, current-work, README, public docs links
- **Not changed:** `src/`, tests, fixtures, schema v8, DB, runtime behavior

## Headline policy

- **Four-way classification:** additive · in-place update · destructive · structural
- **Delete (planned):** itinerary target only; childless hard delete; child entities → blocking; no post-success undo in v4.7.x
- **Reorder (deferred):** separate from delete; same-day vs cross-day split; `expected_order` baseline + TOCTOU
- **Undo / rollback:** v4.8+ candidates — apply journal, soft delete, optional export backup hint
