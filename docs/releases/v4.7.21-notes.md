# v4.7.21 — Fragment apply add_itinerary field expansion (P-6e)

## Summary

**v4.7.21** implements **P-6e**: expands `fragment apply --confirm` field coverage for `add_itinerary` — `category`, `start_time`, `duration_minutes`, `travel_minutes` (alias `travel_time_minutes`) — while keeping `title`, `note`, and `location` from v4.7.19.

Spec: [v4.7.21-fragment-apply-add-itinerary-field-expansion.md](../specifications/v4.7.21-fragment-apply-add-itinerary-field-expansion.md)

## What changed

- **Extended:** `fragment apply --confirm` — shared parser for dry-run preview and confirm insert
- **New fields on confirm:** `category`, `start_time`, `duration_minutes`, `travel_minutes` / `travel_time_minutes`
- **sort_order:** confirm still appends to Day end; ordering hints emit `ordering_hint:` warnings on dry-run
- **Unchanged:** `--dry-run` — read-only DB access, no Trip domain data mutation
- **DB write:** `add_itinerary_item` insert only — no update/delete/reorder
- **Not changed:** fragment validate (file-only), proposal materialize, schema v8

## Usage

```bash
caglla fragment apply fragment.json --dry-run --trip 1 --output preview.json
caglla trip validate-export preview.json
caglla fragment apply fragment.json --confirm --trip 1

caglla itinerary list 1
caglla day show 1 1
caglla trip export 1
```

## Safety

- Invalid `category` / `start_time` / duration fields → blocking error, no DB write
- Ordering hints (`sort_order`, `placement_hint`, `after`/`before`) → warnings only; confirm appends to Day end
- `--dry-run` and `--confirm` **cannot be combined**
- Unsupported intent/target / unresolved required decisions → no DB write

## Compatibility

- **JSON golden:** unchanged
- **Markdown golden:** colophon version only (`4.7.21`)
- **Gate report JSON:** schema version 2 (unchanged)
