# v4.8.13 — Currency CLI write-path hardening

## Summary

**v4.8.13** applies `CurrencyValidationMode::IsoStrict` to **CLI create/update write paths**:

- `expense add` / `expense update` (when `--currency` is explicit)
- `estimate add` / `estimate update` (when `--currency` is explicit)

Cargo version bumped to `4.8.13`. Markdown colophon synced. Domain layer and non-CLI paths remain on `FormatOnly`.

Implementation spec: [v4.8.13-currency-cli-write-path-hardening.md](../specifications/v4.8.13-currency-cli-write-path-hardening.md)

## CLI behavior change

- Valid ISO codes (including withdrawn) succeed with uppercase normalization (`jpy` → `JPY`)
- Unknown codes (`ZZZ`, `ABC`, `JPN`) rejected on CLI add / explicit currency update
- Denylist codes (`XXX`, `XTS`, `XAU`, etc.) rejected on CLI add / explicit currency update
- Format-invalid input rejected (unchanged messages)

## Compatibility

- Legacy unknown currency in existing DB: read/list/show/export still work
- Update without `--currency`: does not re-validate existing currency
- import / validate-export / Fragment apply: format-only (unchanged)
- Receipt/inbox CLI: deferred to v4.8.14+
- DB schema / JSON report shape / JSON golden: unchanged

## Format-only surfaces (unchanged)

- `validate_currency_code()` / domain layer
- `trip import`, `validate-export`, Proposal Fragment apply
- read / list / show / export / export-md
