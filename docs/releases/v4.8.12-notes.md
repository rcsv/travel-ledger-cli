# v4.8.12 — Currency ISO internal registry + validation mode

## Summary

**v4.8.12** introduces the **internal foundation** for Issue #66 Currency ISO validation hardening:

- `CurrencyValidationMode` (`FormatOnly`, `IsoStrict`)
- `validate_currency_code_with_mode()`
- Embedded ISO 4217 registry (262 alpha-3 codes)
- Travel-expense denylist (strict mode only)

**No external CLI behavior change** — all existing call sites remain on `FormatOnly`. Cargo version bumped to `4.8.12`. Markdown colophon synced.

Implementation spec: [v4.8.12-currency-iso-internal-registry-validation-mode.md](../specifications/v4.8.12-currency-iso-internal-registry-validation-mode.md)

## Registry note

The embedded ISO alpha-3 list is derived from the **`iso_currency` 0.5.3 dataset** plus supplemental withdrawn codes. **No `iso_currency` crate dependency was added** — the dataset is embedded as a sorted static slice for zero runtime dependency overhead.

## What changed

- `src/money.rs` → `src/money/mod.rs` + `src/money/currency_registry.rs`
- New validation mode API; default path unchanged
- Unit tests for format-only, strict, denylist, withdrawn codes, lowercase normalize

## Compatibility

- `validate_currency_code()` — `FormatOnly` wrapper; existing behavior preserved
- expense / estimate / receipt / trip import / validate-export / Fragment apply — **not strictened**
- read / list / show / export — legacy unknown currency unaffected
- `currency_minor_unit_digits` unknown fallback unchanged
- JSON report shape / JSON golden / DB schema — unchanged

## What did not change

- CLI create/update behavior
- Proposal Fragment apply strict integration
- import / validate-export policy
- minor units validation
- structured error codes
