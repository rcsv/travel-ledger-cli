# v4.8.1 — P-6p delete_estimate dry-run

## Summary

**v4.8.1** implements Proposal Fragment **`delete_estimate` dry-run** (P-6p).

This release adds the read-only preview path only. `delete_estimate --confirm`, scoped DELETE, and `deleted_estimate_id` remain outside this release.

Implementation spec: [v4.8.1-p6p-delete-estimate-dry-run.md](../specifications/v4.8.1-p6p-delete-estimate-dry-run.md)

Planning spec: [v4.8.0-p6p-delete-estimate-planning.md](../specifications/v4.8.0-p6p-delete-estimate-planning.md)

## What shipped (dry-run only)

- **`fragment apply --dry-run --trip <id>`** supports intent `delete_estimate`
- **Itinerary target only** — reuses existing numeric selector resolver
- **Estimate identity:** `candidate_content.estimate_id` with parent Itinerary verification
- **optional `expected_*` + `expected_updated_at`** baseline — mismatch blocking
- **mutation fields rejected** — delete does not accept amount/title patch fields
- **missing / already deleted** → blocking (not idempotent success)
- **DB read-only** — no DELETE on dry-run path
- Preview report: `estimate_delete_preview`, `estimates_before` / `estimates_after`
- Destructive warning: hard delete, undo not guaranteed

## Not in scope

- `--confirm` / scoped `DELETE ... WHERE id AND itinerary_id`
- `deleted_estimate_id`（v4.8.2 confirm scope）
- Expense mutation, schema / migration changes
