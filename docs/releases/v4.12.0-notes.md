# v4.12.0 — Desktop activity editing workflow

## Summary

**v4.12.0** adds the first focused Activity editing workflow to the Desktop developer preview:

- Open a contextual Activity Inspector from an explicit timeline Edit action
- Edit the selected Activity's title, start time, location, and note
- Save or cancel changes without losing the selected Trip and Day context
- Refresh only the selected Day from SQLite after a successful save and keep the edited Activity selected when possible
- Protect the workflow from duplicate submissions, stale navigation completions, and misleading write failures

The user-facing Desktop vocabulary remains **Activity**. The internal domain, Core API, and storage model continue to use **Itinerary**.

## Desktop Activity Inspector

- Uses explicit Edit, Save, and Cancel actions instead of autosave
- Keeps quick add and Activity editing mutually exclusive
- Preserves the draft and existing timeline after a write failure so the user can correct or retry it
- Treats a successful write followed by a failed timeline refresh as a read failure, avoiding an accidental duplicate update
- Returns focus to the originating Edit action after Cancel when the Activity still exists

## Core write boundary

- Adds a narrow public Itinerary update service for title, start time, location, and note
- Shares neutral content normalization and validation between Itinerary create and update services
- Verifies the Itinerary still belongs to the supplied Trip and Day before updating it
- Updates only the four accepted fields and `updated_at`; Day, order, duration, travel time, category, and child data remain unchanged
- Keeps structured validation, target-not-found, and storage errors across the Core-to-Desktop boundary

## Responsive behavior

- At 1120×720, the timeline and contextual Inspector remain usable side by side inside the Trip Workspace
- At the 720×480 minimum smoke size, the Inspector stacks within the Workspace without introducing a separate app-shell pane
- Long Activity titles, locations, and notes wrap without breaking the shell or clipping Edit, Save, or Cancel controls

## Compatibility and packaging

- No database schema or migration changes
- No Travel Ledger export/import format changes
- No CLI command or behavior changes
- Desktop remains a **source-only developer preview** with bundling disabled (`bundle.active = false`)
- GitHub Release assets remain the CLI archives for Linux amd64, macOS arm64, and Windows amd64; no Desktop installer is included
