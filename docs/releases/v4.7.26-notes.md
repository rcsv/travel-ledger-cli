# v4.7.26 — Fragment apply add_reservation dry-run (P-6h)

## Summary

**v4.7.26** adds **P-6h dry-run path**: `fragment apply --dry-run` previews one Reservation append under a resolved Itinerary target, with gate report and optional preview Trip JSON — no DB writes.

Spec: [v4.7.26-fragment-apply-add-reservation-dry-run.md](../specifications/v4.7.26-fragment-apply-add-reservation-dry-run.md)

## What changed

- **New:** `fragment apply --dry-run` — `add_reservation` for **itinerary target only**
- **New:** gate report fields `reservations_before` / `reservations_after` / `reservation_preview`
- **Extended:** `fragment validate` — recognizes `add_reservation` intent
- **Unchanged:** `add_reservation --confirm`, schema v8, JSON golden

## Usage

```bash
caglla fragment apply fragment.json --dry-run --trip 1 --output preview.json
caglla trip validate-export preview.json
caglla fragment apply fragment.json --dry-run --trip 1 --json
caglla reservation list --itinerary 1 --json
```

## Safety

- Reservation preview only — no INSERT
- trip / day target → blocking (unsupported)
- invalid reservation_type / provider_name → blocking
- required decisions / unresolved / ambiguous → no DB write
- `--dry-run --confirm` → error

## Compatibility

- **JSON golden:** unchanged
- **Markdown golden:** colophon version only (`4.7.26`)
