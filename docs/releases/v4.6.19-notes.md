# v4.6.19 — Day show service boundary

## Summary

**v4.6.19** introduces a minimal **service layer** for the read-only `day show` command, keeping existing CLI human output, JSON shape (`DayShowJson`), Day summary, itinerary ordering, and not-found behavior unchanged while making the use case reusable for future Tauri GUI screens.

Specification: [v4.6.19-day-show-service-boundary.md](../specifications/v4.6.19-day-show-service-boundary.md)

## What changed

- New `src/services/day_show.rs` — `show_day()` returns `DayShowServiceResult` (no terminal I/O)
- `day show` CLI handler calls service, then routes to existing human/JSON output
- `DayShowJson` wrapper remains in CLI layer (`day.rs`)

## Compatibility

- `day show` human output unchanged (Trip, Day, Date, summary, Itineraries)
- `day show --json` unchanged (`DayShowJson` serialization)
- Not found: `Trip not found: <id>` / `Day not found: trip <id> day <n>`
- `day list` / update / swap unchanged
- No DB schema or migration changes
- Golden updates: Okinawa Markdown Colophon `Version:` only

## Non-goals

- No `day list` / update / swap changes
- No reservation / expense show changes
- No Tauri / GUI implementation
