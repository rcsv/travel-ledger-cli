# v4.7.12 — Public examples validation guard

## Summary

**v4.7.12** adds a **CI / `make check` guard** so public schema v8 Trip examples and non-normative Proposal Envelope / Fragment examples keep validating with the correct commands — and cross-validation failures stay detectable.

Spec: [v4.7.12-public-examples-validation-guard.md](../specifications/v4.7.12-public-examples-validation-guard.md)

## What changed

- **New:** `tests/public_examples_validation_guard.rs` — 9 integration tests
- **Not changed:** validate commands behavior, schema v8, JSON golden, DB / materialize

## Guard coverage

| Examples | Expected command |
|---|---|
| `docs/public/examples/*.json` (3 files) | `trip validate-export` → PASS |
| `trip-proposal-envelope.example.json` | `proposal validate` → PASS |
| `proposal-fragment.example.json` | `fragment validate` → PASS |
| Cross-validation (Trip / Envelope / Fragment) | wrong command → FAIL |

## Compatibility

- **JSON golden:** unchanged
- **Markdown golden:** colophon version only (`4.7.12`)
