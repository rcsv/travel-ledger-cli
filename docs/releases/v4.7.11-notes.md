# v4.7.11 — Proposal Fragment file validation

## Summary

**v4.7.11** implements **P-3**: `fragment validate <file>` for minimal file-based validation of Proposal Fragment files — partial proposals against an existing Trip, without DB, materialize, or apply.

Spec: [v4.7.11-proposal-fragment-file-validation.md](../specifications/v4.7.11-proposal-fragment-file-validation.md)

## What changed

- **New:** `fragment validate`, `fragment validate --json` commands
- **New:** `ProposalFragmentValidationReport` — blocking errors / warnings / summary
- **Not changed:** `trip validate-export`, `proposal validate` / `show` / `inspect`, schema v8, JSON golden

## Usage

```bash
caglla fragment validate docs/public/examples-non-normative/proposal-fragment.example.json
caglla fragment validate fragment.json --json
```

## Boundaries

| Document kind | Command |
|---|---|
| schema v8 Trip export | `trip validate-export` |
| Trip Proposal Envelope | `proposal validate` / `show` / `inspect` |
| Proposal Fragment | `fragment validate` |

## Compatibility

- **JSON golden:** unchanged
- **Markdown golden:** colophon version only (`4.7.11`)
