# v4.8.15 — validate-export currency warnings

## Summary

**v4.8.15** adds **currency quality warnings** to `trip validate-export` without breaking import/export round-trip:

- Format-invalid currency remains **error**
- Registry-unknown codes（`ZZZ`, `ABC` 等）→ **warning**
- Denylist codes（`XXX`, `XTS`, `XAU` 等）→ **warning**
- Valid ISO codes（`JPY`, `USD`, withdrawn codes）→ no warning

Cargo version bumped to `4.8.15`. Markdown colophon synced. Import / read / export behavior unchanged.

Implementation spec: [v4.8.15-validate-export-currency-warnings.md](../specifications/v4.8.15-validate-export-currency-warnings.md)

## Behavior change

- `trip validate-export` reports currency quality issues in `warnings[]`
- `valid: true` when only currency quality warnings exist（import 可能）
- `trip import` unchanged — unknown currency still imports successfully

## Compatibility

- DB schema / export JSON schema: unchanged
- `EXPORT_VALIDATION_REPORT_SCHEMA_VERSION`: unchanged（`1`）
- JSON golden: unchanged
- import / read / export / export-md: unchanged
- CLI write path / Fragment apply strict: unchanged

## Format-only surfaces (unchanged)

- `trip import`
- domain layer / `validate_currency_code()` wrapper
- Receipt / inbox CLI
- read / list / show / export / export-md
