# Release v4.2.0 — Travel Book export-md layout

**v4.2.0** improves `trip export-md` so enriched Travel Book data (summaries, notes, reservations, estimates) is readable in a chapter-based Markdown layout aligned with [v4.1.0 chapter structure design](../specifications/v4.1.0-travel-book-chapter-structure-design.md).

## Highlights

- **Travel Book chapters** — Cover → Trip overview → Daily schedule → Reservations → Checklist → Planned cost → Notes → Colophon
- **Entity boundaries** — Trip/Day summaries in overview and daily sections; Note entities in `## Notes`; estimates in `## Planned cost` (not mixed into itinerary blocks)
- **Pre-trip focus** — Expense lines, Actual totals, and Planned vs Actual Difference are omitted from export-md (use `trip stats` for accounting views)
- **Okinawa canonical sample** — `samples/okinawa_sesoko_2026/expected-export-md.md` golden + integration test for deterministic output (timestamp normalized in tests)
- **Sample invariants unchanged** — Itinerary 58 / Expense 49 / ¥561,780 / Receipt 6

## CLI

```bash
caglla trip export-md <trip-id> [--output path.md]
```

Behavior is unchanged at the CLI surface; Markdown structure and section order changed.
