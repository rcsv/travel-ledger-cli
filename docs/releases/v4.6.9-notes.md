# v4.6.9 — Itinerary timeline service boundary

## Summary

**v4.6.9** introduces a minimal **service layer** for the read-only `itinerary timeline` command, keeping existing CLI human output, Sequence-first ordering, and Trip not-found behavior unchanged while making the use case reusable for future Tauri GUI timeline screens.

Specification: [v4.6.9-itinerary-timeline-service-boundary.md](../specifications/v4.6.9-itinerary-timeline-service-boundary.md)

## What changed

- New `src/services/itinerary_timeline.rs` — `get_timeline()` returns `ItineraryTimelineServiceResult` (no terminal I/O)
- `itinerary timeline` CLI handler calls service, then routes to existing human output

## Compatibility

- `itinerary timeline` human output unchanged
- No `--json` flag (unchanged from prior releases)
- Ordering unchanged (`ORDER BY day_number, sort_order, id`)
- `start_time` remains a display label, not primary sort key
- Trip not found unchanged (`Trip not found: <id>`)
- No DB schema or migration changes
- Golden updates: Okinawa Markdown Colophon `Version:` only

## Non-goals

- No timeline display improvements
- No `itinerary list` / `itinerary show` service split
- No Tauri / GUI implementation
