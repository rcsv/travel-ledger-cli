# v4.10.0 — Read-only desktop vertical slice

## Summary

**v4.10.0** ships the first read-only Tauri desktop shell for Travel Ledger:

- `desktop/` — Tauri 2 + React + TypeScript + Vite
- Open an existing Travel Ledger SQLite database via native file dialog
- Trip list → Trip detail → Day tabs → Day itinerary timeline
- Path dependency on the root `travel_ledger_cli` library (`open_db` + v4.9.2 read facade)
- Structured desktop error envelope (`DATABASE_NOT_SELECTED`, `TRIP_NOT_FOUND`, …)
- Root Cargo version bumped to `4.10.0`; Markdown colophon synced

Implementation spec: [v4.10.0-read-only-desktop-vertical-slice.md](../specifications/v4.10.0-read-only-desktop-vertical-slice.md)

## Desktop packaging

- **GitHub Release assets remain CLI 3OS only** (macOS / Linux / Windows) — unchanged from prior releases
- Desktop is **developer preview / source-only**
- Desktop **bundle is not distributed**
- `bundle.active = false` in `desktop/src-tauri/tauri.conf.json`
- Run from source: `cd desktop && npm install && npm run tauri dev` (or `tools/desktop/dev.sh`)

## Manual smoke

GUI manual smoke was performed before this formal release (DB picker → Trip list → Trip detail → Day timeline on a populated sample database).

## Itinerary completion state

**Itinerary has no completion state** (by design):

- No `itinerary done` / `undone` CLI (out of scope)
- No `itinerary_items.is_done` column / migration / `ItineraryDetail.is_done`
- **Checklist** completion (`checklist check` / `uncheck` → `checklist_items.is_done`) remains a **separate responsibility**

A short-lived attempt to add itinerary `is_done` for desktop display was **withdrawn** before this release.

## Boundary

- Read-only UI only (no trip / day / itinerary write forms)
- No root Cargo workspace conversion
- No export/import schema changes
- CLI plain-text / JSON contracts unchanged except for the version colophon in Markdown export
