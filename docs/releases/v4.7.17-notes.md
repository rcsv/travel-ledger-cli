# v4.7.17 — Proposal materialize --confirm (P-6b)

## Summary

**v4.7.17** implements **P-6b**: `proposal materialize <file> --confirm` — adopt a Trip Proposal Envelope as a **new Trip** in the DB, using the same gate as dry-run plus `trip import`-equivalent persistence.

Spec: [v4.7.17-proposal-materialize-confirm.md](../specifications/v4.7.17-proposal-materialize-confirm.md)

## What changed

- **New:** `proposal materialize <file> --confirm` — DB insert after gate + self-validation
- **New:** gate report schema v2 — `confirm`, `trip_id` fields
- **Unchanged:** `--dry-run` remains file-only (DB open before dispatch only for `--confirm`)
- **Persistence:** `import_trip_from_json_with_summary` — same path as `trip import`
- **Not changed:** Fragment apply, existing Trip update, proposal DB storage, schema v8

## Usage

```bash
# Recommended flow
caglla proposal materialize envelope.json --dry-run --output /tmp/candidate.json
caglla trip validate-export /tmp/candidate.json
caglla proposal materialize envelope.json --confirm

caglla trip list
caglla trip show 1
```

## Safety

- No DB write without explicit `--confirm`
- `--dry-run` only: file-only dispatch (no DB open, no side effects)
- `--dry-run` and `--confirm` **cannot be combined** (exit 1) — `dry-run means no side effects`
- Typical flow: `--dry-run` → review → separate `--confirm` run
- `--confirm`: same validation / gate / `trip validate-export`-compatible JSON as dry-run

## Compatibility

- **JSON golden:** unchanged
- **Markdown golden:** colophon version only (`4.7.17`)
- **Gate report JSON:** schema version 2 when using `--json`
