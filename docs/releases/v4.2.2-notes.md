# v4.2.2 — Travel Book Markdown polish

**v4.2.2** polishes `trip export-md` Travel Book output for readability, implementing the small items deferred from the v4.2.1 post-release review. Chapter layout from v4.2.0 is unchanged.

Plan: [v4.2.2-travel-book-markdown-polish-implementation-plan.md](../specifications/v4.2.2-travel-book-markdown-polish-implementation-plan.md)

## Highlights

- **Trip overview** — Omit Stay / Travel / Total time lines when all three are zero (avoids “0m trip” noise on unmeasured itineraries)
- **Okinawa sample** — Reservation remarks and Estimate notes use user-facing Japanese; fixture / ledger origin documented in sample README only
- **Notes chapter** — Trip notes before Day notes (Itinerary-scoped Note entities follow; itinerary `note` field stays in Daily schedule)
- **Reservations** — Skip redundant `Provider:` when it duplicates the reservation title or linked itinerary label

## Unchanged

- Travel Book chapters and entity boundaries (no Expense / Receipt / Difference in export-md)
- trip export JSON schema and SQLite schema
- Okinawa ledger invariants: Itinerary 58 / Expense 49 / ¥561,780 / Receipt 6
