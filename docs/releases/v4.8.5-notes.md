# v4.8.5 — Fragment apply internal structured error model + code registry

## Summary

**v4.8.5** introduces an **internal** structured error model and stable code registry for Fragment apply, with **limited wiring** on representative blocking paths.

External contract unchanged: existing `errors: Vec<String>`, human CLI output, and JSON report shape remain compatible. `structured_errors` is stored internally with `#[serde(skip)]` — **not** exposed in JSON until v4.8.6.

Specification: [v4.8.5-fragment-apply-internal-structured-error-model.md](../specifications/v4.8.5-fragment-apply-internal-structured-error-model.md)

Planning basis: [v4.8.4 structured errors planning](../specifications/v4.8.4-fragment-apply-structured-errors-api-readiness-planning.md)

## What shipped

- **`src/proposal/apply_errors.rs`** — internal types: `ApplyErrorCode`, `ApplyErrorKind`, `ApplyErrorPhase`, `StructuredApplyError`
- **Stable `APPLY_*` code registry** (15 codes)
- **Limited wiring** on representative blocking paths:
  - target resolver (unresolved / ambiguous / not found / unsupported target)
  - required decisions gate
  - unsupported intent / target (simulate + confirm scope)
  - entity not found / parent mismatch / baseline mismatch (`update_estimate` / `delete_estimate`)
  - mutation field rejection / invalid `estimate_id` (`delete_estimate` parse)
- **`push_structured_error`** mirrors legacy `errors[]` (deduped)
- **`structured_errors`** on `FragmentApplyDryRunReport` with `#[serde(skip)]`
- Unit tests for code stability and representative gate paths
- `fragment_apply_cli` integration suite unchanged (207 passed)

## Compatibility

- Legacy `errors[]` messages preserved for wired paths
- Human CLI output unchanged
- JSON report: no public `structured_errors[]` field
- `FRAGMENT_APPLY_REPORT_SCHEMA_VERSION` remains `2`

## Not in scope

- CLI behavior changes
- JSON report breaking changes / schema version bump
- Public `structured_errors[]` exposure (deferred to v4.8.6)
- Full error path migration
- confirm transaction codes (preview mismatch, TOCTOU, scoped write)
- DB schema / migration
