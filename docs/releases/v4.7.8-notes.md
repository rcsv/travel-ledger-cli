# v4.7.8 — Proposal implementation planning

## Summary

**v4.7.8** is a **documentation-only implementation planning** release for Trip Proposal Envelope / Proposal Fragment — defining phased delivery (P-0〜P-6), file-based-first strategy, command candidates, and schema v8 boundary rules **without** CLI implementation.

Spec: [v4.7.8-proposal-implementation-planning.md](../specifications/v4.7.8-proposal-implementation-planning.md)

## What changed

- **New:** Proposal implementation planning spec (phases, file vs DB, commands, risks)
- **Updated:** public docs cross-links (proposals, examples-non-normative, concept specs)

## Headline decisions

- **Phased:** P-1 Envelope validate → P-2 inspect → P-3 Fragment validate → P-4 storage → P-5 gate spec → P-6 materialize/apply
- **Storage:** file-based validation/inspection first; DB deferred to P-4
- **Boundary:** `trip validate-export` stays schema v8 Trip only; separate `proposal validate` / `fragment validate` candidates
- **JSON schema:** still **not finalized**
