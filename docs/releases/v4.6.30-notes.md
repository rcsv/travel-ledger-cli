# v4.6.30 — Expense write path boundary review

## Summary

**v4.6.30** is a **documentation-only boundary review** of expense write commands (`add` / `update` / `delete`) after read-only enriched migration (v4.6.26–v4.6.27).

Review: [v4.6.30-expense-write-path-boundary-review.md](../specifications/v4.6.30-expense-write-path-boundary-review.md)

## Headline findings

- **Write path** — not service-backed; handler orchestrates `expense.rs` mutations
- **Handler** — thin wiring (parse → mutate → display); not over-loaded
- **Persistence** — `add_expense` / `update_expense` / `delete_expense` in `expense.rs` (no repository layer)
- **Read/write asymmetry** — list/show use services; write uses module functions (same as Note/Reservation)
- **Post-mutation display** — add/update use `print_expense_detail` adapter (enrich on read)
- **Itinerary parent** — consistent: add requires `--itinerary`, FK `itinerary_id`
- **Short term** — keep current split (Tier 3+ defer holds)
- **JSON golden** — unaffected (write emits no JSON)

## Non-goals

- Implementation, service layer, adapter removal, output changes
