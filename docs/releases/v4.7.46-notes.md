## Key decisions

- **Estimate identity:** `candidate_content.estimate_id` with parent Itinerary verification
- **Update semantics:** partial update (aligned with `estimate update` CLI); at least one update field required
- **amount / currency:** amount-only allowed; currency-only rejected; amount+currency uses new currency for parse; numeric amount = minor units with no double conversion on confirm
- **field omission / null / clear (finalized):** omission → no change; explicit JSON `null` → blocking; nullable clear only via `clear_title` / `clear_note`; value + clear conflict → blocking; empty string `""` → NULL normalization (`normalize_optional_text` in `src/estimate.rs`)
- **no-op:** zero update fields → blocking; same values → valid success + UPDATE (CLI-aligned)
- **TOCTOU:** `expected_*` baselines + preview `before` values (optimistic concurrency; stale preview rejected)
- **success field:** `updated_estimate_id` (not `inserted_estimate_id`)
