# v4.9.0 — Desktop transition and Trip metadata foundation (documentation-only)

## Summary

**v4.9.0** is a **documentation-only** release that:

- Marks v4.8.x currency hardening as complete（through v4.8.18）
- Documents the **GUI / desktop transition** policy
- Designs **Trip optional metadata** for create-form UX:
  - `main_destination`
  - `main_destination_country_code`
  - `default_currency`

Cargo version bumped to `4.9.0`. Markdown colophon synced. No runtime behavior, DB schema, CLI, or export/import changes.

Implementation / design spec: [v4.9.0-desktop-transition-and-trip-metadata-foundation.md](../specifications/v4.9.0-desktop-transition-and-trip-metadata-foundation.md)

## GUI transition (high level)

- Lower priority: further export/import deepening
- Higher priority: Trip / Day / Itinerary daily operations
- Prefer shared `services/*` use case facade（avoid GUI → CLI string）
- First desktop vertical slice: read-only Trip list / Trip detail / Day timeline

## Trip metadata (high level)

| Field | Role |
|---|---|
| `main_destination` | Representative destination label（not Venue） |
| `main_destination_country_code` | ISO 3166-1 alpha-2 helper（not currency） |
| `default_currency` | Input suggestion only（not Expense currency ownership） |

All fields optional; existing Trips remain valid without them.

Country code validation strictness（format vs ISO registry）is deferred to **v4.9.1**.

## Compatibility

- DB schema / migration: unchanged
- JSON schema / CLI: unchanged
- runtime behavior: unchanged
