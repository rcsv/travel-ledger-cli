# v4.6.33 — Expense write service Phase W-2+W-3 implementation

## Summary

**v4.6.33** implements [v4.6.32](../specifications/v4.6.32-expense-write-service-migration-plan.md) Phase W-2+W-3: thin write services for `expense add` / `update` / `delete` and CLI handler wiring.

Spec: [v4.6.33-expense-write-service-phase-w2-w3.md](../specifications/v4.6.33-expense-write-service-phase-w2-w3.md)

## What changed

- **New services:** `expense_add`, `expense_update`, `expense_delete`
- **Add/update result:** `ExpenseEnrichedPart` (same as read `expense_show`)
- **Delete result:** pre-delete `{ id, amount, currency }` snapshot
- **Handler:** uses `print_expense_detail_from_enriched` for add/update; no direct mutation calls
- **`print_expense_detail`:** removed from write path; retained as dead_code adapter (W-5 cleanup deferred)

## Compatibility

- **JSON golden:** unchanged
- **Human output:** unchanged (`expense_cli` tests pass)
- **`import_expense_v3`:** unchanged
