# v3.2.1 — Module layout refactor

## Summary

**v3.2.1** is a **behavior-preserving** internal refactor. Reorganizes the previously flat `src/` layout into responsibility-based modules. **No user-facing feature changes.**

Requires **v3.2.0** baseline. Current trip export still writes **`schema_version: 6`**.

Design series:

```text
Implementation Plan            → v3.2.1-module-layout-implementation-plan.md
Phase 1 Implementation         → PR #59
Post-Implementation Review     → v3.2.1-module-layout-post-implementation-review.md
Release v3.2.1                 → this release
```

---

## Changed (internal only)

### New module layout

```text
src/
  cli/           Clap argument and action definitions (from main.rs)
  commands/      CLI-specific handlers (about, db dispatch)
  domain/        Travel domain models (former models.rs)
  storage/       SQLite connection, migration, DB status (former db.rs)
  analysis/      doctor, advisor, statistics (former stats.rs)
  io/            markdown export, trip diff
  output/        Shared output helpers (print_json)
  main.rs        Thin entry point — dispatch only
```

Root-level command modules remain for incremental migration:

```text
checklist, day, estimate, expense, itinerary, money, note,
participant, reservation, summary, trip
```

### Notable moves

| Before | After |
|---|---|
| Clap defs in `main.rs` | `cli/args.rs` |
| `models.rs` | `domain/models.rs` |
| `db.rs` | `storage/db.rs` |
| `doctor.rs` / `advisor.rs` / `stats.rs` | `analysis/` |
| `markdown.rs` / `diff.rs` | `io/` |
| `trip::print_json` | `output::json::print_json` |
| `about.rs` | `commands/about.rs` |

## Unchanged

| Item | Notes |
|---|---|
| CLI command names, flags, output | Same as v3.2.0 |
| JSON output schemas | Same envelope and field names |
| DB schema / migrations | Same tables and migration functions |
| export/import JSON format | Still schema v6 for current export |
| `db path` / `db status` | open-before-dispatch preserved |
| `db reset` | Same behavior |
