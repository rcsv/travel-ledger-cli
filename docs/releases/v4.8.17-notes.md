# v4.8.17 — Currency hardening follow-up review (documentation-only)

## Summary

v4.8.13〜v4.8.16 の currency hardening を段階導入した結果について、**strict / warning / format-only の適用面を棚卸し**し、残 scope と将来方針を明文化する follow-up review です。

Implementation review: [v4.8.17-currency-hardening-follow-up-review.md](../specifications/v4.8.17-currency-hardening-follow-up-review.md)

## Review conclusions (high level)

- **Strict (IsoStrict) surfaces**: CLI write path / Proposal Fragment apply / Receipt CLI
- **Warning-only surfaces**: `trip validate-export`（expenses / estimates）
- **Format-only surfaces**: trip import / read-export / domain layer（互換維持のため）
- **Minor units**: separate, risk-managed release recommended（currency strict と分離）
- **Trip import strict**: default は format-only を維持し、将来 opt-in strict を検討

## Compatibility

- DB schema / migration: unchanged
- JSON schema / schema_version: unchanged
- runtime behavior: unchanged
