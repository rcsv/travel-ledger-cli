# v4.7.4 — Materialize gate concept / validation rules

## Summary

**v4.7.4** documents the **adoption gate**: how Trip Proposal Envelopes become **new** schema v8 Trips and how Proposal Fragments **merge into existing** Trips — with blocking validation vs non-blocking warnings, required decisions, and expiry handling — following [v4.7.2](../specifications/v4.7.2-trip-proposal-envelope-concept-spec.md) and [v4.7.3](../specifications/v4.7.3-proposal-fragment-concept-spec.md).

## What changed

- **New:** Materialize gate concept / validation rules specification
- **Updated:** `docs/public/proposals.md` — adoption gate entry-level overview
- **Not changed:** export schema, CLI behavior, JSON golden

## Headline decisions

- **Adoption gate** — human review before anything becomes schema v8; not auto-ingest from AI
- **Materialize** — Envelope → new Trip; **Apply** — Fragment → existing Trip (names conceptual)
- **Blocking validation** — missing dates, title, target Trip, deleted entities — stops adoption
- **Non-blocking warnings** — expiry, stale source, overlaps, ambiguities — human decides
- **Reject / defer** — conceptual outcomes; no implementation in v4.7.4
- **schema v8** — unchanged; gate output must conform
