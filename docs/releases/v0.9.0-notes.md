# v0.9.0 - Structured DoctorIssue Targets

## Added

- Added `DoctorIssueTarget` to represent structured issue targets.
  - `Trip`
  - `Day(i64)`
  - `Itinerary(i64)`
- Added `target` to `DoctorIssue`.
- Added helper methods for target-aware issue handling:
  - `target_day()`
  - `target_itinerary_id()`
- Added target-aware advisor command hints.
- Added itinerary-specific MissingDuration issues.

## Changed

- `analyze_trip_issues()` now assigns structured targets to each issue.
- `trip advisor --with-commands` now uses structured issue targets instead of relying primarily on message parsing.
- MissingDuration advice can now show concrete update commands such as:

```bash
cargo run -- itinerary update 3 --duration 60
```

- `trip doctor` keeps user-facing MissingDuration output aggregated, preserving the existing display style.
- Updated README with a short note about more specific command examples.

## Quality

- `cargo fmt` passed.
- `cargo clippy -- -D warnings` passed.
- `cargo test` passed: 107 passed, 0 failed.
- `cargo build` passed.

## Notes

This release strengthens the internal issue model used by Doctor and Advisor. It prepares the CLI for future improvements such as more precise command hints and possible assisted fixes.
