# v4.7.28 — Fragment apply update_itinerary dry-run (P-6i)

## Summary

**v4.7.28** adds **P-6i dry-run path**: `fragment apply --dry-run` previews in-place Itinerary field updates on a resolved itinerary target, with field-level before/after in the gate report and optional preview Trip JSON — no DB writes.

Spec: [v4.7.28-fragment-apply-update-itinerary-dry-run.md](../specifications/v4.7.28-fragment-apply-update-itinerary-dry-run.md)

## What changed

- **New:** `fragment apply --dry-run` — `update_itinerary` for **itinerary target only**
- **New:** gate report `preview.itinerary_field_changes[]` — `{ field, before, after }` (changed fields only)
- **Extended:** `fragment validate` — recognizes `update_itinerary` intent
- **Extended:** `adoption_hints.conflicts` merged into `required_decisions` (all intents)
- **Unchanged:** `update_itinerary --confirm`, schema v8, JSON golden

## Usage

```bash
caglla fragment apply fragment.json --dry-run --trip 1 --output preview.json
caglla trip validate-export preview.json
caglla fragment apply fragment.json --dry-run --trip 1 --json
caglla itinerary list 1
```

## Safety

- Itinerary patch preview only — no UPDATE / INSERT / DELETE
- trip / day target → blocking (unsupported)
- unresolved / ambiguous target → blocking
- `required_decisions` / `adoption_hints.conflicts` / `expected_<field>` mismatch → blocking
- no-op (no effective change) → blocking
- invalid category / time / negative numeric → blocking
- child Expense / Reservation / Note unchanged on dry-run
- `--dry-run --confirm` → error

## Compatibility
