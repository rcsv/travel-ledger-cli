# v4.7.18 — Fragment apply dry-run (P-6c)

## Summary

**v4.7.18** implements **P-6c**: `fragment apply <file> --dry-run --trip <id>` — apply preview / apply simulation for Proposal Fragments against an existing Trip. Uses **read-only DB access** to load the target Trip; does not modify Trip / Day / Itinerary data.

Spec: [v4.7.18-fragment-apply-dry-run.md](../specifications/v4.7.18-fragment-apply-dry-run.md)

## What changed

- **New:** `fragment apply --dry-run --trip <id>` with optional `--output`, `--json`
- **New:** `FragmentApplyDryRunReport` — target resolution + apply preview summary
- **New:** in-memory apply simulation (`add` intent → itinerary preview)
- **Unchanged:** `fragment validate` (file-only), `--confirm`, DB updates, schema v8

## Usage

```bash
caglla trip export 1 --output current.json
caglla fragment apply fragment.json --dry-run --trip 1 --output preview.json
caglla trip validate-export preview.json
caglla trip diff current.json preview.json   # optional
```

## Safety

- `dry-run means no side effects` — **Trip domain data を変更しない**（Trip / Day / Itinerary の mutation なし）
- `fragment apply --dry-run` — **read-only DB access** で既存 Trip を読み取り、in-memory simulation のみ
- preview Trip JSON は **`--output` 指定時のみ** ファイルへ書き出し
- `fragment validate` は従来通り **file-only**（DB を開かない）
- `fragment apply --confirm` — **[v4.7.19 で実装](../specifications/v4.7.19-fragment-apply-confirm.md)**
