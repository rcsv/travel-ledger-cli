# v4.7.3 — Proposal Fragment concept specification

## Summary

**v4.7.3** documents the **Proposal Fragment** concept: partial suggestions for **existing** schema v8 Trips — target, intent, conflicts, and adoption boundaries — complementing [v4.7.2](../specifications/v4.7.2-trip-proposal-envelope-concept-spec.md) whole-trip proposals.

Spec: [v4.7.3-proposal-fragment-concept-spec.md](../specifications/v4.7.3-proposal-fragment-concept-spec.md)

## What changed

- **New:** Proposal Fragment concept specification
- **Updated:** `docs/public/proposals.md` — Fragment entry-level overview
- **Not changed:** export schema, CLI behavior, JSON golden

## Headline decisions

- **Proposal Fragment** — partial candidate for an existing Trip / Day / Itinerary; **not** a small Trip
- **Outside schema v8** until human adoption — not in `trip export`
- **target** — trip / day / itinerary / unresolved (unresolved allowed)
- **intent** — add / enrich / replace_candidate / reorder_hint / warning (conceptual, not a fixed enum)
- **conflicts** — target_missing, time_overlap, stale_source, etc. — warnings for review, not auto-reject
- **Expiry** — same soft-warning policy as v4.7.2
- **Adoption / materialize detail** — deferred to v4.7.4
