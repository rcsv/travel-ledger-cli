# v4.7.5 — Public examples / AI JSON generation guide

## Summary

**v4.7.5** adds **public examples** and an **AI JSON generation guide**: how to author schema v8 Trips vs Trip Proposal Envelopes vs Proposal Fragments, with anti-fabrication rules and validate-export boundaries — following [v4.7.4](../specifications/v4.7.4-materialize-gate-concept-validation-rules.md).

- **Updated:** public docs index, proposals cross-links
- **Not changed:** export schema, CLI behavior, JSON golden

## Headline content

- **schema v8 Trip** — only when dates confirmed and plan adopted; `trip validate-export` applies
- **Trip Proposal Envelope** — whole-trip unadopted draft; not validate-export
- **Proposal Fragment** — partial suggestion for existing Trip; not validate-export
- **AI rules** — no invented bookings/prices/hours; use `missing_fields` / `warnings` / `assumptions`
- **Gate** — human materialize/apply before canonical Trip; AI must not bypass
