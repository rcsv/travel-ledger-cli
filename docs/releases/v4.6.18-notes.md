# v4.6.18 — Note show service boundary

## Summary

**v4.6.18** introduces a minimal **service layer** for the read-only `note show` command, keeping existing CLI human output, JSON shape (`Note`), owner fields, and not-found behavior unchanged while making the use case reusable for future Tauri GUI screens.

Specification: [v4.6.18-note-show-service-boundary.md](../specifications/v4.6.18-note-show-service-boundary.md)

## What changed

- New `src/services/note_show.rs` — `show_note()` returns `NoteShowServiceResult` (no terminal I/O)
- `note show` CLI handler calls service, then routes to existing human/JSON output

## Compatibility

- `note show` human output unchanged (`print_note_detail` with owner_type / owner_id)
- `note show --json` unchanged (`Note` serialization)
- Not found: `Note not found: <id>`
- `note list` / write commands unchanged
- No DB schema or migration changes
- Golden updates: Okinawa Markdown Colophon `Version:` only

## Non-goals

- No `note list` / write command changes
- No `day show` / `reservation show` changes
- No Tauri / GUI implementation
