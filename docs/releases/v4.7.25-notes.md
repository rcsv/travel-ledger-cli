# v4.7.25 — Fragment apply add_expense --confirm (P-6g)

## Summary

**v4.7.25** completes **P-6g confirm path**: `fragment apply --confirm` inserts one Expense for `add_expense` intent on **itinerary target only**, using the same gate as v4.7.24 dry-run.

Spec: [v4.7.25-fragment-apply-add-expense-confirm.md](../specifications/v4.7.25-fragment-apply-add-expense-confirm.md)

## What changed

- **New:** `fragment apply --confirm` — `add_expense` for itinerary target
- **New:** gate report `inserted_expense_id` on confirm success
- **Extended:** `validate_confirm_scope` — supports `add_expense` (itinerary only)
- **Unchanged:** `add_expense --dry-run`, schema v8

## Usage

```bash
caglla fragment apply fragment.json --dry-run --trip 1 --output preview.json
caglla fragment apply fragment.json --confirm --trip 1 --json
caglla expense list --itinerary 1 --json
caglla expense show <id>
```

## Safety

- Expense INSERT only — no update/delete
- trip / day target → blocking
- required decisions / unresolved / ambiguous → no DB write
- `--dry-run --confirm` → error
