# v4.4.0 — Travel Book presentation model review

## Summary

**v4.4.0** is a **documentation-only release**. It records a design review for a **renderer-independent Travel Book presentation model**, using Okinawa Sesoko 2026 `trip export-md` output as the validation fixture. Markdown is treated as one renderer, not the source of truth for display decisions.

Review: [v4.4.0-travel-book-presentation-model-review.md](../specifications/v4.4.0-travel-book-presentation-model-review.md)

## Architectural principle

```text
Domain → Travel Book presentation model → Renderer (Markdown / GUI / Web)
```

## Findings (headline)

- **Domain model** — Trip / Day / Itinerary / Reservation / Estimate / Note responsibilities unchanged
- **Presentation model** — Chapter order, visibility rules, field labels, category display names, reservation dedup/period formatting, planned cost column logic, notes ordering
- **Markdown renderer only** — Heading levels, pipe tables, blank lines, checklist syntax, colophon layout
- **Defer** — Itinerary reservation/cost cues in Daily schedule; schedule role (fixed/flexible/optional); chapter heading i18n; Venue / Maps
- **Do not do** — Large abstraction in v4.4.0; Markdown-only business rules; Expense / Receipt in Travel Book

## Version bump only

`Cargo.toml` / `Cargo.lock` → `4.4.0`. Okinawa golden `Version:` colophon line updated for version alignment. No CLI behavior, schema, or Travel Book body changes.

## Recommended next step

**v4.4.1 — Category display name in Travel Book** — use `ItineraryCategory::definition().display_name` via presentation rules (not Markdown-only Japanese hacks).

## Non-goals

- Code or Travel Book display changes in v4.4.0 (beyond colophon version)
- GUI / native app implementation
- DB schema changes
