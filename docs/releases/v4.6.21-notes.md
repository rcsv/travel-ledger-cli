# v4.6.21 — Expense show service boundary

## Summary

**v4.6.21** introduces a minimal **service layer** for the read-only `expense show` command, keeping existing CLI human output (`print_expense_detail`), JSON shape (`ExpenseJson` via `expense_to_json`), and not-found behavior unchanged while making the use case reusable for future Tauri GUI screens.

Specification: [v4.6.21-expense-show-service-boundary.md](../specifications/v4.6.21-expense-show-service-boundary.md)

## What changed

- New `src/services/expense_show.rs` — `show_expense()` returns `ExpenseShowServiceResult` (no terminal I/O)
- `expense show` CLI handler calls service, then routes to existing human/JSON output helpers

## Compatibility

- `expense show` human output unchanged (`print_expense_detail` via `expense_to_json`)
- `expense show --json` unchanged (`ExpenseJson` via `expense_to_json`)
- Not found: `Expense not found: <id>`
- `expense list` / add / update / delete unchanged
- `itinerary show` / trip stats / Travel Book export unchanged
- Receipt / Participant unchanged
- No DB schema or migration changes
- Golden updates: Okinawa Markdown Colophon `Version:` only

## Non-goals

- No write command service split
- No Tauri / GUI implementation
