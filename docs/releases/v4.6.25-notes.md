# v4.6.25 — Expense output DTO migration plan

## Summary

**v4.6.25** is a **documentation-only migration plan** for Option C from [v4.6.24](../specifications/v4.6.24-expense-dto-context-ownership-review.md): service returns `Expense` + enriched context parts; CLI maps to unchanged `ExpenseJson`. This release completes **Phase 1** (plan only). No code changes.

Plan: [v4.6.25-expense-output-dto-migration-plan.md](../specifications/v4.6.25-expense-output-dto-migration-plan.md)

## Headline

- **Option C** is the migration baseline
- **Short term:** keep `ExpenseJson` / `expense_to_json(&conn, ...)`
- **Next implementation:** Phase 2+3 — enriched service results + CLI mapper (recommended v4.6.26)
- **Gates:** JSON golden unchanged; human output unchanged
- **itinerary show aggregate** remains a separate review topic
