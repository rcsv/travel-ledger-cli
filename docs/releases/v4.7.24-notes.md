# v4.7.24 — Fragment apply add_expense dry-run (P-6g)

## Summary

**v4.7.24** adds **P-6g dry-run path**: `fragment apply --dry-run` previews one Expense append under a resolved Itinerary target, with gate report and optional preview Trip JSON — no DB writes.

Spec: [v4.7.24-fragment-apply-add-expense-dry-run.md](../specifications/v4.7.24-fragment-apply-add-expense-dry-run.md)

## What changed

- **New:** `fragment apply --dry-run` — `add_expense` for **itinerary target only**
- **New:** gate report fields `expenses_before` / `expenses_after` / `expense_preview`
- **Extended:** `fragment validate` — recognizes `add_expense` intent
- **Unchanged:** `add_expense --confirm`, schema v8, JSON golden

## Usage

```bash
caglla fragment apply fragment.json --dry-run --trip 1 --output preview.json
caglla trip validate-export preview.json
caglla fragment apply fragment.json --dry-run --trip 1 --json
caglla expense list --itinerary 1 --json
```

## Safety

- Expense preview only — no INSERT
- trip / day target → blocking (unsupported)
- invalid amount / currency → blocking
- required decisions / unresolved / ambiguous → no DB write
- `--dry-run --confirm` → error

## Compatibility

- **JSON golden:** unchanged
- **Markdown golden:** colophon version only (`4.7.24`)
