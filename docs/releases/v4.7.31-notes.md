# v4.7.31 — Fragment apply delete_itinerary dry-run (P-6j)

## Summary

**v4.7.31** adds **P-6j Phase 1 dry-run path**: `fragment apply --dry-run` previews permanent removal of a resolved **itinerary target** from an in-memory Trip export, with `delete_preview` child counts in the gate report — no DB writes. `--confirm` is not supported in this release.

Spec: [v4.7.31-p6j-delete-itinerary-dry-run.md](../specifications/v4.7.31-p6j-delete-itinerary-dry-run.md) · Policy: [v4.7.30-p6j-destructive-structural-apply-policy.md](../specifications/v4.7.30-p6j-destructive-structural-apply-policy.md)

## What changed

- **New:** `fragment apply --dry-run` — `delete_itinerary` for **itinerary target only**
- **New:** gate report `preview.delete_preview` — target summary + `blocking_children` counts
- **Extended:** `fragment validate` — recognizes `delete_itinerary` intent (empty body allowed)
- **Unchanged:** `delete_itinerary --confirm`, schema v8, JSON golden, DB migration

## Usage

```bash
caglla fragment apply fragment.json --dry-run --trip 1 --output preview.json
caglla trip validate-export preview.json
caglla fragment apply fragment.json --dry-run --trip 1 --json
caglla itinerary list 1
```

## Safety

- Itinerary delete preview only — no INSERT / UPDATE / DELETE
- trip / day target → blocking (unsupported)
- unresolved / ambiguous target → blocking
- `required_decisions` / `adoption_hints.conflicts` → blocking
- blocking children (Expense / Estimate / Reservation / Note entity) → blocking; DB unchanged
- inline `itinerary.note` text → **not** a blocking child
- Venue / Place → **not** blocking (no Venue model in v4.7.31)
- childless itinerary → preview removes itinerary from export; DB unchanged
- `--dry-run --confirm` → error
