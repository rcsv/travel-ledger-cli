# v4.7.22 — Fragment apply add_note dry-run (P-6f)

## Summary

**v4.7.22** implements **P-6f**: `fragment apply --dry-run` support for `add_note` intent — preview Note additions to Trip / Day / Itinerary targets without DB writes.

Spec: [v4.7.22-fragment-apply-add-note-dry-run.md](../specifications/v4.7.22-fragment-apply-add-note-dry-run.md)

## What changed

- **New:** `fragment apply --dry-run` — `intent: add_note` for trip / day / itinerary targets
- **New:** `add_note` in `fragment validate` intent enum
- **New:** gate report preview fields `notes_before` / `notes_after` for add_note
- **New:** `required_decisions` non-empty → `valid: false` on dry-run (preview still available via `--output`)
- **Unchanged:** `add_itinerary --confirm` from v4.7.19〜21
- **Not changed:** `add_note --confirm`, Note DB INSERT, schema v8

## Usage

```bash
caglla fragment apply fragment.json --dry-run --trip 1 --output preview.json
caglla trip validate-export preview.json
caglla fragment apply fragment.json --dry-run --trip 1 --json
```

## Safety

- Dry-run only — no Note INSERT
- `add_note --confirm` blocked (unsupported intent for confirm)
- unresolved / ambiguous target → blocking, DB unchanged
- `required_decisions` non-empty → valid=false, DB unchanged

## Compatibility

- **JSON golden:** unchanged
- **Markdown golden:** colophon version only (`4.7.22`)
- **Gate report JSON:** schema version 2 (optional `notes_before` / `notes_after`)
