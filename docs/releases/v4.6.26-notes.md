# v4.6.26 — Expense output DTO migration Phase 2+3

## Summary

**v4.6.26** implements [v4.6.25](../specifications/v4.6.25-expense-output-dto-migration-plan.md) **Option C Phase 2+3**: read-only `expense list` / `expense show` resolve payer/beneficiary/shared context in the service layer; the CLI maps enriched parts to unchanged `ExpenseJson` / `ExpenseListJson` output.

Specification: [v4.6.26-expense-output-dto-migration-phase-2-3.md](../specifications/v4.6.26-expense-output-dto-migration-phase-2-3.md)

Implementation: `bed654a` — Introduce enriched expense output context for read-only services

## What changed

- `ExpenseEnrichedPart` / `ExpenseBeneficiaryPart` in `src/expense.rs`
- `enrich_expense(conn, &expense)` — DB context resolution for read-only services
- `enriched_expense_to_json(part)` — conn-free mapper to existing `ExpenseJson` shape
- `print_expense_detail_from_enriched` / `print_expense_list_from_enriched` for read-only human output
- `ExpenseListServiceResult` → `Vec<ExpenseEnrichedPart>`
- `ExpenseShowServiceResult` → `ExpenseEnrichedPart`
- `main.rs` expense list/show use enriched service results + mapper

## Compatibility

- JSON output / shape unchanged (`ExpenseJson`, `ExpenseListJson`)
- Human output unchanged for `expense list` / `expense show`
- Not found: `Expense not found: <id>`
- `expense add` / `update` / `delete` unchanged
- `expense_to_json` / `print_expense_detail` / `print_expense_list` retained as write-path adapters
- No DB schema or migration changes
- JSON golden unchanged; Okinawa Colophon `Version:` only

## Non-goals

- Removing `expense_to_json` adapter (Phase 4 candidate)
- Write command changes
- Tauri / GUI
