# v1.8.1 — Travel activity terminology consistency

## Summary

v1.8.1 is a **documentation-only release**. No Rust source, DB schema, or CLI behavior changes.

Building on v1.8.0’s Itinerary model spec, this release **aligns terminology across README, specifications, samples, and release notes** so Caglla.Travel reads as a tool for **planning, executing, and reflecting on trips** — not only as a post-trip log.

## Terminology change

| Before (v1.8.0 wording) | After (v1.8.1) |
|---|---|
| Itinerary is an **event** | Itinerary is a **unit of travel activity** |
| 出来事 | **行動** / **行動単位** |
| event ledger | **行動台帳** |

**Unchanged principle:** **Itinerary is not a venue.** Place / `location` remains optional.

### Core message (Japanese)

> Itinerary は旅行中の行動を表す最小単位である。  
> 計画時には予定として、旅行後には実績として扱える。

Caglla.Travel treats **before the trip (plan)**, **during the trip (execute)**, and **after the trip (reflect)** in one model. “出来事” implied past-only nuance; “行動” covers future plans and recorded activities alike.
