# v4.6.43 — Release workflow asset upload follow-up

## Summary

**v4.6.43** is a housekeeping release documenting Release workflow / asset upload procedures after v4.6.41 (macOS manual upload) and v4.6.42 (successful workflow-first approach). Includes a minimal `release.sh` fix: edit title/notes **after** assets are ready.

Spec: [v4.6.43-release-workflow-asset-upload-follow-up.md](../specifications/v4.6.43-release-workflow-asset-upload-follow-up.md)

## What changed

- **Docs:** release procedure, asset verification, macOS recovery, follow-up commit timing
- **CONTRIBUTING.md:** align release/version rules with repository practice (includes prior `4378a0d` commit)
- **`release.sh`:** wait for 3 OS assets before `gh release edit` (avoids v4.6.41 race)
- **Workflow YAML:** unchanged

## Headline procedure

```text
tag push → Release workflow (3 parallel jobs) → wait assets → edit title/notes → verify
```

Do **not** run `gh release edit` while the Release workflow is still uploading assets.
