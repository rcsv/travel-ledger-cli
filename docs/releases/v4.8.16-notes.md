# v4.8.16 — Receipt / inbox CLI strict currency integration

## Summary

**v4.8.16** applies `CurrencyValidationMode::IsoStrict` to **Receipt / inbox CLI currency write paths**:

- `receipt add` — `--currency`（`--amount` とセット、常に strict）
- `receipt update` — `--currency` 明示時のみ strict
- `receipt assign` — CLI `--currency` 明示時のみ strict

Cargo version bumped to `4.8.16`. Markdown colophon synced. Domain layer, import, read, and export behavior unchanged.

Implementation spec: [v4.8.16-receipt-inbox-cli-strict-currency-integration.md](../specifications/v4.8.16-receipt-inbox-cli-strict-currency-integration.md)

## CLI behavior change

- Valid ISO codes succeed with uppercase normalization
- Unknown registry / denylist / format-invalid currency rejected on Receipt CLI write
- `receipt update` without `--currency` does not re-validate existing currency
- `receipt assign` using stored receipt currency does not re-strict existing values

## Compatibility

- `receipt.rs` domain layer / trip import / validate-export / read / export: unchanged
- DB schema / JSON golden: unchanged
- memo-only update and assign with existing receipt currency: non-breaking

## Format-only surfaces (unchanged)

- `trip import` / `import_receipt_v7`
- validate-export receipt checks
- read / list / show / export / export-md
