## Planning conclusions

- **Additive introduction:** optional `structured_errors[]`; keep existing `errors[]` for backward compatibility
- **Stable `code` + unstable `message`:** JSON codes for API/GUI; human CLI text remains free to evolve
- **Shared registry:** dry-run and confirm use the same code set; `phase` distinguishes gate vs transaction
- **`required_decisions`:** separate `kind=decision_required` from field-level validation errors
- **Boundaries:** retry token / ETag / strict idempotency remain out of scope for this planning track
