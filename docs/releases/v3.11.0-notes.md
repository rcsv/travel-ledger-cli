# v3.11.0 — DB Use implementation

## Summary

**v3.11.0** implements **`caglla db use`** — persist the default database path in CWD `./caglla.toml` for v3.9.0 DB path resolution Phase 2. Requires **v3.10.0** design baseline. Trip export still writes **`schema_version: 8`**.

Design: [v3.10.0-db-use-concept-design.md](../specifications/v3.10.0-db-use-concept-design.md)  
Implementation: [v3.11.0-db-use-implementation-plan.md](../specifications/v3.11.0-db-use-implementation-plan.md)

---

## Added

### `caglla db use <path>`

Saves `[database].path` to `./caglla.toml` in the current working directory.

```bash
caglla db use ./data/okinawa.db
```

- Does **not** open or create the SQLite file
- Prints config path and saved path
- Notes when the target DB file does not exist yet (exit 0)

### `caglla db use --clear`

Removes `[database].path` from config. Deletes `caglla.toml` when no other settings remain. Without overrides, resolution falls back to default `./caglla.db`.

### Path storage

- Prefers portable relative paths (`./data/app.db`, `../shared/other.db`)
- Relativizes absolute paths under CWD when possible
- Preserves unknown TOML keys; comments are not guaranteed on rewrite

### Resolution unchanged

```text
--db > CAGLLA_DB > caglla.toml > ./caglla.db
```

`db use` updates the config layer only — not a per-command override like `--db`.

---

## Unchanged

| Item | Notes |
|---|---|
| SQLite schema / migrations | No changes |
| `trip export` JSON schema | Still **v8** |
| `db path` / `db status` on missing DB | Still do **not** create the database file |
| Parent-dir config search | Not implemented |
| User-global config / profiles | Not implemented |

---

## Upgrade notes

After `db use`, run `caglla db path` to confirm `Path source: config`. Use `--db` or `CAGLLA_DB` when you need a one-off override.
