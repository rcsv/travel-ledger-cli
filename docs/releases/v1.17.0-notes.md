# v1.17.0 — Summary implementation

## Summary

Implements **Trip / Day Summary** — short, reader-oriented overview text for sharing, guidebooks, and export roundtrip.

Design series (completed before this release):

```text
v1.14.0  Responsibilities Review
v1.15.0  Entity Design
v1.16.0  Implementation Plan
v1.17.0  Implementation  ← this release
```

## Added

### Database

| Column | Change |
|---|---|
| `trips.summary` | `TEXT NULL` — new column via migration |
| `days.summary` | `days.description` **renamed** to `summary` (existing DB migration) |

- Existing rows remain `NULL` — no backfill
- Empty string input is normalized to `NULL` at the application layer

### CLI — Trip

| Command | Option / behavior |
|---|---|
| `trip add` | `--summary` (optional) |
| `trip update` | `--summary`, `--clear-summary` |
| `trip show` | Displays summary section when set |
| `trip show --json` | `"summary": null \| "..."` |

### CLI — Day

| Command | Option / behavior |
|---|---|
| `day update` | **New command** — `--summary`, `--clear-summary` |
| `day show` | Displays summary section when set |
| `day show --json` | `"summary"` field |

`trip list` / `day list` (text) do **not** show summary in the first cut.

### Export / import (schema v3)

- `trip.summary` in export JSON
- `days[].summary` in export JSON
- Omitted keys when `NULL` (`skip_serializing_if`)
- Legacy v3 import without `summary` keys → `NULL` (backward compatible)
- `trip duplicate` preserves summary via export/import roundtrip
- `validate-export` checks summary length

### Markdown export

`trip export-md` outputs:

- Trip summary under `# Title` (when set)
- Day summary under `## Day N` (when set)
- Plain text only — no Markdown interpretation

### Diff

`trip diff` compares:

- `trip.summary`
- `Day N summary` (per `day_number`)

## Validation

| Rule | Behavior |
|---|---|
| trim | Leading/trailing whitespace removed |
| empty → NULL | `""` or whitespace-only → stored as `NULL` |
| length | Trip: 2000 chars max; Day: 1000 chars max |
| Markdown | Not interpreted on input |

## Breaking changes

| Change | Impact |
|---|---|
| **`day list --json`**: `description` → **`summary`** | JSON consumers of `day list --json` must use the `summary` key |

No breaking change for export v3 (summary keys are new optional fields).

## Not in this release

- `Itinerary.summary` — `itinerary.title` remains the action label
- Canonical sample (`okinawa_sesoko_2026`) — **no summary data added** (optional future)
- `trip stats` / `trip doctor` — summary not included

## CLI direction (future)

Caglla CLI will continue to prioritize **inspection, validation, and integration** commands:

```text
show / list / validate / export / import / diff / export-md / doctor / stats
```

Fine-grained hand-entry edit commands may become **deprecated / obsolete candidates** once a mature GUI exists. Summary editing via CLI remains supported in v1.17.0; this note describes long-term product direction only.

See also: [long-term-version-strategy.md](../long-term-version-strategy.md)
