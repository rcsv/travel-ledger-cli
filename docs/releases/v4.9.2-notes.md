# v4.9.2 — Desktop readiness service facade

## Summary

**v4.9.2** introduces a read-only service facade for Desktop readiness:

- Public API: `list_trip_summaries` / `get_trip_detail` / `get_day_timeline`
- UI-independent DTOs: `TripSummary`, `TripDetail`, `DaySummary`, `DayDetail`, `ItineraryDetail`
- Structured read errors: `ServiceError` + `ReadServiceErrorCode`
- **Library target** (`travel_ledger_cli`): `open_db` + read facade importable from external crates
- CLI read migration: `trip list/show`, `day list/show` reuse facade via adapters
- Legacy `trip_list/show` / `day_list/show` modules removed

Cargo version bumped to `4.9.2`. Markdown colophon synced. No export/import schema changes. No DB migration.

Implementation spec: [v4.9.2-desktop-read-service-facade.md](../specifications/v4.9.2-desktop-read-service-facade.md)

## Library boundary

```rust
use travel_ledger_cli::{open_db, list_trip_summaries, get_trip_detail, get_day_timeline};
```

Tauri / workspace setup is deferred; the crate is import-ready.
