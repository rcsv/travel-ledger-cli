# v4.1.1 — Okinawa Travel Book sample enrichment plan

## Summary

**v4.1.1** is a **documentation-only release**. It plans **Okinawa canonical sample** enrichment so `samples/okinawa_sesoko_2026/` can serve as both a **post-trip ledger** fixture and a **Travel Book** fixture before v4.2.0 `export-md` layout work. Defines entity targets, responsibility boundaries, itinerary mapping (Day + sort_order + title), and golden/test update policy. No seed, CLI, SQLite, or export schema changes.

Requires **v4.1.0** baseline. Trip export still writes **`schema_version: 8`**.

Design: [v4.1.1-okinawa-travel-book-sample-enrichment-plan.md](../specifications/v4.1.1-okinawa-travel-book-sample-enrichment-plan.md)

## Added

### Okinawa enrichment plan

[v4.1.1-okinawa-travel-book-sample-enrichment-plan.md](../specifications/v4.1.1-okinawa-travel-book-sample-enrichment-plan.md):

- **Dual role** — ledger validation unchanged; add Summary / Note / Reservation / Estimate for Travel Book chapters
- **Targets** — Trip summary (1), Day summaries (4), Trip notes (2), Day notes (4), Reservations (5), Estimates (7–9)
- **Boundaries** — Summary = Abstract; Note = annotation; Reservation = real arrangements only; Estimate = planned-cost fixture with explicit notes; Receipt unchanged and not shown in Travel Book
- **Reservation confirmation** — only when evidenced in ledger (e.g. parking `EXAMPLE-PARKING-001`); no fabricated codes
- **Itinerary mapping** — canonical key: Day + sort_order + title; reference IDs from current seed
- **Golden policy (v4.1.2)** — add `trip.summary` to normalize; compare `days[].summary`, nested reservations/estimates, `notes[]`
- **Invariants** — 58 itineraries, 49 expenses, ¥561,780 Actual, 6 receipts unchanged
