# v4.6.41 — Reservation write service Phase R-2+R-3 implementation

## Summary

**v4.6.41** implements [v4.6.40](../specifications/v4.6.40-reservation-write-service-migration-plan.md) Phase R-2+R-3: thin write services for `reservation add` / `update` / `delete` and CLI handler wiring.

Spec: [v4.6.41-reservation-write-service-phase-r2-r3.md](../specifications/v4.6.41-reservation-write-service-phase-r2-r3.md)

## What changed

- **New services:** `reservation_add`, `reservation_update`, `reservation_delete`
- **Add/update result:** `Reservation` + `day_number` / `itinerary_title` (same optional semantics as read `reservation_show`; no enriched layer)
- **Delete result:** pre-delete `{ id, provider_name }` snapshot
- **Shared helper:** `load_reservation_display_context` in `reservation.rs` (show + write)
- **Clear flags:** interpreted inside `reservation_update` service; handler passes raw clap mirror fields
- **Handler:** thin service calls; success `println!` + `print_reservation_detail_with_context` (add/update)
- **Adapter:** write path no longer calls `print_reservation_detail(&conn, ...)` — function retained for R-5 (v4.6.42)

## Compatibility

- **JSON golden:** unchanged
- **Human output:** unchanged (`reservation_cli` tests pass)
- **`import_reservation_v3`:** unchanged
