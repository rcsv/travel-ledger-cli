# v4.8.11 — Currency ISO validation hardening planning

## Summary

**v4.8.11** is a **documentation-only** formal release: planning for Issue #66 **Currency ISO validation hardening**.

No Rust code, tests, fixtures, CLI behavior, JSON report shape, or DB schema changes beyond Markdown colophon version sync. Cargo version bumped to `4.8.11`.

Planning spec: [v4.8.11-currency-iso-validation-hardening-planning.md](../specifications/v4.8.11-currency-iso-validation-hardening-planning.md)

## Planning conclusion

- **P0 / P1: none** — staged hardening is non-breaking under documented compatibility policy
- **DB schema unchanged** — `currency TEXT` maintained
- **Read / export / export-md non-breaking** — legacy unknown currency values preserved
- **Create / update / apply strictening** — new input paths enforce ISO 4217 registry + denylist (staged v4.8.12+)
- **Import vs validate-export split** — import stays format-only by default; validate-export adds warnings for unknown/denylisted codes
- **Minor units out of scope** — `currency_minor_unit_digits` ISO-backed migration deferred to v4.8.16+ optional
- **Structured errors** — Fragment apply uses existing `APPLY_FIELD_INVALID`; no new code at planning stage
- **`schema_version: 2` unaffected** — Fragment apply report additive only

## Current state inventory

- `validate_currency_code()` — trim, uppercase, ASCII alpha-3 only; unknown codes allowed (including `XXX`)
- Surfaces: expense/estimate CLI, trip import/validate-export, receipt inbox, Fragment apply parse paths
- Samples/golden: primarily `JPY`, `USD`

## Proposed validation policy

- ISO 4217 alpha-3 registry (active + withdrawn) for write paths
- Denylist: `XXX`, `XTS`, precious metals (`XAU`, etc.)
- Lowercase → uppercase normalize (unchanged)
- Historical/withdrawn codes allowed on write

## Backward compatibility

- Existing DB unknown currency: readable, exportable, not migrated
- Import: no blanket reject; optional strict flag deferred to v4.8.15
- validate-export: warnings (not errors) for registry unknown / denylist
