# v4.6.32 — Expense write service migration plan

## Summary

**v4.6.32** is an **implementation-ready documentation-only migration plan** (Phase W-1) building on [v4.6.31](../specifications/v4.6.31-expense-write-path-migration-plan.md) (Phase W-0). It fixes service API contracts, result types, phase gates, and defer boundaries for thin write services.

Plan: [v4.6.32-expense-write-service-migration-plan.md](../specifications/v4.6.32-expense-write-service-migration-plan.md)

## Headline decisions

- **Services:** `expense_add` / `expense_update` / `expense_delete` (1 command = 1 module)
- **Add/update result:** `ExpenseEnrichedPart` — same as read `expense_show` (not `expense_id` only, not `ExpenseJson`)
- **Delete result:** `{ id, amount, currency }` — no enrich; amount-only display unchanged
- **Handler keeps:** success `println!`, `print_expense_detail_from_enriched` (add/update)
- **`parse_*`:** called from service only; functions stay in `expense.rs` initially
- **`import_expense_v3`:** separate path — commonization deferred (W-7+)
- **`last_insert_rowid` → `RETURNING id`:** Phase W-H optional hardening after W-3 (low priority)
- **Repository / DB FK:** out of scope — defer to v4.7.x / migration runner track
- **Next implementation:** v4.6.33 — Phase W-2+W-3 together (recommended)
