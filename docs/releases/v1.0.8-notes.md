# v1.0.8 Export metadata

## Summary

- Added optional export metadata fields: `generator`, `generator_version` (alongside existing `exported_at`).
- Extended `validate-export` text/JSON output with export metadata.
- Extended `trip import` summary with an `Export:` section.
- Import logic and validation semantics (`valid` / `warnings`) are unchanged.

## Added

### Export metadata fields

New optional root fields on `trip export` JSON:

| Field | Value |
|---|---|
| `generator` | `"caglla-cli"` |
| `generator_version` | CLI version at export time (`CARGO_PKG_VERSION`) |
| `exported_at` | unchanged (RFC3339, set at finalize) |

Example:

```json
{
  "schema_version": 1,
  "generator": "caglla-cli",
  "generator_version": "1.0.8",
  "exported_at": "2026-06-07T12:34:56Z",
  "trip": { "...": "..." },
  "itinerary_items": [],
  "checklist_items": []
}
```

Metadata is set in `finalize_trip_export()` only; `build_trip_export()` leaves these fields unset.

### trip validate-export — Metadata section

Text output now includes (after Summary, before Warnings):

```text
Metadata:
  Generator : caglla-cli
  Version   : 1.0.8
  Exported  : 2026-06-07T12:34:56Z
```

Legacy files (missing keys):

```text
Metadata:
  Generator : 不明
  Version   : 不明
  Exported  : 不明
```

`--json` adds flat fields (always present; `null` when the key is missing in the file):

```json
{
  "generator": "caglla-cli",
  "generator_version": "1.0.8",
  "exported_at": "2026-06-07T12:34:56Z"
}
```

### trip import — Export section

Import success output now includes (after Schema):

```text
Export:
  generator : caglla-cli
  version   : 1.0.8
```

Legacy imports:

```text
Export:
  generator : 不明
  version   : 不明
```

`exported_at` is not shown in the import summary.

## Warning policy (validate-export)

Unchanged from v1.0.7 for metadata:

| Condition | Warning? |
|---|---|
| Missing `generator` | No |
| Unknown `generator` value | No |
| Old `generator_version` | No |
| Invalid `exported_at` (RFC3339) | Yes |

Generator metadata is informational only; it does not affect `valid` or import.

## Compatibility

- Legacy export files (no `generator` / `generator_version`) remain importable.
- `generator: "unknown"` and old version strings are accepted on import.
- `trip diff` / roundtrip comparison ignores generator metadata (`comparable_trip_export`).
- No database schema changes.
