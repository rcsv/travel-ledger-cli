# v4.7.16 — Proposal materialize dry-run (P-6a)

## Summary

**v4.7.16** implements **P-6a**: `proposal materialize <file> --dry-run` — generate a schema v8 Trip JSON **candidate** from a Trip Proposal Envelope without DB writes.

Spec: [v4.7.16-proposal-materialize-dry-run.md](../specifications/v4.7.16-proposal-materialize-dry-run.md)

## What changed

- **New:** `proposal materialize <file> --dry-run` with optional `--output`, `--start`, `--end`, `--json`
- **New:** `ProposalMaterializeDryRunReport` — envelope validation + materialize gate + `trip_export_valid`
- **New:** `candidate_days` / `candidate_itineraries` → schema v8 `days` / `itineraries` mapping
- **New:** integration tests — generated JSON passes `trip validate-export`
- **Not changed:** `--confirm`, DB insert, Fragment apply, schema v8, JSON golden

## Usage

```bash
caglla proposal materialize tests/fixtures/proposals/materialize-ready-envelope.json --dry-run --output /tmp/candidate-trip.json
caglla trip validate-export /tmp/candidate-trip.json

# flexible_dates envelope — supply dates via CLI
caglla proposal materialize envelope.json --dry-run --start 2026-05-01 --end 2026-05-01
```

## Safety pattern

```text
proposal validate → materialize --dry-run → schema v8 JSON → trip validate-export
```

File-only dispatch before DB open (same as `proposal validate`).

## Output streams

| Mode | stdout | Trip JSON candidate |
|---|---|---|
| human, no `--output` | gate report then Trip JSON on stdout (preview only) | mixed on stdout — not for piping |
| `--output <file>` | gate report only | **file** — use for `trip validate-export` |
| `--json` | gate report JSON only | file when `--output` set; otherwise not emitted |

Trip JSON candidate for downstream commands: use `--output`. `--json` is the gate report, not the Trip export.

## Compatibility

- **JSON golden:** unchanged
- **Markdown golden:** colophon version only (`4.7.16`)
- **Runtime:** no DB migration; no auto-adopt
