# v4.7.35 — P-6k reorder_itinerary dry-run (same-day)

## Summary

**v4.7.35** adds **dry-run** support for the structural apply intent **`reorder_itinerary`** (same-day only).
It produces a structured preview of the **Day itinerary order changes** by computing proposed `after_sort_order` values **without mutating the DB**.

Key points:

- **same-day only** (Day internal reorder). cross-day move is **rejected / deferred** (future intent).
- **Sequence-first ordering**: `sort_order` is the ordering source of truth. `start_time` is a label and is not modified.
- **expected_order required**: baseline mismatch / ambiguity is blocking.
- **sparse `sort_order` preserved**: reuses existing Day `sort_order` slots for preview calculation.
- **confirm remains blocked**: `reorder_itinerary --confirm` is not implemented in v4.7.35 (no DB mutation path).
- **GUI-ready selectors**: `expected_order` / `after_order` accept **ID-based arrays** for future drag-and-drop.

## What changed

- **Added:** `fragment apply --dry-run` preview for `reorder_itinerary`
- **Added:** structured `preview.reorder_preview` with `itinerary_order_changes`
- **Added:** fixtures + CLI integration tests for valid reorder, invalid inputs, cross-day rejection, and confirm blocking

## Preview shape (excerpt)

```json
{
  "preview": {
    "action": "reorder_itinerary",
    "reorder_preview": {
      "day_number": 2,
      "itinerary_order_changes": [
        {
          "itinerary_id": 101,
          "title": "Example itinerary",
          "before_sort_order": 1000,
          "after_sort_order": 2000
        }
      ]
    }
  }
}
```
