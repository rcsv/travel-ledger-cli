# v3.3.0 — Planned vs Actual difference

## Summary

**v3.3.0** adds **Trip-level Planned vs Actual difference** as a **derived** display layer. No DB schema or export/import schema changes.

Requires **v3.2.1** baseline. Current trip export still writes **`schema_version: 6`**.

Design series:

```text
Implementation Plan            → v3.3.0-planned-vs-actual-implementation-plan.md
Phase 1–2 Implementation       → PR #60
Post-Implementation Review     → v3.3.0-planned-vs-actual-post-implementation-review.md
Release v3.3.0                 → this release
```

### Terminology

| Term | Meaning |
|---|---|
| **Planned** | Sum of `Estimate.amount` by currency (v3.1.0) |
| **Actual** | Sum of `Expense.amount` by currency (existing) |
| **Difference** | `Actual − Planned` by currency — **new in v3.3.0** |

| Sign (Actual − Planned) | Meaning |
|---|---|
| **negative** | Spent less than planned |
| **positive** | Spent more than planned |
| **zero** | Matched planned amount |

---

## Added

### `trip stats` (human)

When **both** Estimates and Expenses exist on the trip, a new block appears after Planned / Actual totals:

```text
Difference:
  JPY -7,500
```

### `trip stats --json`

Additive field when the display gate passes:

```json
"difference_totals": { "JPY": -7500 }
```

Omitted (not `null`) when gate does not pass.

### `trip export-md` Overview

Overview section includes `- Difference:` with per-currency lines when the gate passes. Itinerary-level planned / actual / difference cards are **not** added in v3.3.0.

---

## Display rule

Difference is shown **only** when:

```text
estimate_count > 0 && expense_count > 0
```

Estimate-only or Expense-only trips do **not** show Difference.

---

## Currency rules

- Per-currency difference (no FX conversion)
- Keys = union of `estimate_totals` and `expense_totals`
- Missing side treated as **0** for that currency

Example:

```text
Planned:  JPY 10,000 / USD 0.50
Actual:   JPY  9,500
Difference: JPY -500 / USD -0.50
```
