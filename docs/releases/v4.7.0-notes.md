# v4.7.0 — Schema-publication / Travel Ledger public direction concept review

## Summary

**v4.7.0** opens the v4.7.x chapter: a **documentation-only concept review** for Travel Ledger as a public schema, reference implementation, future GUI consumer, and AI/provider proposal integration — after v4.6.x internal CLI boundary work is complete.

Spec: [v4.7.0-schema-publication-travel-ledger-public-direction-concept-review.md](../specifications/v4.7.0-schema-publication-travel-ledger-public-direction-concept-review.md)

## Headline decisions

- **Travel Ledger** — canonical trip data format; **CLI** is reference implementation / validator / converter
- **Current canonical export** — **schema v8** (nested Trip/Day/Itinerary model established at v3+)
- **Publication path** — grow public docs inside CLI repo first; **no immediate repo split**
- **Proposal Envelope** — **outside** schema v8 Trip; undated proposals do not become Trips without materialize
- **Materialize** — adoption gate: Proposal → formal Trip with dates
- **GUI** — future schema consumer via service layer (v4.6.x groundwork)
- **Expiry** — `valid_until` or default `created_at + 1 year`; expired = warning not hard block
