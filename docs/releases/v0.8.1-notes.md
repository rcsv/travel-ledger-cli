# v0.8.1 - Advisor command hints

## Added

- Added `--with-commands` option to `trip advisor`.
- Added `Try` sections that show copyable CLI command examples for advisor issues.
- Added command hints for:
  - EmptyItinerary
  - NoRestaurant
  - HighTravelTime
  - MissingDuration
  - OverloadedDay
- Added `generate_command_hints()` for rule-based command hint generation.
- Added `extract_day_from_message()` and `day_for_issue()` to reflect `Day N` in command examples.
- Added `samples/advisor/generate_outputs_with_commands.sh`.
- Added `samples/advisor/outputs_with_commands/` validation outputs.

## Changed

- Updated README with `trip advisor --with-commands`.
- Documented the CLI command structure:
  - `trip ...` for trip-level commands
  - `itinerary ...` for itinerary operations
  - `checklist ...` for checklist operations
- Updated advisor verification summary for v0.8.1.

## Fixed

- Adjusted restaurant command hints to use executable two-step commands:
  - `itinerary add ... "Lunch"`
  - `itinerary update <itinerary_id> --category restaurant`
- Avoided invalid `itinerary add --category ...` command hints.

## Quality

- `cargo fmt` passed.
- `cargo clippy -- -D warnings` passed.
- `cargo test` passed: 101 passed, 0 failed.
- `cargo build` passed.

## Notes

This release improves `trip advisor` from advice-only output to actionable command guidance, while keeping the default v0.8.0 advisor output unchanged.
