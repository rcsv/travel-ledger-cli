# v4.8.18 — validate-export Receipt currency warnings

## Summary

**v4.8.18** extends `trip validate-export` currency quality warnings to **Receipt export data**:

- Export JSON `receipts[].currency` — unknown registry / travel-expense denylist codes → **warning**
- Target path: `receipts[{i}].currency`（schema ≥ v7）
- Format-invalid currency remains **error** (existing validate-export contract)
- Warning-only reports keep `valid: true`

Cargo version bumped to `4.8.18`. Markdown colophon synced. DB schema, import, read, and export behavior unchanged. `EXPORT_VALIDATION_REPORT_SCHEMA_VERSION` unchanged.

Implementation spec: [v4.8.18-validate-export-receipt-currency-warnings.md](../specifications/v4.8.18-validate-export-receipt-currency-warnings.md)

## validate-export policy (Receipt)

| Condition | Severity |
|---|---|
| format invalid（空/英字3文字でない） | error（既存維持） |
| registry unknown（例: `ZZZ`, `ABC`） | warning |
| denylist（例: `XXX`, `XTS`, `XAU`） | warning |
| valid ISO | pass |

## Compatibility

- `trip import` still accepts unknown receipt currencies (format-only)
- No JSON schema breaking changes
- No new structured warnings (plain `warnings[]` strings)
- JSON golden: unchanged
