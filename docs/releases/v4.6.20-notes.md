# v4.6.20 — Reservation show service boundary

## Summary

**v4.6.20** introduces a minimal **service layer** for the read-only `reservation show` command, keeping existing CLI human output, JSON shape (`Reservation`), and not-found behavior unchanged while making the use case reusable for future Tauri GUI screens.

Specification: [v4.6.20-reservation-show-service-boundary.md](../specifications/v4.6.20-reservation-show-service-boundary.md)

## What changed

- New `src/services/reservation_show.rs` — `show_reservation()` returns `ReservationShowServiceResult` (no terminal I/O)
- `reservation show` CLI handler calls service, then routes to existing human/JSON output

## Compatibility

- `reservation show` human output unchanged (existing detail display + optional itinerary context)
- `reservation show --json` unchanged (`Reservation` serialization)
- Not found: `Reservation not found: <id>`
- `reservation list` / add / update / delete unchanged
- `itinerary show` Reservations section unchanged
- No DB schema or migration changes
- Golden updates: Okinawa Markdown Colophon `Version:` only

## Non-goals

- No `expense show` changes
- No other write command service split
- No Tauri / GUI implementation
