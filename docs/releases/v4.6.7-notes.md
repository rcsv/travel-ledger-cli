# v4.6.7 — Day list service boundary

## Summary

**v4.6.7** introduces a minimal **service layer** for the read-only `day list` command, keeping existing CLI human/JSON output and Trip not-found behavior unchanged while making the use case reusable for future Tauri GUI Day overview screens.

Specification: [v4.6.7-day-list-service-boundary.md](../specifications/v4.6.7-day-list-service-boundary.md)

## What changed

- New `src/services/day_list.rs` — `list_days()` returns `DayListServiceResult` (no terminal I/O)
- `day list` CLI handler calls service, then routes to `print_day_list_json` / `print_day_list_display`
- Removed DB-coupled `day::run_day_list`

## Compatibility

- `day list` human output unchanged
- `day list --json` output unchanged (`DayListJson`)
- Day ordering unchanged (`ORDER BY day_number`)
- Trip not found: `Trip not found: <id>`
- No DB schema or migration changes
- Golden updates: Okinawa Markdown Colophon `Version:` only

## Non-goals

- No `day show` / `day update` / `day swap`
- No itinerary list/timeline service split
- No Tauri / GUI implementation
