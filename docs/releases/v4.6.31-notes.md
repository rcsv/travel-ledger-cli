# v4.6.31 — Expense write path migration plan

## Summary

**v4.6.31** is a **documentation-only migration plan** after [v4.6.30](../specifications/v4.6.30-expense-write-path-boundary-review.md). It defines phased migration of expense add/update/delete to thin write services (Option W-B).

Plan: [v4.6.31-expense-write-path-migration-plan.md](../specifications/v4.6.31-expense-write-path-migration-plan.md)

## Headline decisions

- **Pattern:** 1 command = 1 service module (`expense_add` / `expense_update` / `expense_delete`)
- **Handler keeps:** clap args, success `println!`, human display
- **Service owns:** parse shared options, mutation orchestration, enriched result (add/update)
- **Delete result:** snapshot `{ id, amount, currency }` — no enrich; amount-only display unchanged
- **Repository layer:** out of scope — defer to v4.7.x
- **Adapter:** `print_expense_detail` until W-3; then `_from_enriched`; cleanup at W-5
- **Next implementation:** v4.6.32 — Phase W-2+W-3 together (recommended)

## Phases (summary)

| Phase | Scope |
|---|---|
| W-0 | v4.6.31 plan (this release) |
| W-1 | CLI unchanged |
| W-2 | Add thin write services |
| W-3 | Wire handlers (human output gate) |
| W-4/W-5 | Optional cleanup |
| W-6 | GUI/Tauri (v4.7.x) |

## Non-goals

- Implementation, repository, output changes
