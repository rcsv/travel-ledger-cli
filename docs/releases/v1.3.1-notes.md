# v1.3.1 Transaction-safe cascade deletion

## Summary

- Wrapped Note cascade deletes and parent row changes in **database transactions** for safer atomicity.
- Fixes a case where `trip update` (period shrink) could persist Trip date changes even when Day deletion failed partway through.
- No CLI behavior changes, no DB schema changes, no export format changes.

## Added

- **`db::with_transaction`** — shared helper to run delete/update steps in one transaction with rollback on error
- **Transaction boundaries** on:
  - `trip delete` — Note cascade + Trip row delete
  - `trip update` — Trip row update + Day sync (including shrink Day/Note deletion)
  - `itinerary delete` — Itinerary Note delete + Itinerary row delete
- **Tests** — rollback on shrink failure, `with_transaction` unit test, CLI integration tests

## Improved

- **Period shrink atomicity** — if a later Day cannot be removed (e.g. Itinerary on Day 3), earlier Day/Note deletions and Trip date updates are rolled back
- **Delete safety** — partial Note deletion without parent deletion (or the reverse) is avoided within each targeted operation
