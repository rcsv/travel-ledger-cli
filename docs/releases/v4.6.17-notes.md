# v4.6.17 — Checklist show service boundary

## Summary

**v4.6.17** introduces a minimal **service layer** for the read-only `checklist show` command, keeping existing CLI human output, JSON shape (`ChecklistItem`), checked/unchecked state, and not-found behavior unchanged while making the use case reusable for future Tauri GUI screens.

Specification: [v4.6.17-checklist-show-service-boundary.md](../specifications/v4.6.17-checklist-show-service-boundary.md)

## What changed

- New `src/services/checklist_show.rs` — `show_checklist()` returns `ChecklistShowServiceResult` (no terminal I/O)
- `checklist show` CLI handler calls service, then routes to existing human/JSON output

## Compatibility

- `checklist show` human output unchanged (`print_checklist_detail` with `[x]` / `[ ]` marks)
- `checklist show --json` unchanged (`ChecklistItem` serialization)
- Not found: `Checklist item not found: <id>`
- `checklist list` / write commands / auto-generate unchanged
- No DB schema or migration changes
- Golden updates: Okinawa Markdown Colophon `Version:` only

## Non-goals

- No `checklist list` / write command changes
- No auto-generate changes
- No Tauri / GUI implementation
