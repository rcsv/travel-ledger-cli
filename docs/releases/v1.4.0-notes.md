# v1.4.0 Export Schema v2 (Notes)

## Summary

- **Export schema v2** — `schema_version: 2` と `notes[]` を追加
- **Note export / import** — Trip / Day / Itinerary Note をバックアップ・復元可能に
- **v1 後方互換** — `schema_version` 未指定 / `1` の export は引き続き import 可能
- **`validate-export` 拡張** — Note と schema 関連の検証を追加

## Added

- **`notes[]`** in trip export JSON (schema v2)
- **Note export formats** — `owner_type: trip | day | itinerary` with `day_number` / `itinerary_key` references (no DB ids)
- **Note import** — restores Notes after Trip / Itinerary / Checklist creation
- **`trip validate-export`** — `notes` check, Note field validation, itinerary_key resolution checks
- **Import summary** — `Note : N 件` line
- **Docs** — [export-schema.md](../specifications/export-schema.md), README updates

## Schema

| Topic | v1.4.0 behavior |
|---|---|
| **Current export** | `schema_version: 2`, includes `notes` (may be `[]`) |
| **v1 import** | `schema_version` omitted or `1`, no `notes` — still supported |
| **Itinerary Note key** | Resolved by day+sort_order → day+start_time+title → day+title |

## Not in this release

- Photo, Expense, Itinerary Checklist, Shared Expense, Multi Currency
- `trip diff` Note comparison (deferred)

## Breaking Changes

None for import. Export output changes from schema v1 to v2 (new `schema_version` value and `notes` key).

## Verification

- `make check`
- Export → Import → Export roundtrip preserves Notes
- v1 export files still import successfully
