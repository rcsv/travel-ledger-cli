# v4.7.23 — Fragment apply add_note --confirm (P-6f)

## Summary

**v4.7.23** completes **P-6f confirm path**: `fragment apply --confirm` inserts one Note for `add_note` intent on trip / day / itinerary targets, using the same gate as v4.7.22 dry-run.

Spec: [v4.7.23-fragment-apply-add-note-confirm.md](../specifications/v4.7.23-fragment-apply-add-note-confirm.md)

## What changed

- **New:** `fragment apply --confirm` — `add_note` for trip / day / itinerary targets
- **New:** gate report `inserted_note_id` on confirm success
- **Extended:** `validate_confirm_scope` — supports `add_itinerary` (day) and `add_note` (trip/day/itinerary)
- **Unchanged:** `add_note --dry-run`, `add_itinerary --confirm`, schema v8

## Usage

```bash
caglla fragment apply fragment.json --dry-run --trip 1 --output preview.json
caglla fragment apply fragment.json --confirm --trip 1
caglla note list --trip 1 --json
caglla note show <id>
```

## Safety

- Note INSERT only — no update/delete
- required decisions / unresolved / ambiguous → no DB write
- `--dry-run --confirm` → error
