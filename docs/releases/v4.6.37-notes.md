# v4.6.37 — Note write service Phase N-2+N-3 implementation

## Summary

**v4.6.37** implements [v4.6.36](../specifications/v4.6.36-note-write-service-migration-plan.md) Phase N-2+N-3: thin write services for `note add` / `update` / `delete` and CLI handler wiring.

Spec: [v4.6.37-note-write-service-phase-n2-n3.md](../specifications/v4.6.37-note-write-service-phase-n2-n3.md)

## What changed

- **New services:** `note_add`, `note_update`, `note_delete`
- **Add/update result:** domain `Note` (same as read `note_show`; no enriched layer)
- **Delete result:** pre-delete `{ id, title }` snapshot
- **Handler:** thin service calls; success `println!` + `print_note_detail` (add/update)
- **Owner resolution:** moved into `note_add` service; handler no longer calls `resolve_note_owner_for_add`

## Compatibility

- **JSON golden:** unchanged
- **Human output:** unchanged (`note_cli` tests pass)
- **`import_export_notes`:** unchanged
