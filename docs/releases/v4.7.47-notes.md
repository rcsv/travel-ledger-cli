## What shipped (dry-run only)

- **`fragment apply --dry-run --trip <id>`** supports intent `update_estimate`
- **Itinerary target only** — reuses existing numeric selector resolver (ID → sort_order fallback → ambiguity reject). The numeric selector is a JSON number; string `"2"` is treated as a title.
- **Estimate identity:** `candidate_content.estimate_id` with parent Itinerary verification
- **Partial update:** `amount`, `currency`, `title`, `note`, `sort_order`, `clear_title`, `clear_note`
- **amount / currency:** amount-only allowed; currency-only rejected; numeric JSON amount = minor units (no double conversion)
- **null / clear:** explicit JSON `null` blocking; clear flags only for title/note; value + clear conflict blocking; empty string → NULL normalization
- **expected_* baseline** stale detection → blocking
- **no-op** (same proposed values) → valid success preview
- **DB read-only** — `update_estimate` service not called on dry-run path
- Preview report: `estimate_update_preview`, `estimate_field_changes`

## Not in scope
