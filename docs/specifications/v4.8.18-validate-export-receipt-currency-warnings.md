# v4.8.18 — validate-export Receipt currency warnings (implementation)

| Release | v4.8.18（released — 本書） |

v4.8.15 で導入した `trip validate-export` の currency quality warning（unknown / denylist）を、export JSON の `receipts[].currency` に横展開する implementation release です。

本 release は **validate-export の data quality signal** に限定し、既存 DB / import / read / export の挙動は変更しません。

---

## 0. Status（結論サマリ）

| 観点 | v4.8.18 判定 |
|---|---|
| 目的 | validate-export で Receipt currency の unknown / denylist を warning 化 |
| 適用面 | `trip validate-export` — `receipts[]`（schema ≥ v7） |
| 既存面 | expense / estimate warning（v4.8.15）は変更なし |
| DB schema | **変更なし** |
| `EXPORT_VALIDATION_REPORT_SCHEMA_VERSION` | **変更なし**（現状維持） |
| structured warning schema | **追加なし** — plain `warnings[]` string |

---

## 1. Warning / error 方針（Receipt）

対象 path:

- `receipts[{i}].currency`

| Condition | Severity |
|---|---|
| format invalid（空/英字3文字でない） | **error**（既存 Receipt export validation を維持） |
| alpha-3 format OK, registry unknown（例 `ZZZ`, `ABC`） | **warning** |
| alpha-3 format OK, denylist（例 `XXX`, `XTS`, `XAU`） | **warning** |
| valid ISO active / withdrawn（例 `JPY`, `HRK`） | pass（warning なし） |
| lowercase input（例 `jpy`） | normalize 後に valid なら pass |

### Warning messages

```text
{field_path}: currency code is not a known ISO 4217 code: {CODE}
{field_path}: currency code is not allowed for travel expenses: {CODE}
```

`field_path` 例: `receipts[0].currency`

---

## 2. Implementation

### Helper（`src/money/mod.rs`）

v4.8.15 の helper を再利用する。

```rust
pub(crate) fn validate_export_currency_quality_warning(
    code: &str,
    field_path: &str,
) -> Option<String>
```

- format invalid → `None`（既存 error path に委譲）
- denylist → travel-expense warning
- unknown ISO → registry warning
- valid ISO → `None`

### Collector（`src/trip.rs`）

`collect_export_currency_quality_warnings(export, effective_schema)` を拡張し、schema ≥ v7 の `receipts[]` を走査して `report.warnings` に追加する。
