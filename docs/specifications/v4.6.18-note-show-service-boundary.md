v4.6.12 `note list` に続き、**`note show` 単独**で service 層を導入する。

| Item | Status |
|---|---|
| Target | v4.6.18 — note show service boundary |
| Target command | **`note show` のみ** |
| Scope | service 層導入（structured result）、CLI 出力互換維持 |

---

## Status

| 観点 | 判定 |
|---|---|
| service layer | **`src/services/note_show.rs`** 追加 |

| terminal I/O | service 内に `println!` / `eprintln!` なし |

## Background

v4.6.17 `checklist show` の次段階として、itinerary detail から自然にぶら下がる `note show` を service 化する。

目標形:

```text
CLI / Tauri
  → service (use case, structured result, no I/O)
  → repository / domain
```

`note show` は ID 指定で1件の `Note` を取得する read-only command。write 系（add / update / delete）には触れず、既存挙動を保ったまま service boundary のみ導入する。

---

## Target command

- `note show <id>`
- `--json` フラグあり（`Note` をそのまま serialize）

---

## Current flow before split

```text
main.rs (NoteAction::Show)
  → note::get_note(conn, id)
  ├─ --json  → print_json(&note)
  └─ human   → print_note_detail(&note)
```

human 表示では `owner_type` / `owner_id` を `print_note_detail` が `Note` から出力する。Trip / Day / Itinerary の追加 context 表示はない。

---

## Service boundary introduced

新規:

```text
src/services/note_show.rs
```

API:

```rust
pub struct NoteShowServiceResult {
    pub note: Note,
}

pub fn show_note(conn: &Connection, id: i64) -> Result<NoteShowServiceResult>
```

service は `note::get_note` を呼び、**structured result のみ**返す。

human 表示（`print_note_detail`）は CLI 層に残す。

---

## CLI compatibility

- human: `print_note_detail(&result.note)` — owner_type / owner_id 含む詳細表示
- JSON: `result.note` をそのまま serialize

---

## JSON compatibility

- `--json` 時は `Note` を serialize（shape 不変、wrapper なし）

---

## Error compatibility

- Note 不存在: `Note not found: <id>`

---

## Test coverage

- service test:
  - 既存 note 取得
  - owner_type / owner_id 維持
  - not found message 維持
- 既存 `note_cli` / `not_found_cli` を回帰として維持

## What changed

- `src/services/note_show.rs` 追加
- `src/services/mod.rs` に `note_show` module 追加
- `main.rs` の `NoteAction::Show` が service 経由に

---

## What did not change

- `note list` / add / update / delete（対象外）
- day / reservation / expense / checklist show（対象外）
- write command / migration / FK（変更なし）

---

## Lessons learned

- `Note` は owner 情報を内包するため、show service は `trip show` / `checklist show` と同様に薄い
- `note list` の `NoteListJson` wrapper とは異なり、show は wrapper なし
