# v4.10.3 — Desktop settings navigation polish

| Release | v4.10.3（released — 本書） |
|---|---|
| Status | **released** |
| Prior | [v4.10.2 Desktop read-only usability polish](v4.10.2-desktop-read-only-usability-polish.md)（released） |

---

## 1. Goal

左サイドバー下部に **Settings** 入口を置き、Database / About を収められる小さな設定画面構造を追加する。プロフィール機能・ユーザー設定の保存は導入しない。read-only vertical slice の責務は広げない。

Cargo version は formal release で **`4.10.3`**。

---

## 2. Decisions

| Topic | Decision |
|---|---|
| Routing | 新規ルーター不要。`viewMode: "trips" \| "settings"` の React state |
| Icons | アイコンライブラリ未導入のため **テキストのみ**（独自 SVG を増やさない） |
| App version | `@tauri-apps/api/app` の `getVersion()`（既存依存）。Tauri 外テストは mock / fallback |
| Header actions | DB 選択済み時の Change / Forget は Settings → Database へ移動。ヘッダーは DB ファイル名のみ |
| DB 未選択 | `Open Database` をヘッダーに明示的に残す（Settings 奥へ隠さない） |
| Selection | Settings を開いても selected DB / Trip / Day を破棄しない。Trip list へ戻ると維持 |
| Profile | **今回は導入しない**（将来候補） |

---

## 3. UI

### Sidebar

- Trip list 領域は scroll
- Settings ボタンはサイドバー最下部に固定
- Settings 選択中は Settings ボタンを強調（Trip 選択強調と区別）

### Settings view

**Database**

- filename / full path（長 path は wrap / break）
- Change Database / Forget Database
- read-only であることの表示
- Forget はファイル非削除を明示

**About**

- Travel Ledger Desktop
- developer preview · read-only
- app version（`getVersion()`）

### Behavior

- Change cancel / failure → 現行 DB・画面状態維持
- Change success → 新 DB に切替（Trip 再ロード・先頭 Trip）。Settings を開いたままでも可
- Forget success → settings + runtime clear、未選択画面、`Open Database` 表示
- startup restore は v4.10.1/v4.10.2 のまま
