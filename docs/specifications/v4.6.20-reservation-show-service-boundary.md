## Background

`reservation show` は ID 指定で1件の `Reservation` を表示する read-only command。

- JSON では `Reservation` 単体を serialize する（wrapper なし）
- human 表示では itinerary 由来の文脈（day / title）を併記する（既存挙動）

Tauri / GUI からも同じ read-only use case を利用できるよう、service boundary を導入する。

目標形:

```text
CLI / Tauri
  → service (use case, structured result, no I/O)
  → repository / domain
```

---

## Target command

- `reservation show <id>`
- `--json` フラグあり（`Reservation` をそのまま serialize）

---

## Current flow before split

```text
main.rs (ReservationAction::Show)
  → reservation::get_reservation(conn, id)
  → reservation::print_reservation_detail(conn, &reservation)
  ├─ (print_reservation_detail 内で itinerary を参照して day/title を表示)
  ├─ --json  → print_json(&reservation)
  └─ human   → print_reservation_detail(...)
```

---

## Service boundary introduced

新規:

```text
src/services/reservation_show.rs
```

API:

```rust
pub struct ReservationShowServiceResult {
    pub reservation: Reservation,
    pub day_number: Option<i64>,
    pub itinerary_title: Option<String>,
}

pub fn show_reservation(conn: &Connection, id: i64) -> Result<ReservationShowServiceResult>
```

Service は `Reservation` 本体と、human 表示用の itinerary context（day / title）を structured result として返す。

---

## CLI compatibility

`main.rs` は service を呼び、既存の CLI 出力関数に委譲する。

- human: `print_reservation_detail_with_context(&result.reservation, result.day_number, result.itinerary_title.as_deref())`
- JSON: `result.reservation` をそのまま serialize

---

## JSON compatibility

- `--json` 時は `Reservation` を serialize（shape 不変、wrapper なし）

---

## Error compatibility

- Reservation 不存在: `Reservation not found: <id>`

---

## itinerary context compatibility

- itinerary 参照が失敗する場合でも、既存挙動どおり show 自体は失敗させない（context は `None`）

## What changed

- `src/services/reservation_show.rs` 追加

---

## What did not change

- `reservation list` / add / update / delete（対象外）
- `itinerary show` の Reservations 表示（対象外）
- expense / note / checklist（対象外）
- write command / migration / FK（変更なし）

---

## Lessons learned

- show 系でも「表示用文脈」がある場合は、service result に context を含めて CLI に委譲するのが最小
- JSON に wrapper がない command は `result.reservation` を直接 serialize
