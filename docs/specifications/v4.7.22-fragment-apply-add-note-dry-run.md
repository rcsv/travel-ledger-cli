## 1. 目的

```text
fragment validate
  → fragment apply --dry-run --trip <id>
  → target 解決
  → add_note preview
  → gate report / --output preview Trip JSON
  → DB は変更しない
```

Proposal Fragment が Note 追加を提案した場合に、既存 Trip / Day / Itinerary へどの Note が追加されるかを、DB 更新なしで確認できるようにします。

---

## 2. supported scope

```text
intent:
  add_note（dry-run のみ）

target:
  trip
  day
  itinerary

DB:
  read-only
  write なし
```

### 2.1 Note preview の保存先（export モデル）

| target | export `notes[]` エントリ |
|---|---|
| `trip` | `owner_type: trip` |
| `day` | `owner_type: day` + `day_number` |
| `itinerary` | `owner_type: itinerary` + `itinerary_key` |

### 2.2 Fragment フィールド

| ソース | export フィールド | 備考 |
|---|---|---|
| `candidate_content.body` | `body` | **優先** — 必須 |
| `fragment.notes` | `body` | `body` 未指定時のフォールバック |
| `candidate_content`（文字列） | `body` | フォールバック |
| `candidate_content.title` | `title` | optional |

`body` が空の場合は blocking error。

---

## 3. target 解決

| target_type | 要件 |
|---|---|
| `trip` | `--trip <id>` の Trip が存在 |
| `day` | `day_reference` 必須、Trip 内に存在 |
| `itinerary` | `day_reference` + `itinerary_reference`（sort_order または title）、一意に解決 |
| `unresolved` | **blocking** |
| ambiguous itinerary | **blocking** |

`--trip` は CLI 上の正本。`trip_reference` 不一致は **warning**。

---

## 4. gate report / output

```text
--json:
  gate report JSON（FragmentApplyDryRunReport）

--output:
  preview Trip JSON（schema v8）

stdout human:
  gate report / preview summary
```

`preview` サマリ（`add_note` 時）:

| フィールド | 意味 |
|---|---|
| `action` | `add_note` |
| `notes_before` / `notes_after` | preview 前後の Note 件数 |
| `itineraries_before` / `itineraries_after` | 変更なし（同一値） |

### required decisions

`required_decisions` が非空の場合:

- preview Trip JSON は生成される（`--output` 指定時）
- `valid: false` — valid confirm candidate ではない
- DB 不変

---

## 5. dry-run 原則

```text
fragment apply --dry-run:
  read-only DB access
  in-memory simulation
  Trip / Day / Itinerary / Note domain data は変更しない
```

`add_note --confirm` は **v4.7.22 では未実装**。

---

## 6. 確認方法
