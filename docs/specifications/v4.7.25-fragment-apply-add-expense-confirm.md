# v4.7.25 — Fragment apply add_expense --confirm (P-6g)

[v4.7.24](v4.7.24-fragment-apply-add-expense-dry-run.md) の `add_expense` dry-run に続き、**P-6g** として `fragment apply --confirm` による Expense INSERT を実装します。

| Item | Status |
|---|---|
| Release | v4.7.25 |
| Scope | `fragment apply <file> --confirm --trip <id>` — `add_expense`、Expense INSERT 1件 |
| Phase | **P-6g** |

---

## Status（結論サマリ）

| 観点 | v4.7.25 判定 |
|---|---|
| Command | **`fragment apply --confirm`** — `add_expense` 対応 |
| intent | **`add_expense` のみ**（confirm） |
| target | **itinerary のみ** |
| DB | **expenses INSERT 1件のみ** |
| dry-run | **v4.7.24 維持** — DB 非更新 |

---

## 1. 処理フロー

```text
fragment validate
  → target itinerary 解決
  → add_expense preview（v4.7.24 同型 gate）
  → confirm scope 検証
  → expenses INSERT 1件
  → expense list / expense show で確認
```

---

## 2. supported scope

```text
intent:   add_expense
target:   itinerary のみ
DB:       expenses INSERT 1件のみ
```

Trip / Day 直下への Expense 追加は **unsupported**。

### field mapping（v4.7.24 維持）

| ソース | DB / export フィールド |
|---|---|
| `candidate_content.amount` | `amount`（整数 minor units） |
| `candidate_content.currency` | `currency`（3文字コード） |
| `title` / `description` / `label` | `title` |
| `note` / `memo` / `fragment.notes` | `note` |

---

## 3. gate report

| フィールド | 意味 |
|---|---|
| `inserted_expense_id` | confirm 成功時の新規 Expense ID |
| `expense_preview` | dry-run 同型 preview サマリ |

---

## 4. 安全条件

以下を満たさない場合は DB 更新しません:

- Fragment validation valid
- Trip 存在
- target itinerary 一意解決（unresolved / ambiguous は blocking）
- intent `add_expense`
- amount / currency valid
- preview 生成 + trip validate-export 合格
- blocking errors なし
- required decisions 空
- `--confirm` 明示

---

## 5. Non-goals（v4.7.25）

- Expense update / delete
- 複数 expense 一括追加
- Trip / Day 直下 Expense
- add_reservation / update_itinerary / delete / reorder
- schema v8 変更 / DB migration
