## Background

v4.6.20 `reservation show` の次段階として、itinerary detail から自然にぶら下がる `expense show` を service 化する。

`expense show` は ID 指定で1件の `Expense` を取得する read-only command。

- JSON では `expense_to_json(conn, expense)` が `ExpenseJson` を組み立てる（beneficiaries / payer name 含む）
- human 表示では `print_expense_detail(conn, expense)` が同 helper 経由で詳細を出力する

`reservation show` と異なり、human / JSON 用の追加 context は service result ではなく既存 CLI helper が解決する。service は `Expense` 本体の取得のみ担当する（`note show` と同型）。

目標形:

```text
CLI / Tauri
  → service (use case, structured result, no I/O)
  → repository / domain
```

---

## Target command

- `expense show <id>`
- `--json` フラグあり（`ExpenseJson` via `expense_to_json`）

---

## Current flow before split

```text
main.rs (ExpenseAction::Show)
  → expense::get_expense(conn, id)
  ├─ --json  → expense_to_json(conn, &expense) → print_json
  └─ human   → print_expense_detail(conn, &expense)
```

---

## Service boundary introduced

新規:

```text
src/services/expense_show.rs
```

API:

```rust
pub struct ExpenseShowServiceResult {
    pub expense: Expense,
}

pub fn show_expense(conn: &Connection, id: i64) -> Result<ExpenseShowServiceResult>
```

Service は `get_expense(conn, id)` で `Expense` を取得し、structured result として返す。

---

## CLI compatibility

`main.rs` は service を呼び、既存の CLI 出力 helper に委譲する。

- human: `print_expense_detail(&conn, &result.expense)`
- JSON: `expense_to_json(&conn, &result.expense)` → `print_json`

---

## JSON compatibility

- `--json` 時は `ExpenseJson` を serialize（shape 不変、CLI 層 helper）

---

## Lessons learned

- `expense show` は `note show` と同様、service は `Expense` 本体のみで十分
- beneficiaries / payer name 等の JSON 拡張は `expense_to_json` が CLI 層で解決（既存パターン維持）
- `expense list` の `ExpenseListJson` wrapper とは異なり、show は helper 経由の `ExpenseJson`
