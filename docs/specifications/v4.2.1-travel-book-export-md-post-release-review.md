## 1. Summary

v4.2.0 は [v4.1.0 章立て](v4.1.0-travel-book-chapter-structure-design.md) を初めて沖縄 sample の実データで検証できる状態にした。**章順・entity 分離・会計データの除外**は設計意図どおり機能している。

一方、沖縄 sample を Travel Book として人に渡すと、次のギャップが目立つ。

1. **Trip overview の時間メトリクスがすべて `0m`** — 未入力と計算ゼロの区別がつかない
2. **fixture 由来の英語注記**が Reservations / Planned cost に露出 — しおり本文として不自然
3. **Notes の並び**が Day → Trip で、Trip 全体の補足が末尾に寄る
4. **Reservations** の Provider 重複表示など、細部の冗長さ

**結論:** v4.2.0 の章構造は **Keep**。実装バグや責務逸脱はない。改善は **小さな polish** に限定し、**v4.2.2** に分離するのが適切。v4.2.1 は本 Review と分類表の確定のみ（documentation-only）。

## 3. Findings by review point

### 3.1 Trip overview metrics（Stay / Travel / Total Time）

**観測（沖縄 sample）:**

```text
- Stay Time: 0m
- Travel Time: 0m
- Total Time: 0m
```

**原因:** 沖縄 seed は行動台帳中心で、Itinerary の `duration_minutes` / `travel_minutes` をほぼ設定していない。stats は正しく **合計 0** を返しているが、Travel Book 読者には「未集計」と区別できない。

| 選択肢 | 評価 |
|---|---|
| 常に表示（現状） | データがある Trip では有用。全ゼロ時はノイズ |
| ゼロ値を非表示 | Trip overview がすっきりする。部分ゼロ時のルール要定義 |
| `not recorded` 等の明示 | 未入力と計算結果 0 の区別に有効。文言・i18n が必要 |

**分類:** **Defer → v4.2.2**（ゼロ行の抑制、または全ゼロ時は時間ブロックごと省略）

**Keep:** Days / Itineraries / Checklist 行は沖縄 sample で意味があるため残す。

---

### 3.2 Fixture / internal wording

**観測:**

| 場所 | 例 |
|---|---|
| Planned cost `Note` 列 | `Travel Book planned-cost fixture; not original pre-trip budget` |
| Reservations `Remark` | `canonical sample fixture — derived from actual ledger` |

**性質:** [v4.1.1 enrichment plan](v4.1.1-okinawa-travel-book-sample-enrichment-plan.md) が **sample 正本**として seed に入れた注記。export-md は DB 内容をそのまま出している（generator のバグではない）。

| 観点 | 判定 |
|---|---|
| 開発者・検証用 | fixture 出所の説明として有用 |
| 旅行しおりとして共有 | 不適切 — 読者向けではない |
| 本番 Trip の一般ルール | ユーザーが書いた remark/note は **そのまま出す**のが正しい |

**分類:**

| 項目 | 分類 |
|---|---|
| export-md が remark/note を透過表示する | **Keep** |
| 沖縄 seed の fixture 英文をしおり向けに言い換え or 削除 | **Defer → v4.2.2**（sample seed / golden のみ。README に正本注記を残す） |
| export 時に fixture 文字列を自動除去 | **Do not do**（マジック文字列・本番データ誤削除リスク） |
| `debug` / `sample` output profile | **Defer**（v4.1.0 Non-goals。将来検討可） |

---

### 3.3 Notes placement and ordering

**観測（沖縄）:** `## Notes` は Day 1〜4 の Note が先、Trip Note が後。Itinerary スコープ Note は sample に無し。

**並びの実装:** `list_all_notes_for_trip` の `ORDER BY owner_type, owner_id` により `day` < `itinerary` < `trip`（辞書順）。

| 案 | 旅行しおりとしての読み味 |
|---|---|
| Trip → Day（番号順）→ Itinerary | 全体方針を先に読める。v4.1.0 の「Trip overview に summary」方針と整合 |
| Day → Trip（現状） | Daily schedule 直後に日別メモを追う流れには近いが、Trip 補足が末尾 |
| Day ごとに inline | 日別旅程と一体だが、Notes 章の責務と重複 |
| Itinerary のみ inline | 細かい行動メモ向き。長文 Note entity には不向き |

**分類:**

| 項目 | 分類 |
|---|---|
| Note entity を `## Notes` に集約（Daily schedule 外） | **Keep**（v4.1.0 / v4.2.0 決定） |
| Itinerary の単行 `note` フィールドを Daily schedule 内に表示 | **Keep** |
| export 並びを Trip → Day → Itinerary に変更 | **Defer → v4.2.2** |
| 全 Note を Daily schedule に inline | **Do not do**（責務重複・長文で可読性低下） |

### 3.4 Reservations display

**観測:** 種別見出し（Parking / Flight / Hotel…）でグループ。Daily schedule **の後** — v4.1.0 章順どおり。

**良い点:**

- confirmation は台帳根拠がある P1 G のみ（`EXAMPLE-PARKING-001`）— plan どおり
- Hotel に Period 表示 — チェックイン〜チェックアウトが一目で分かる
- 予約詳細を Daily schedule から分離 — 58 件の旅程がさらに膨らまない

**改善余地:**

| 項目 | 内容 |
|---|---|
| `**Day 1 / …** — {provider}` と直後の `Provider:` 行 | 同一情報の二重表示 |
| 全件に fixture `Remark` | §3.2 と同根（sample データ） |
| 便名が provider 兼 title | 実データ由来。現状 **Keep** |

**分類:**

| 項目 | 分類 |
|---|---|
| Reservations 章の位置・フィールドセット | **Keep** |
| Provider 行の冗長抑制 | **Defer → v4.2.2** |
| 日別章内の予約ハイライト 1 行 | **Defer → v4.3.0**（v4.1.0 任意項目） |
| confirmation の public profile マスク | **Defer**（v4.1.0 §3.5 将来） |

---

### 3.5 Planned cost display

**観測:**

```text
- Estimates: 8
- Planned total:
  - JPY 679,000
### Day N / {time} {title}
| Item | Amount | Note |
```

**良い点:**

- 章頭サマリー → Itinerary 別表の順はスキャンしやすい
- 8 件規模では Day / 時刻 / タイトル付きグループが追跡可能
- Estimate を Daily schedule から分離 — 「予定費用」と「行動」が混ざらない

**注意:**

- ¥679,000 は **fixture 合計**であり Actual ¥561,780 とは一致しない（意図どおり。Estimate ≠ Expense）
- 各表 `Note` 列の fixture 英文は §3.2 と同根

**分類:**

| 項目 | 分類 |
|---|---|
| 独立 `## Planned cost` 章 | **Keep** |
| 章頭 Planned total → 明細表 | **Keep** |
| Itinerary 単位グループ | **Keep**（現行件数帯） |
| Day 単位小計 | **Defer**（件数増加時に v4.3.0 で検討） |
| 明細を先・合計を後 | **Do not do**（現行の方がしおり向き） |

---

### 3.6 Chapter responsibility（Expense / Receipt / Difference）

**観測（沖縄 golden）:** `Expenses:` / `Difference:` / Receipt 言及 **なし**。`trip stats` テストで Actual ¥561,780・Expense 49 は維持。

| データ | Travel Book（export-md） | 会計ビュー |
|---|---|---|
| Estimate | Planned cost 章 | `trip stats` Planned |
| Expense | **含めない** | `trip stats` Actual |
| Difference | **含めない** | `trip stats` / `--json` |
| Receipt | **含めない** | `receipt list` / export JSON |

**分類:** 境界は **Keep**。Travel Book と台帳レビューを混ぜない方針を維持。

**Do not do:** v4.2.x で Expense / Receipt を Travel Book に載せること（別プロダクト線または ledger-oriented export が必要）。
