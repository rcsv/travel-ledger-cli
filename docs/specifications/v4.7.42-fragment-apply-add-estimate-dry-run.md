## 1. Semantic boundary

```text
Estimate = Planned Money
Expense  = Actual Money
```

- `add_estimate` は **Estimate preview のみ** — Expense 行は作成・プレビューしない
- `add_expense` は変更しない

---

## 2. supported scope

```text
intent:
  add_estimate（dry-run のみ）

target:
  itinerary のみ（trip / day は unsupported）

DB:
  read-only
  write なし
```

### 2.1 Estimate preview の保存先（export モデル）

| target | export 配置 |
|---|---|
| `itinerary` | `days[].itineraries[].estimates[]` に 1 件 append |

### 2.2 Fragment フィールド

| ソース | 意味 | 備考 |
|---|---|---|
| `candidate_content.amount` | 計画金額 | **必須** — string（CLI 小数）または integer（minor units） |
| `candidate_content.currency` | 通貨 | **必須** — 3 文字コード |
| `title` / `description` / `label` | `title` | optional |
| `note` / `memo` / `fragment.notes` | `note` | optional — Estimate note のみ |
| `sort_order` | 並び順 | optional integer、省略時 `0` |

`amount` / `currency` が欠落または不正の場合は blocking error。

---

## 3. target 解決

| target_type | 要件 |
|---|---|
| `itinerary` | `day_reference` + `itinerary_reference`（ID / sort_order / title）、一意に解決 |
| `trip` / `day` | **unsupported** — blocking |
| `unresolved` | **blocking** |
| ambiguous itinerary | **blocking** |

`--trip` は CLI 上の正本。target Itinerary は選択 Trip に属すること。

---

## 4. gate report / preview

`FragmentApplyPreviewSummary` 追加フィールド:

| フィールド | 意味 |
|---|---|
| `action` | `add_estimate` |
| `estimates_before` / `estimates_after` | Trip 全体の Estimate 件数 |
| `estimate_preview` | 追加予定 Estimate サマリ |

`estimate_preview` DTO:

| フィールド | 意味 |
|---|---|
| `target_itinerary_id` | 解決済み Itinerary DB ID |
| `target_itinerary_title` | 人間可読タイトル |
| `amount` | minor units（整数） |
| `currency` | 正規化後コード |
| `title` / `note` | optional |
| `sort_order` | 解決後値 |

`inserted_estimate_id` は **v4.7.43 confirm** まで未実装。

### invalid report

- blocking errors → `valid=false`、DB 不変、preview ファイル未出力（既存慣行）
- `required_decisions` 非空 → `valid=false`、DB 不変

---

## 5. dry-run 原則

```text
fragment apply --dry-run:
  read-only DB access
  in-memory simulation
  estimates / expenses / reservations / notes / itinerary_items 等は変更しない
```

`estimate::add_estimate` は **呼び出さない**。

---

## 6. 確認方法

```bash
caglla fragment apply fragment.json --dry-run --trip 1 --output preview.json
caglla trip validate-export preview.json
caglla fragment apply fragment.json --dry-run --trip 1 --json
caglla estimate list --itinerary <id> --json   # dry-run 前後で件数不変
```
