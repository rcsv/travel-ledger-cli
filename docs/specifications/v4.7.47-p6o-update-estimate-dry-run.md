## 1. Semantic boundary

```text
Estimate = Planned Money
Expense  = Actual Money
```

- `update_estimate` は **既存 Estimate 1 件の preview のみ**
- Expense 行は作成・更新しない
- Estimate 新規作成・削除・bulk / upsert は対象外

---

## 2. supported scope

```text
intent:
  update_estimate（dry-run のみ）

target:
  itinerary のみ（trip / day は unsupported）

DB:
  read-only
  write なし
```

### 2.1 更新可能 field（部分更新）

| field | 備考 |
|---|---|
| `amount` | string（`parse_amount_for_currency`）または integer（minor units） |
| `currency` | **amount と同時指定のみ** |
| `title` | string；`normalize_optional_text`；`""` → NULL |
| `note` | string；`normalize_optional_text`；`""` → NULL |
| `sort_order` | integer |
| `clear_title` | `true` → proposed title は NULL |
| `clear_note` | `true` → proposed note は NULL |

少なくとも 1 件の有効な更新 field が必要。unknown field は **warning**（既存 Fragment 方針）。

未対応 clear: `clear_amount` / `clear_currency` / `clear_sort_order`

### 2.2 Estimate identity

| ソース | 意味 |
|---|---|
| `candidate_content.estimate_id` | **必須** integer — DB primary key |

title / sort_order / amount による Estimate selector は使用しない。解決済み Itinerary の子であることを必須確認。

---

## 3. target 解決

既存 Proposal Fragment itinerary resolver を再利用。

numeric selector 契約:

```text
1. Itinerary ID として解決
2. ID で解決しない場合のみ sort_order fallback
3. ambiguity は reject
```

`itinerary_reference` の数値 selector は JSON **number** 型（string は title 照合）。

---

## 4. gate report / preview

`FragmentApplyPreviewSummary` 追加フィールド:

| フィールド | 意味 |
|---|---|
| `action` | `update_estimate` |
| `estimate_update_preview` | 解決済み target / estimate identity |
| `estimate_field_changes` | `{ field, before, after }[]`（`update_itinerary` 同型） |

`estimate_update_preview` DTO:

| フィールド | 意味 |
|---|---|
| `target_itinerary_id` | 解決済み Itinerary DB ID（identity 正本） |
| `target_itinerary_title` | 表示用 |
| `target_estimate_id` | 更新対象 Estimate DB ID（identity 正本） |

nullable field の NULL は `before` / `after` で `"-"` 表現（`update_itinerary` 整合）。

`updated_estimate_id` は report 型に存在するが **v4.7.48 confirm** まで未設定。これは今回の dry-run 契約上の欠陥ではなく、明示的な scope 境界である。

### invalid report

- blocking errors → `valid=false`、DB 不変
- `required_decisions` 非空 → `valid=false`、DB 不変
- stale `expected_*` baseline mismatch → blocking

---

## 5. dry-run 原則

```text
fragment apply --dry-run:
  read-only DB access
  in-memory export patch のみ
  estimates / expenses / itinerary_items 等は変更しない
```

`estimate::update_estimate` は **呼び出さない**。

推奨内部責務分離（confirm 再利用向け）:

```text
parse_update_estimate_fields
resolve target
load current estimate
validate parent relation
compute proposed estimate
compute field changes
build dry-run report
```
