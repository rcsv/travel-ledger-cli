# v4.7.14 — Public examples guard CI isolation hotfix

[v4.7.12](v4.7.12-public-examples-validation-guard.md) で追加した public examples validation guard が、Rust CI 上で `duplicate column name: day_id` により flake していた問題の hotfix です。

| Item | Status |
|---|---|
| Release | v4.7.14 |
| Scope | guard CI 安定化 — test isolation + file-only command dispatch |
| 種別 | **hotfix**（機能拡張ではない） |

---

## Status（結論サマリ）

| 観点 | v4.7.14 判定 |
|---|---|
| 原因 | `trip validate-export` が DB open 前に dispatch されず、並行テストが同一 `./caglla.db` で migration 競合 |
| 対応 1 | `trip validate-export` / `trip diff` を **DB open 前** dispatch（file-only） |
| 対応 2 | guard テストに **isolated temp workdir**（`validate_export_cli.rs` と同型） |
| DB migration | **なし** |
| validate 仕様 | **変更なし** |

---

## 1. 背景

v4.7.13 release 後、Rust CI で以下が断続的に失敗:

```text
guard_schema_v8_public_examples_pass_trip_validate_export
Error: itinerary_items.day_id 列の追加に失敗しました
Caused by: duplicate column name: day_id
```

v4.7.13 は documentation-only。guard ロジック・public examples 自体は変更なし。`cargo test` 並行実行時に複数 CLI プロセスが **同一 CWD の default DB** を開き migration したことが原因。

---

## 2. 根本原因

```text
main.rs:
  proposal validate / fragment validate → DB open 前 dispatch（v4.7.9+）
  trip validate-export              → DB open 後 dispatch（v4.7.14 まで）

trip validate-export の実装は JSON file read のみだが、
CLI 入口で open_db_at() が先に走っていた。
```

並行 integration test（`db reset` 等）と guard が同じ `./caglla.db` を共有 → migration race。

---

## 3. 対応

### 3.1 `src/main.rs`（最小）

`TripAction::ValidateExport` と `TripAction::Diff` を Proposal / Fragment と同様、**DB open 前**に dispatch。

```text
責務整合:
  trip validate-export — file validation のみ、DB 不要
  trip diff            — 2 file 比較のみ、DB 不要
```

### 3.2 `tests/public_examples_validation_guard.rs`

各テストで一意な temp workdir を `current_dir` に設定。

```text
validate_export_cli.rs と同型の isolation
example JSON は manifest からの絶対 path のまま
```
