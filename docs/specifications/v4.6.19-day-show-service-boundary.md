## Background

v4.6.18 `note show` の次段階として、day detail 入口の `day show` を service 化する。

目標形:

```text
CLI / Tauri
  → service (use case, structured result, no I/O)
  → repository / domain
```

`day show` は Trip + Day 番号で1日分の詳細と配下 Itinerary 一覧を表示する read-only command。write 系（update / swap）には触れず、既存挙動を保ったまま service boundary のみ導入する。

---

## Target command

- `day show <trip_id> <day_number>`
- `--json` フラグあり（`DayShowJson` wrapper）

---

## Current flow before split

```text
main.rs (DayAction::Show)
  → services::day_show::show_day(conn, trip_id, day_number)
  ├─ --json  → print_day_show_json (DayShowJson)
  └─ human   → print_day_show_display
```

human 表示では Trip 名、Day 番号、日付、summary、Itinerary 一覧を表示。Note / Expense / Reservation は含まない。

---

## Service boundary introduced

新規:

```text
src/services/day_show.rs
```

API:

```rust
pub struct DayShowServiceResult {
    pub trip: Trip,
    pub day: Day,
    pub date: String,
    pub itineraries: Vec<ItineraryItem>,
}

pub fn show_day(conn: &Connection, trip_id: i64, day_number: i64) -> Result<DayShowServiceResult>
```

service は trip / day 解決、日付計算、itinerary 一覧取得までを orchestrate し、**structured result のみ**返す。

`DayShowJson` wrapper と human 表示は CLI 層（`day.rs`）に残す。

---

## CLI compatibility

- human: `print_day_show_display(&result.trip, &result.day, day_number, &result.date, &result.itineraries)`
- JSON: `print_day_show_json(...)` で `DayShowJson` を組み立て

---

## JSON compatibility

- `--json` 時は `DayShowJson` を serialize（shape 不変、CLI 層 wrapper）

---

## Error compatibility

- Trip 不存在: `Trip not found: <id>`
- Day 不存在: `Day not found: trip <trip_id> day <day_number>`

## What changed

- `src/services/day_show.rs` 追加
- `src/services/mod.rs` に `day_show` module 追加
- `main.rs` の `DayAction::Show` が service 経由に
- `day.rs` に `print_day_show_json` / `print_day_show_display` を抽出

---

## What did not change

- `day list` / update / swap（対象外）
- itinerary / note / trip service（対象外）
- write command / migration / FK（変更なし）

---

## Lessons learned

- `day show` は trip + day + date + itineraries が use case の最小単位
- `DayShowJson` wrapper は `day list` の `DayListJson` と同様 CLI 層に残す
- `day update` 後の human 表示は write 系のため `get_day` 直結のまま（対象外）
