## 4. Safety / transaction behavior

### 4.1 `delete_estimate` confirm — structured errors と write safety

| 観点 | v4.8.10 判定 |
|---|---|
| transaction 使用 | `with_transaction` 維持 — 変更なし |
| structured error push | `fail_confirm_transaction` → `Err` → **rollback** |
| partial write | scoped DELETE exactly-one contract 維持 |
| legacy `errors[]` | confirm 失敗時 mirror — CLI human output 不変 |
| destructive safety | hard delete + TOCTOU / preview / baseline guards 維持 |

### 4.2 Error 分類妥当性

| 検出 | Code | 妥当性 |
|---|---|---|
| gate vs recomputed drift | `APPLY_PREVIEW_MISMATCH` | ✓ — inter-gate confirm drift の第一防御 |
| revalidate field drift | `APPLY_TOCTOU_DRIFT` | ✓ — gate snapshot vs 現行 DB |
| `expected_*` vs 現行値 | `APPLY_BASELINE_MISMATCH` | ✓ — adoption baseline contract |
| DELETE affected = 0 | `APPLY_SCOPED_WRITE_ZERO_ROWS` | ✓ — exactly-one 前段 |
| DELETE affected > 1 | `APPLY_SCOPED_WRITE_MULTIPLE_ROWS` | ✓ — DB 制約上 rare |

**P0/P1:** write safety / rollback への regression なし。
