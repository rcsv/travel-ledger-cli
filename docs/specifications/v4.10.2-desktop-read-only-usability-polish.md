# v4.10.2 — Desktop read-only usability polish

| Release | v4.10.2（released — 本書） |
|---|---|
| Status | **released** |
| Prior | [v4.10.1 Desktop database reopening and usability polish](v4.10.1-desktop-database-reopening-and-usability-polish.md)（released） |

---

## 1. Goal

write path に進む前に、read-only Desktop（Trip list → Trip detail → Day timeline）の見やすさ・状態遷移・文言を磨く。**機能追加より polish。** Desktop は引き続き developer preview / source-only（`bundle.active = false`）。

Cargo version は formal release で **`4.10.2`**。

---

## 2. Decisions

| Topic | Decision |
|---|---|
| Last selected Trip persistence | **しない**。open / restore 後は従来どおり先頭 Trip（と先頭 Day）を自動選択。設定永続化は DB path のみ（v4.10.1） |
| Missing metadata | **非表示**（“Not set” は出さない）。ある項目だけ出す |
| Itinerary presentation | **travel activity**（sequence-first）。task / checklist 風にしない。完了状態は導入しない |
| Day tabs | Day number + weekday + date（短縮）で見当を付けやすくする |
| Forget / Open copy | Forget は設定だけ忘れること、DB ファイルを削除しないことを明示。過度に怖い文言は緩和 |

---

## 3. UI polish scope

### Trip list

- Trip 件数（例: `Trips (3)`）
- 選択状態の視認性を強化
- date range / main destination / default currency（あるときだけ）
- 0 件 empty state をやさしい文言に

### Trip detail

- summary / destination / country / currency / date range / days count
- 空メタは出さない
- Days が 0 件の empty state

### Day timeline

- 日付・曜日を day 見出しに
- itinerary 0 件 empty state
- `sort_order` を主、`start_time` は任意ラベル
- duration / travel_time / category / location / note は控えめなメタ行
- Itinerary completion UI は導入しない

### States / copy

- Starting / restore warning / DB 未選択 / Trip 0 / Day 空 / timeline 空 / load error
- Change cancel / Change fail で現行選択維持（v4.10.1 契約維持）
- Forget 後は未選択へ

---

## 4. Out of scope

- Trip / Day / Itinerary write、write facade
- DB create / delete UI、recent DB list
- Checklist UI、Itinerary completion、schema migration
- export / import、workspace 化、desktop bundle、formal release

Itinerary completion 正本は引き続き Checklist（`checklist_items.is_done`）。

---

## 5. Tests

Frontend:

- restore / selected / empty / error copy の確認を拡張
- Day weekday / trip count / empty itinerary wording
- Change fail 維持・Forget 後未選択（既存を維持）

Backend: v4.10.1 の restore/select/config テストをそのまま利用。追加は最小。
