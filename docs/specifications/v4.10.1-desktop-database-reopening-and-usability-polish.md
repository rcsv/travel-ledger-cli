## 1. Goal

最後に正常選択された Travel Ledger SQLite DB を、起動時に安全に復元し、Change / Forget 操作で日常利用しやすくする。**write UI は導入しない。** Desktop は引き続き developer preview / source-only（`bundle.active = false`）。

---

## 2. Pre-investigation（v4.10.0 現状）

| Topic | Finding |
|---|---|
| `DesktopState` | `desktop/src-tauri/src/state.rs`。`Mutex<Option<PathBuf>>` のみ。`lib.rs` で `.manage(DesktopState::default())` |
| `select_database` | `probe_database`（存在・file・`open_db`・`list_trip_summaries`）成功後のみ state 更新。Mutex は path 書き込み時のみ短い保持 |
| Frontend 初期状態 | `databasePath = null` → empty state。DB 選択後 `loadTrips` → 先頭 Trip / Day 自動選択 |
| Tauri 2 config dir | Rust: `app.path().app_config_dir()` → `{config_dir}/{bundle_identifier}`（`com.rcsv.traveledger.desktop`） |
| Test 境界 | `service.rs` helpers は Tauri runtime 非依存。一時 DB + CLI seed |
| Path の frontend 露出 | `DatabaseInfo.path` は `Path::display()` 文字列。フル path を返す |

安全性維持要件（変更しない）:

- path 存在 / regular file
- `open_db` + facade probe 成功
- validation 前に state を更新しない
- 存在しない path から新規 DB 作成しない
- DB アクセス中に Mutex を保持しない

## 3. Persistence choice

**Backend-owned JSON file**（`tauri-plugin-store` は不採用）。

| Option | Verdict |
|---|---|
| `tauri-plugin-store` | 依存追加・能力設定増加。temp dir 注入が重く、frontend からの書込面も広がりやすい |
| **small JSON in app config dir** | `serde_json` 既存。atomic rename。config path を `DesktopState` に注入でき unit test 容易 |

保存先:

```text
{app_config_dir}/desktop-settings.json
```

形式:

```json
{
  "last_database_path": "/absolute/or/native/path"
}
```

- CLI の `db use` / `caglla.toml` とは **別の Desktop ローカル設定**
- frontend `localStorage` は正本にしない
- frontend は設定ファイル path を指定できない
- path は dialog / OS が返す文字列をそのまま保存（UTF-8 妥当な path 前提；非 UTF-8 は `to_string_lossy` で保存する場合あり — dialog API は String）
- 書き込みは temp → rename（部分書き込み回避）
- config directory 不在時は `create_dir_all`

## 4. Commands

| Command | Role |
|---|---|
| `select_database(path)` | 検証 → state → **永続化** → `DatabaseInfo` |
| `restore_last_database()` | 保存設定を読み検証。成功時だけ state 更新。Tagged restore result |
| `forget_database()` | state clear + 設定削除。**DB ファイルは削除しない** |
| `list_trip_summaries` / `get_trip_detail` / `get_day_timeline` | 変更なし（facade） |

### `RestoreLastDatabaseResult`（serde tagged）

```text
{ "status": "restored", "database": { "path", "trip_count" } }
{ "status": "not_found" }
{ "status": "invalid_cleared", "code": "...", "message": "..." }
```

- `not_found` — 設定なし / 空
- `invalid_cleared` — 存在しない・directory・open 失敗・壊れた JSON など。**設定を clear**し、アプリ起動は継続
- 予期しない config **write** 失敗のみ command `Err(DesktopError)`（`DATABASE_CONFIG_WRITE_FAILED`）

### Error codes（追加は必要最小）

| Code | When |
|---|---|
| existing | `DATABASE_*`, `TRIP_NOT_FOUND`, … |
| `DATABASE_CONFIG_READ_FAILED` | 設定読取の I/O 失敗（壊れた JSON は invalid_cleared 側） |
| `DATABASE_CONFIG_WRITE_FAILED` | 設定の書込 / Forget 時削除失敗 |

`DATABASE_RESTORE_FAILED` は `invalid_cleared` の具体 code（`DATABASE_PATH_INVALID` / `DATABASE_OPEN_FAILED` 等）で十分のため **追加しない**。

## 5. Change / Forget UI

### Change Database

- dialog 再オープン
- キャンセル → 現状維持
- validation 失敗 → **現状の DB / UI 維持** + error banner
- 成功 → state・設定・Trip 一覧切替、旧 Trip/Day/timeline clear

### Forget Database

- 確認ダイアログ（DB ファイルは削除しない旨を明示）
- `forget_database` → frontend clear → 未選択画面

## 6. Startup states

```text
Starting (restoring)
  ├─ restored        → Trip list load
  ├─ not_found       → DB not selected
  └─ invalid_cleared → warning banner + DB not selected
```

Header: DB file name + Change + Forget（フル path は `title` tooltip）。
