## Purpose

```text
caglla db use <path> で CWD の ./caglla.toml に [database].path を保存し、
以降のコマンド（override なし）が同じ DB を参照できるようにする。
```

v3.9.0 read resolution の Phase 2（write 側）。

---

## Resolution priority（変更なし）

```text
1. --db <path>
2. CAGLLA_DB
3. ./caglla.toml [database].path
4. ./caglla.db
```

`db use` は **3 番を更新**する。当該 invocation の即時 override ではない。

---

## Implemented

### CLI

```bash
caglla db use <path>
caglla db use ./data/okinawa.db
caglla db use --clear
```

- `DbAction::Use` — open 前 dispatch（`run_before_open_db`）
- `<path>` と `--clear` は相互排他
- DB open / 作成なし

### Config write (`src/config.rs`)

- `run_db_use` / `set_db_use_config` / `clear_db_use_config`
- CWD `./caglla.toml` のみ
- `toml::Value` merge — 未知 top-level / `[database]` 下キー保持
- atomic write（temp + rename）
- 相対パス優先保存（`format_path_for_config_storage`）

### Read adjustment

- `[database].path` が無いが他セクションがある `caglla.toml` → default `./caglla.db`
- empty `[database].path` → 従来どおりエラー

### Tests

- `src/config.rs` unit tests（書き込み・clear・相対化・invalid TOML 非破壊）
- `tests/db_cli.rs` integration（use → path、clear、override、未存在 DB note）
