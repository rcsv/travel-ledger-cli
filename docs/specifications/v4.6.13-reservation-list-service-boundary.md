## Background

目標形:

```text
CLI / Tauri
  → service (use case, structured result, no I/O)
  → repository / domain
```

`reservation list` は Trip / Itinerary 単位で予約一覧を表示する read-only 導線。`itinerary show` の human 専用 Reservations セクションとは別 handler。v4.6.13 では既存挙動を保ったまま service boundary のみ導入する。

---

## Target command

- `reservation list --trip <id>`
- `reservation list --itinerary <id>`
- `--json` フラグあり（`ReservationListJson`）

`--trip` と `--itinerary` は同時指定不可。

---

## Current flow before split

```text
main.rs (ReservationAction::List)
  → reservation::resolve_reservation_list_target(trip, itinerary)
  ├─ Trip      → list_reservations_for_trip → context_rows + reservations
  └─ Itinerary → list_reservations_for_itinerary → reservations
  ├─ --json  → print_json(&ReservationListJson { trip_id, itinerary_id, reservations })
  └─ human   → print_reservation_list(target, &reservations, trip_context)
```

Trip 一覧は context 付き human 表示、Itinerary 一覧はテーブル形式。

---

## Service boundary introduced

新規:

```text
src/services/reservation_list.rs
```

API:

```rust
pub struct ReservationListServiceResult {
    pub target: ReservationListTarget,
    pub reservations: Vec<Reservation>,
    pub trip_context: Option<Vec<ReservationWithContext>>,
}

pub fn list_reservations(
    conn: &Connection,
    trip: Option<i64>,
    itinerary: Option<i64>,
) -> Result<ReservationListServiceResult>
```

- Trip target: `trip_context` に Day / Itinerary 文脈を含む（human 表示用）
- Itinerary target: `trip_context` は `None`
- `ReservationListJson` wrapper は CLI 層に残す

---

## CLI compatibility

- human: `print_reservation_list(result.target, &result.reservations, result.trip_context.as_deref())`
- JSON: `ReservationListJson` を CLI 層で組み立てて serialize

---

## JSON compatibility

- `--json` 時は `ReservationListJson`（`trip_id`, `itinerary_id`, `reservations`）を serialize（shape 不変）

---

## Error compatibility

- target 未指定: `--trip または --itinerary のいずれかを指定してください`
- 同時指定: `--trip と --itinerary は同時に指定できません`
- Trip 不存在: `Trip not found: <id>`
- Itinerary 不存在: `Itinerary not found: <id>`

---

## Ordering compatibility

| target | ordering |
|---|---|
| Trip | `ORDER BY i.day ASC, i.sort_order ASC, i.id ASC, r.id ASC` |
| Itinerary | `ORDER BY id ASC` |

service は順序を変えない。

## What did not change

- `itinerary show` の Reservations 表示（対象外）
- `expense list` / `checklist list`（対象外）

---

## Lessons learned

- Trip / Itinerary で human 表示が異なる → `trip_context` を result に含める
- JSON は reservations のみで足りるが、human は Trip 時に context が必要
- `itinerary show` の Reservations は別 handler のまま維持（混同しない）
