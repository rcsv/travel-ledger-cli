## 1. Scope（本 release）

### 1.1 `APPLY_REQUIRED_DECISION` — `kind` 整合

| 項目 | v4.8.7 | v4.8.8 |
|---|---|---|
| `code` | `APPLY_REQUIRED_DECISION` | 変更なし |
| `kind` | `blocking`（provisional） | **`decision_required`** |
| `phase` | `gate` / `confirm_scope` | 変更なし |
| `required_decisions[]` | 維持 | 維持 |
| legacy `errors[]` | mirror 維持 | mirror 維持 |

**client 分岐:** `kind=decision_required` は planning 意図どおりのヒントだが、**分岐の正本は引き続き `code`** とする。`message` は表示用参考のみ。

### 1.2 `add_estimate` unsupported target wiring

| 項目 | 値 |
|---|---|
| `code` | `APPLY_UNSUPPORTED_TARGET` |
| `phase` | `simulate` |
| optional | `intent=add_estimate`, `target_type=<resolved>` |
| legacy `errors[]` | 従来文言を mirror（互換維持） |

v4.8.7 review で unwired とされた simulate path。`add_expense` と同型の `fail_simulate` wiring。

### 1.3 confirm transaction — delete_estimate 代表 path（2 件）

| Path | `code` | `phase` | 備考 |
|---|---|---|---|
| gate preview vs recomputed mismatch | `APPLY_PREVIEW_MISMATCH` | `confirm_transaction` | gate と confirm 間の DB drift は verify 段階で検出 |
| revalidate `TOCTOU mismatch:` 系 | `APPLY_TOCTOU_DRIFT` | `confirm_transaction` | revalidate 内 defensive path |

**DB write safety / rollback:** 変更なし。transaction 内 early return + rollback 既存挙動を維持。

**deferred（v4.8.8 非 scope）:**

- confirm transaction 全面 wiring（update_estimate / delete_itinerary / move 等）
- `APPLY_SCOPED_WRITE_ZERO_ROWS` / `APPLY_SCOPED_WRITE_MULTIPLE_ROWS`
- confirm baseline mismatch の structured 化（revalidate 内 `baseline mismatch:` は `APPLY_TRANSACTION_FAILED` fallback または `errors[]` のみ）

## 3. JSON contract（維持）

### 3.1 `schema_version: 2` 維持理由

- 新規 field 追加なし（既存 `structured_errors[]` への要素追加のみ）
- legacy `errors[]` 維持
- unknown `code` / `kind` / `phase` forward-compatible

### 3.2 structured error 要素

**Required:** `code`, `kind`, `message`, `phase`

**Optional:** `field_path`, `intent`, `target_type`（存在時のみ出力）

**Success report:** 空 `structured_errors` キー省略（v4.8.6 維持）

### 3.3 Breaking / non-breaking

| 変更 | 分類 |
|---|---|
| 新 path で `structured_errors[]` が出る | **non-breaking**（additive） |
| `APPLY_REQUIRED_DECISION` の `kind` → `decision_required` | **non-breaking**（同一 `code`、kind enum 値は既に contract 上定義済み） |
| legacy `errors[]` 文言 | **変更なし** |
| `required_decisions[]` | **変更なし** |
