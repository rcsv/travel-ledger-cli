# Day list service boundary

v4.6.4〜v4.6.6 の read-only service boundary に続き、**`day list` 単独**で service 層を導入する。

| Item | Status |
|---|---|
| Target | v4.6.7 — day list service boundary |
| Target command | **`day list` のみ** |
| Scope | service 層導入（structured result）、CLI 出力互換維持 |

---

## Status

| 観点 | 判定 |
|---|---|
| service layer | **`src/services/day_list.rs`** 追加 |
| target command | `day list` が service 経由 |
| terminal I/O | service 内に `println!` / `eprintln!` なし |
| CLI / JSON 出力 | **不変** |
| Trip not found | **不変**（`Trip not found: <id>`） |
| DB schema / migration | **変更なし** |

---

## Background

目標形:

```text
CLI command
  → service/usecase
  → repository
  → SQLite

Tauri command
  → same service/usecase
  → repository
  → SQLite
```

`day list` は GUI の Trip detail → Day overview 導線に直結する。v4.6.7 では `day show` / `itinerary list` 等には触れない。

---

## Target command

- `day list <trip_id>`
- `day list <trip_id> --json`

---

## Current flow before split

```text
main.rs (DayAction::List)
  → day::run_day_list(conn, trip_id, json)
       ├─ trip::get_trip + day::list_days
       ├─ --json  → DayListJson を組み立てて print_json
       └─ human   → print_day_list（println）
```

DB 取得と terminal 出力が `run_day_list` に混在していた。

---

## Service boundary introduced

新規:

```text
src/services/day_list.rs
```

API:

```rust
pub struct DayListServiceResult {
    pub trip: Trip,
    pub days: Vec<Day>,
}

pub fn list_days(conn: &Connection, trip_id: i64) -> Result<DayListServiceResult>
```

service は `trip::get_trip` と `day::list_days` を orchestrate し、**structured result のみ**返す。

表示分離（`day.rs`）:

```text
print_day_list_json(trip, trip_id, days)   — JSON 出力のみ
print_day_list_display(trip, days)         — human 出力のみ
```

---

## CLI compatibility

- human 出力は従来どおり `Trip:` 見出し + `Day N  YYYY-MM-DD` 行
- Day 並び順は `ORDER BY day_number`（`day::list_days`）

---

## JSON compatibility

- `day list --json` は引き続き `DayListJson` を serialize（`trip_id` / `trip_name` / `days[]`）

---

## Error compatibility

- Trip 不存在: `Trip not found: <id>`（`trip::get_trip` 経由、service test で確認）

---

## Test coverage

- service test:
  - 4日 Trip → 4 days 返却
  - day_number 順を維持
  - Trip not found message 維持
- 既存 CLI テスト（`tests/day_cli.rs`, `tests/not_found_cli.rs`）を回帰として維持

---

## What changed

- `src/services/day_list.rs` 追加
- `day::run_day_list` を廃止し `print_day_list_json` / `print_day_list_display` に分離
- `main.rs` の `DayAction::List` が service 経由に

---

## What did not change

- `day show` / `day update` / `day swap`（対象外）
- `itinerary list` / `timeline`（対象外）
- `trip list` / `show` / `stats`（対象外）
- write command / migration / FK（変更なし）

---

## Lessons learned

- `day list` の JSON は `Vec<Day>` ではなく `DayListJson` — service は `Trip` + `Vec<Day>` を返し、JSON 組み立ては CLI 層に残す
- `trip::get_trip` を service 内で呼ぶことで Trip not found を既存と同じに保てる
