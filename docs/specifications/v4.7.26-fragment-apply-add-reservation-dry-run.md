## 1. 目的

```text
fragment validate
  → fragment apply --dry-run --trip <id>
  → target itinerary 解決
  → add_reservation preview
  → gate report / --output preview Trip JSON
  → trip validate-export PASS
  → DB は変更しない
```

Proposal Fragment が Reservation 追加を提案した場合に、既存 Trip の Itinerary 配下へどの Reservation が追加されるかを、DB 更新なしで確認できるようにします。

---

## 2. supported scope

```text
intent:
  add_reservation（dry-run のみ）

target:
  itinerary のみ（trip / day は unsupported）

DB:
  read-only
  write なし
```

### 2.1 Reservation preview の保存先（export モデル）

| target | export 配置 |
|---|---|
| `itinerary` | `days[].itineraries[].reservations[]` に1件 append |

Trip / Day 直下への Reservation 追加は **unsupported**。

### 2.2 Fragment フィールド（alias 対応）

| ソース | export フィールド | 備考 |
|---|---|---|
| `candidate_content.reservation_type` / `type` | `reservation_type` | **必須** — `validate_reservation_type` |
| `candidate_content.provider` / `provider_name` | `provider_name` | **必須** — `validate_provider_name` |
| `confirmation` / `confirmation_code` | `confirmation_code` | optional |
| `site_url` / `reservation_site_url` | `reservation_site_url` | optional |
| `remark` / `note` / `memo` | `remark` | optional（Note entity は作成しない） |
| `start_at` / `end_at` | 同名 | optional |

`reservation_type` / `provider_name` が欠落または不正の場合は blocking error。

---

## 3. target 解決

| target_type | 要件 |
|---|---|
| `itinerary` | `day_reference` + `itinerary_reference`（sort_order または title）、一意に解決 |
| `trip` / `day` | **unsupported** — blocking |
| `unresolved` | **blocking** |
| ambiguous itinerary | **blocking** |

---

## 4. gate report / output

```text
--json:
  gate report JSON（FragmentApplyDryRunReport）

--output:
  preview Trip JSON（schema v8）

stdout human:
  gate report / preview summary
```

`preview` サマリ（`add_reservation` 時）:

| フィールド | 意味 |
|---|---|
| `action` | `add_reservation` |
| `reservations_before` / `reservations_after` | preview 前後の Reservation 件数（Trip 全体） |
| `reservation_preview` | 追加予定の reservation_type / provider_name / remark 等 |

### required decisions

`required_decisions` が非空の場合:

- preview Trip JSON は生成される（`--output` 指定時）
- `valid: false` — valid confirm candidate ではない
- DB 不変

---

## 5. dry-run 原則

```text
fragment apply --dry-run:
  read-only DB access
  in-memory simulation
  Trip / Day / Itinerary / Reservation domain data は変更しない
```

`add_reservation --confirm` は **v4.7.26 では未実装**。

---

## 6. 確認方法

```bash
caglla fragment apply fragment.json --dry-run --trip 1 --output preview.json
caglla trip validate-export preview.json
caglla fragment apply fragment.json --dry-run --trip 1 --json
caglla reservation list --itinerary <id> --json   # dry-run 前後で件数不変
```
