## 1. 目的

Trip Proposal Envelope JSON file を、**採用前の proposal** として最低限検証する CLI 入口を作る。

```text
入力: ローカル JSON ファイル
処理: parse → blocking validation → warnings
出力: human-readable / --json
DB: 触らない
trip import / trip validate-export: 触らない
```

---

## 2. Command

```bash
caglla proposal validate <file>
caglla proposal validate <file> --json
```

| 項目 | 内容 |
|---|---|
| 対象 | Trip Proposal Envelope JSON file |
| 非対象 | schema v8 Trip export、Proposal Fragment |
| exit code | blocking errors なし → 0、あり → 非 0 |
| warnings | exit code に影響しない |

---

## 3. Validation rules（v4.7.9 最小）

### 3.1 Blocking errors

| Rule | 内容 |
|---|---|
| JSON parse | parse 不可 |
| Trip export 混同 | `schema_version` / トップレベル `trip` / `itinerary_items` 等 |
| structure | `proposal` object 必須 |
| title | `proposal.title` 必須（非空） |
| destination | `proposal.destination` 必須（非空） |
| date_policy | 必須；`fixed_dates` / `flexible_dates` / `undated` のみ |
| fixed_dates | 確定日付不足（`start_date`+`end_date` または confirmed `candidate_date_ranges`） |
| empty body | candidate_days / candidate_itineraries / candidate_date_ranges / notes がすべて空 |

### 3.2 Non-blocking warnings

| Rule | 内容 |
|---|---|
| valid_until | 過去日 |
| stale | valid_until なし + created_at から 1 年超 |
| created_at | 欠如 |
| source / provider | 欠如 |
| materialize_hints | missing_fields / assumptions / warnings 非空 |
| candidate_days | 空 |
| candidate_itineraries | 空 |

```json

  "schema_version": 1,
  "file": "path",

  "document_kind": "trip_proposal_envelope",

```

## 5. 境界（維持）

```text
trip validate-export  → schema v8 Trip のみ
proposal validate     → Trip Proposal Envelope のみ
```

schema v8 Trip file を `proposal validate` に渡すと **blocking error**（schema_version / trip object 検出）。
