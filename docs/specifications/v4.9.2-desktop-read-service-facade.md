## 1. Goal

CLI と将来の Desktop が、同じ read-only use case facade から以下を取得できる状態にする。

1. Trip list
2. Trip detail と Day 一覧
3. Day timeline と Itinerary 一覧

GUI / Tauri / HTML / HTTP API は導入しない。write use case・export schema 変更・DB migration は含まない。

## 2. Initial investigation（棚卸し）

### 2.1 既存 `src/services/`

| モジュール | 役割 | 戻り値 |
|---|---|---|
| `trip_list` | `list_trips` | `Vec<Trip>`（domain 直返し） |
| `trip_show` | `show_trip` | `Trip` |
| `day_list` | `list_days` | `Trip` + `Vec<Day>` |
| `day_show` | `show_day` | `Trip` + `Day` + `date` + `Vec<ItineraryItem>` |
| `itinerary_timeline` | `get_timeline` | `Trip` + `Vec<ItineraryItem>`（trip 全体） |

共通: `&rusqlite::Connection` 入力、`anyhow::Result`、terminal I/O なし。v4.9.2 以前は GUI 向け DTO・structured error なし。

### 2.2 CLI read path（移行前）

| Command | Service | 表示 / JSON |
|---|---|---|
| `trip list` | `trip_list` | `trip::print_trip_list` / `TripCliJson` |
| `trip show` | `trip_show` | `trip::print_trip_detail` / `TripCliJson` |
| `day list` | `day_list` | `day::print_day_list_*` / private `DayListJson` |
| `day show` | `day_show` | `day::print_day_show_*` / private `DayShowJson` |
| `itinerary timeline` | `itinerary_timeline` | `itinerary::print_itinerary_timeline`（JSON なし） |

### 2.3 DB connection / path

```
Cli::parse()
  → config::resolve_db_path_for_cli(--db)
  → storage::db::open_db_at(path)
  → services::*( &conn, ... )
```

- path 解決は CLI（`main.rs`）のみ。service は `&Connection` のみ受け取る。
- global connection / 固定 DB path / pool / async は導入しない。

### 2.4 型の依存関係

```
domain models (Trip, Day, ItineraryItem)
  ↑ repository (trip.rs, day.rs, itinerary.rs)
  ↑ services (thin orchestration)
  ↑ main.rs + print_* / TripCliJson / DayListJson（presentation）
```

- `TripCliJson` は export JSON と分離済み（v4.9.1）。
- `DayListJson` / `DayShowJson` は `day.rs` 内 private（CLI 専用）。

### 2.5 既存 ordering

| Entity | Order | Tie-break |
|---|---|---|
| Trip list | `trips.id` ASC | — |
| Day list | `day_number` ASC | — |
| Itinerary（list / timeline / day show） | `day_number`, `sort_order` ASC | `id` ASC |

正本: [ordering-model.md](ordering-model.md)。sequence-first（`start_time` は順序に不使用）。

### 2.6 既存 error

- `anyhow::Error` + 固定文言（例: `Trip not found: {id}`）
- `storage::db::map_query_row` が `QueryReturnedNoRows` を domain 文言へ変換
- structured error の先行実装: `proposal/apply_errors.rs`（Fragment apply 専用）

## 3. Library target（Desktop import 境界）

v4.9.2 以前は **binary crate のみ** で、`src/services/mod.rs` の re-export は外部 crate から利用できなかった。

v4.9.2 で **最小 library target** を追加:

| Artifact | Path | Role |
|---|---|---|
| Library | `src/lib.rs` | read facade + `open_db` を公開 |
| Binary | `src/main.rs` | `travel_ledger_cli::run()` の thin wrapper |
| CLI dispatch | `src/cli_run.rs` | 既存 command handler（lib 内部） |

### 3.1 外部 crate 向け公開 API（`travel_ledger_cli`）

```rust
// Read facade
pub use services::{
    list_trip_summaries, get_trip_detail, get_day_timeline,
    TripSummary, TripDetail, DaySummary, DayDetail, ItineraryDetail,
    ServiceError, ReadServiceErrorCode,
};

// DB boundary
pub fn open_db(path: &str) -> anyhow::Result<rusqlite::Connection>;
```

- workspace 化・Tauri 依存追加は **v4.9.2 scope 外**（次フェーズ）
- 将来 Tauri crate は `travel_ledger_cli = { path = "..." }` で依存し、上記 API を invoke
- path 解決（`--db` / `caglla.toml`）は引き続き CLI 責務。Desktop は選択 path を `open_db` に渡す

### 3.2 検証

- `tests/lib_boundary.rs` — 外部 crate と同様に `use travel_ledger_cli::{open_db, list_trip_summaries, ...}` で compile + smoke test

## 4. Public facade API（service 層）

`src/services/mod.rs` から re-export。library crate root でも同じ surface を公開。

```rust
pub fn list_trip_summaries(conn: &Connection) -> Result<Vec<TripSummary>, ServiceError>;
pub fn get_trip_detail(conn: &Connection, trip_id: i64) -> Result<TripDetail, ServiceError>;
pub fn get_day_timeline(
    conn: &Connection,
    trip_id: i64,
    day_number: i64,
) -> Result<DayDetail, ServiceError>;
```

- 入力: 呼び出し元が開いた `&Connection`（Desktop は `open_db(path)` 後に渡す）
- 出力: UI 非依存 DTO または `ServiceError`
- terminal I/O なし。`rusqlite::Error` を公開 API から直接返さない

## 5. Ordering contract

| Use case | Contract |
|---|---|
| `list_trip_summaries` | `id` ASC |
| `get_trip_detail.days` | `day_number` ASC |
| `get_day_timeline.itineraries` | `sort_order` ASC, then `id` ASC（単日内） |

DB の偶然の返却順に依存しない。repository 層の `ORDER BY` を正本とする。

## 6. Structured error contract

`ServiceError { code: ReadServiceErrorCode, message: String }`

| Code | When | CLI message 例 |
|---|---|---|
| `TRIP_NOT_FOUND` | trip ID 不在 | `Trip not found: {id}` |
| `DAY_NOT_FOUND` | day_number 不在 | `Day not found: trip {trip_id} day {day_number}` |
| `STORAGE_FAILURE` | DB / SQL 失敗 | 既存 context 付き文言 |
| `DATA_MAPPING_FAILURE` | 日付導出失敗等 | 例: trip に `start_date` なし |

- `Display` は人間向け message（CLI stderr 互換）
- `code` は GUI 機械判定用（serde `SCREAMING_SNAKE_CASE`）
- `rusqlite::Error` は `classify_read_error` でラップし UI へ露出しない

## 7. CLI reuse 範囲

| Command | v4.9.2 | 備考 |
|---|---|---|
| `trip list` | **facade** | `list_trip_summaries` → adapter → plain / `TripCliJson` |
| `trip show` | **facade** | `get_trip_detail`（days は表示に不使用） |
| `day list` | **facade** | `get_trip_detail` → adapter → plain / `DayListJson` |
| `day show` | **facade** | `get_day_timeline` → adapter |
| `itinerary timeline` | **未移行** | trip 全体 flat timeline。facade は day-scoped。後続で `get_trip_itinerary_timeline` 候補 |
| `itinerary list` | 未移行 | 同上 |
| `itinerary show` | 未移行 | reservation 追加 fetch が main.rs に残存 |

未移行理由: v4.9.2 の公開 API は list / detail / day timeline の 3 use case に限定。trip-wide flat list は別 DTO が必要。

互換維持:

- plain-text output / CLI JSON contract / exit status / error message / golden — 変更なし

### 7.1 Legacy service 撤去（v4.9.2）

| 削除 module | 置き換え |

| `trip_list` | `list_trip_summaries` |
| `trip_show` | `get_trip_detail` |
| `day_list` | `get_trip_detail` |

有用な unit test は `read_facade.rs` へ移行。production path と test path の乖離を防ぐ。

## 8. DB connection boundary

| Layer | Responsibility |
|---|---|
| CLI `cli_run.rs` | `resolve_db_path_for_cli` → `open_db`（lib） |
| Library `open_db` | `storage::db::open_db_at` の公開ラッパ |
| Facade | `&Connection` のみ。path・global state 非依存 |
| Future Tauri | ユーザー選択 path を `travel_ledger_cli::open_db` → 同 facade を invoke |

今回不要: connection pool / async / managed state / DB picker / singleton

## 9. Future Tauri command 境界

```text
Tauri command (future)
  → travel_ledger_cli::open_db(user_selected_path)
  → travel_ledger_cli::list_trip_summaries(&conn)
  → serialize DTO → frontend

禁止:
  - CLI バイナリ spawn
  - stdout パース
  - main.rs print_* の直接利用
```

Tauri command は facade + DTO のみ。presentation（human text）は CLI adapter に残す。
