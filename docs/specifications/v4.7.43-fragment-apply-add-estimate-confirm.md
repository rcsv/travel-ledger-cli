## 1. Semantic boundary

```text
Estimate = Planned Money
Expense  = Actual Money
```

- `add_estimate --confirm` は **Estimate のみ** INSERT する
- Expense 行は作成・更新しない
- Estimate-to-Expense 変換は行わない

---

## 2. supported scope

```text
intent:   add_estimate
target:   itinerary のみ
DB:       estimates INSERT 1件のみ
```

### 2.1 許可される mutation

```text
INSERT exactly one estimates row
```

許可カラム:

```text
itinerary_id
title
amount
currency
note
sort_order
created_at
updated_at
```

### 2.2 field mapping（v4.7.42 維持）

| ソース | DB フィールド |
|---|---|
| `candidate_content.amount` | `amount`（minor units 整数） |
| `candidate_content.currency` | `currency`（3文字コード） |
| `title` / `description` / `label` | `title` |
| `note` / `memo` / `fragment.notes` | `note` |
| `sort_order` | `sort_order`（省略時 `0`） |

### 2.3 amount 入力（critical）

| 形式 | 例 | DB `amount` |
|---|---|---|
| string decimal text | `"12.50"` + `USD` | `1250` |
| integer minor units | `1250` + `USD` | `1250` |

confirm は **parsed minor units を直接** `add_estimate_minor_units` へ渡す。  
parsed `1250` を `"1250"` 文字列化して decimal parser に渡すと `125000` になるため、**禁止**。

---

## 3. target 解決

v4.7.42 dry-run と同型:

| selector | 動作 |
|---|---|
| numeric `itinerary_reference` | itinerary ID → sort_order fallback → ambiguity reject |
| title selector | 一意解決必須 |
| `--trip` scoping | 必須 |
| trip / day target | **unsupported** |
| unresolved / ambiguous | **blocking** |

confirm では transaction 内で target を **再解決**し、`estimate_preview.target_itinerary_id` と一致することを要求する。

---

## 4. 処理フロー / transaction boundary

```text
fragment apply gate（dry-run 同型 preview）
  → confirm scope 検証
  → with_transaction("add_estimate confirm")
      → Fragment JSON 再 parse
      → intent == add_estimate 確認
      → candidate fields 再 parse
      → target 再解決（現行 DB）
      → gate preview と amount/currency/title/note/sort_order/target_itinerary_id 照合
      → add_estimate_minor_units で INSERT 1件
      → get_estimate で inserted row 検証
  → commit
```

TOCTOU: gate 通過後の DB 変化に対し、transaction 内で再検証する。不一致は rollback。

---

## 5. gate report

`FragmentApplyDryRunReport` 追加フィールド:

| フィールド | 意味 |
|---|---|
| `inserted_estimate_id` | confirm 成功時の新規 Estimate ID（`Option<i64>`、`#[serde(skip_serializing_if = "Option::is_none")]`） |

成功例:

```json
{
  "confirm": true,
  "valid": true,
  "inserted_estimate_id": 123,
  "preview": {
    "action": "add_estimate",
    "estimate_preview": {
      "target_itinerary_id": 42,
      "target_itinerary_title": "Okinawa Churaumi Aquarium",
      "amount": 1250,
      "currency": "USD",
      "title": "Admission tickets",
      "note": "Estimated total",
      "sort_order": 0
    }
  }
}
```

dry-run では `inserted_estimate_id` は出力しない。

---

## 6. 成功検証

INSERT 後:

- inserted ID > 0
- `estimate show <id>` で row 読み込み可能
- `itinerary_id` / `amount` / `currency` / `title` / `note` / `sort_order` が preview と一致
- **exactly one** `estimates` row inserted
- Expense / Reservation / Note / Itinerary / Trip / Day は変更しない

---

## 7. 失敗 / rollback

以下は `valid = false`、blocking error、`inserted_estimate_id` なし、transaction rollback、domain tables 不変:

- invalid Fragment / missing Trip / missing target / unresolved / ambiguous target
- unsupported target type / cross-trip target
- invalid amount / currency / optional field / sort_order
- non-empty `required_decisions`
- gate preview mismatch
- insert failure / stored-row verification failure
