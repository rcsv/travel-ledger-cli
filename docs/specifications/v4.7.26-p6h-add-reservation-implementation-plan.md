## 1. 背景

Reservation は既存 CLI / export / write service が揃っている。

| 層 | 既存資産 |
|---|---|
| DB | `reservations`（`itinerary_id NOT NULL`） |
| export | `days[].itineraries[].reservations[]` — `ExportReservationV3` |
| CLI | `reservation add/list/show/update/delete` |
| write service | `services::reservation_add::add_reservation` |
| fragment apply | **未対応**（`VALID_INTENTS` に `add_reservation` なし） |

Fragment apply では **既存 write path を再利用**し、P-6g と同じ gate / preview / confirm 境界を踏襲する。

---

## 2. 推奨フェーズ分割

### Phase A — v4.7.26 P-6h dry-run（先行）

```text
fragment apply <file> --dry-run --trip <id>
  → fragment validate（add_reservation 認識）
  → target itinerary 解決
  → add_reservation preview
  → gate report / --output preview Trip JSON
  → trip validate-export PASS
  → DB 非更新
```

| 項目 | 内容 |
|---|---|
| Command | `fragment apply --dry-run --trip <id>` |
| intent | `add_reservation` |
| target | itinerary のみ |
| DB | read-only |
| preview 配置 | `days[].itineraries[].reservations[]` に 1件 append |

**gate report 追加候補（optional fields）:**

| フィールド | 意味 |
|---|---|
| `action` | `add_reservation` |
| `reservations_before` / `reservations_after` | Trip 全体の reservation 件数 |
| `reservation_preview` | 追加予定の reservation サマリ |

`required_decisions` 非空 → `valid: false`、DB 不変（P-6g 同型）。

### Phase B — v4.7.27 P-6h confirm（後続）

```text
fragment apply <file> --confirm --trip <id>
  → v4.7.26 と同型 gate
  → validate_confirm_scope（add_reservation + itinerary）
  → reservations INSERT 1件
  → inserted_reservation_id
```

| 項目 | 内容 |
|---|---|
| DB operation | **`reservations` INSERT 1件のみ** |
| write path | `services::reservation_add::add_reservation` |
| gate report | `inserted_reservation_id` |
| 確認 | `reservation show` / `reservation list --itinerary` |

## 3. field mapping（fragment → export / DB）

既存 `reservation add` CLI / `ReservationAddParams` / `ExportReservationV3` に合わせる。

| Fragment `candidate_content` ソース | export / DB フィールド | 必須 | 検証 |
|---|---|---|---|
| `reservation_type` / `type` | `reservation_type` | **必須** | `validate_reservation_type` |
| `provider` / `provider_name` | `provider_name` | **必須** | `validate_provider_name` |
| `confirmation` / `confirmation_code` | `confirmation_code` | optional | |
| `site_url` / `reservation_site_url` | `reservation_site_url` | optional | |
| `remark` / `note` / `memo` | `remark` | optional | |
| `start_at` | `start_at` | optional | |
| `end_at` | `end_at` | optional | |

**有効な `reservation_type`:** `hotel`, `flight`, `restaurant`, `rental_car`, `activity`, `parking`, `ticket`, `other`

不正な `reservation_type` / 空 `provider` → **blocking error**（dry-run / confirm 共通）。

## 8. unsupported scope / non-goals

以下は P-6h では **実施しない**:

- Reservation update / delete
- 複数 reservation 一括追加
- Trip / Day 直下への Reservation 追加
- `add_note` / `add_expense` 拡張
- `update_itinerary` / delete / reorder（→ P-6i 以降）
- Proposal / Fragment DB storage
- fragment show / inspect / import / list
- Proposal materialize 拡張
- schema v8 変更
- DB migration 追加
- JSON golden 変更（原則なし）
- Markdown golden（colophon のみ、バージョン bump 時）

---

## 9. 安全条件（confirm 時）

P-6g `add_expense --confirm` と同型:

- Fragment validation valid
- Trip 存在
- itinerary target 一意解決
- `reservation_type` / `provider_name` valid
- preview 生成可能
- preview Trip JSON が `trip validate-export` 相当に PASS
- blocking errors なし
- `required_decisions` 空
- `--confirm` 明示
- `--dry-run --confirm` → error
