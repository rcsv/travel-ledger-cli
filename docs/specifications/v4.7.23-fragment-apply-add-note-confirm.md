# v4.7.23 — Fragment apply add_note --confirm (P-6f)

[v4.7.22](v4.7.22-fragment-apply-add-note-dry-run.md) の `add_note` dry-run に続き、**P-6f** として `fragment apply --confirm` による Note INSERT を実装します。

| Item | Status |
|---|---|
| Release | v4.7.23 |
| Scope | `fragment apply <file> --confirm --trip <id>` — `add_note`、Note INSERT 1件 |
| Phase | **P-6f** |

---

## Status（結論サマリ）

| 観点 | v4.7.23 判定 |
|---|---|
| Command | **`fragment apply --confirm`** — `add_note` 対応 |
| intent | **`add_note` のみ**（confirm） |
| target | **trip / day / itinerary** |
| DB | **notes INSERT 1件のみ** |
| dry-run | **v4.7.22 維持** — DB 非更新 |

---

## 1. 処理フロー

```text
fragment validate
  → target 解決
  → add_note preview（v4.7.22 同型 gate）
  → confirm scope 検証
  → notes INSERT 1件
  → note list / note show で確認
```

---

## 2. supported scope

```text
intent:   add_note
target:   trip / day / itinerary
DB:       notes INSERT 1件のみ
```

### Note body（v4.7.22 維持）

```text
candidate_content.body
→ candidate_content が文字列
→ fragment.notes
```

---

## 3. gate report

| フィールド | 意味 |
|---|---|
| `inserted_note_id` | confirm 成功時の新規 Note ID |
| `inserted_itinerary_id` | add_itinerary confirm 時（v4.7.19〜） |

---

## 4. 安全条件

以下を満たさない場合は DB 更新しません:

- Fragment validation valid
- Trip 存在
- target 一意解決（unresolved / ambiguous は blocking）
- intent `add_note`
- body 非空
- preview 生成 + trip validate-export 合格
- blocking errors なし
- required decisions 空
- `--confirm` 明示

---

## 5. Non-goals（v4.7.23）

- Note update / delete
- 複数 note 一括追加
- add_expense / add_reservation / update_itinerary / delete / reorder
- schema v8 変更 / DB migration
