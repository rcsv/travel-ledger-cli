## 1. 目的

```text
fragment validate
  → fragment apply --dry-run --trip <id>
  → target itinerary 解決
  → add_expense preview
  → gate report / --output preview Trip JSON
  → trip validate-export PASS
  → DB は変更しない
```

Proposal Fragment が Expense 追加を提案した場合に、既存 Trip の Itinerary 配下へどの Expense が追加されるかを、DB 更新なしで確認できるようにします。

---

## 2. supported scope

```text
intent:
  add_expense（dry-run のみ）

target:
  itinerary のみ（trip / day は unsupported）

DB:
  read-only
  write なし
```

### 2.1 Expense preview の保存先（export モデル）

| target | export 配置 |
|---|---|
| `itinerary` | `days[].itineraries[].expenses[]` に1件 append |

Trip / Day 直下への Expense 追加は **unsupported**。

### 2.2 Fragment フィールド

| ソース | export フィールド | 備考 |
|---|---|---|
| `candidate_content.amount` | `amount` | **必須** — 整数（minor units） |
| `candidate_content.currency` | `currency` | **必須** — 3文字通貨コード |
| `candidate_content.title` / `description` / `label` | `title` | 人間向けラベル（優先順: title → description → label） |
| `candidate_content.note` / `memo` / `fragment.notes` | `note` | 補足情報（optional） |

`amount` / `currency` が欠落または不正の場合は blocking error。

---

## 3. target 解決

| target_type | 要件 |
|---|---|
| `itinerary` | `day_reference` + `itinerary_reference`（sort_order または title）、一意に解決 |
| `trip` / `day` | **unsupported** — blocking |
| `unresolved` | **blocking** |
| ambiguous itinerary | **blocking** または required decision |

`--trip` は CLI 上の正本。`trip_reference` 不一致は **warning**。

---

## 4. gate report / output

```text
--json:
  gate report JSON（FragmentApplyDryRunReport）

--output:
  preview Trip JSON（schema v8）

stdout human:
  gate report / preview summary
```

`preview` サマリ（`add_expense` 時）:

| フィールド | 意味 |
|---|---|
| `action` | `add_expense` |
| `expenses_before` / `expenses_after` | preview 前後の Expense 件数（Trip 全体） |
| `expense_preview` | 追加予定の amount / currency / title / note |

### required decisions

`required_decisions` が非空の場合:

- preview Trip JSON は生成される（`--output` 指定時）
- `valid: false` — valid confirm candidate ではない
- DB 不変

---

## 5. dry-run 原則

```text
fragment apply --dry-run:
  read-only DB access
  in-memory simulation
  Trip / Day / Itinerary / Expense domain data は変更しない
```

`add_expense --confirm` は **v4.7.24 では未実装**。

---

## 6. 確認方法

```bash
caglla fragment apply fragment.json --dry-run --trip 1 --output preview.json
caglla trip validate-export preview.json
caglla fragment apply fragment.json --dry-run --trip 1 --json
caglla expense list --itinerary <id> --json   # dry-run 前後で件数不変
```
