## Status

| 観点 | 判定 |
|---|---|
| `reservation_add` service | **追加** — `ReservationAddParams` / `ReservationAddServiceResult` |
| `reservation_update` service | **追加** — clear フラグ解釈を service 内 |
| `reservation_delete` service | **追加** — `{ id, provider_name }` snapshot |
| add/update result | **`Reservation` + display context** — read `reservation_show` と同型 optional セマンティクス |
| delete result | **`{ id, provider_name }`** — display context なし |
| enriched layer | **なし** |
| `load_reservation_display_context` | **`reservation.rs` に追加** — show / write 共通 |
| handler | service 経由 + `print_reservation_detail_with_context`（add/update） |
| `print_reservation_detail(&conn, ...)` | **write から除去** — 関数は R-5 まで残置 |
| `reservation.rs` mutation | **残置** — service から呼び出し |
| JSON output | **不変** |
| human output | **不変**（`reservation_cli` tests） |

## Write flow（実装後）

```text
reservation add:
  handler → ReservationAddParams
  service → add_reservation → get_reservation → load_reservation_display_context
  handler → println! + print_reservation_detail_with_context

reservation update:
  handler → ReservationUpdateParams (clear flags raw)
  service → clear 解釈 → update_reservation → get_reservation → load_reservation_display_context
  handler → println! + print_reservation_detail_with_context

reservation delete:
  handler → ReservationDeleteParams
  service → get_reservation (snapshot) → delete_reservation
  handler → println! + provider display
```
