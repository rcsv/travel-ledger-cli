## 1. 目的

Proposal Fragment JSON file を、**既存 Trip への未採用部分提案** として最低限検証する CLI 入口を作る。

```text
入力: ローカル JSON ファイル
処理: parse → blocking validation → warnings
出力: human-readable / --json
DB: 触らない
trip import / trip validate-export: 触らない
proposal show / inspect / materialize: 触らない
```

Proposal Fragment は「小さい Trip」ではない。schema v8 Trip export の外側に置く。

## 3. Validation rules（v4.7.11 最小）

### 3.1 Blocking errors

| Rule | 内容 |
|---|---|
| JSON parse | parse 不可 |
| Trip export 混同 | `schema_version` / トップレベル `trip` / `itinerary_items` / `days` のみ export 等 |
| Envelope 混同 | `proposal` / `materialize_hints`（fragment なし）等 |
| structure | `target` object 必須、`fragment` object 必須 |
| target_type | `target.target_type` 必須；`trip` / `day` / `itinerary` / `unresolved` のみ |
| intent | `fragment.intent` 必須；`add` / `enrich` / `replace_candidate` / `reorder_hint` / `warning` のみ |
| empty body | `candidate_content` と `notes` がともに空に近い |

### 3.2 Non-blocking warnings

| Rule | 内容 |
|---|---|
| unresolved target | `target_type` が `unresolved`、または `unresolved_target_hints` 非空 |
| ambiguous reference | day / itinerary 参照に trip 文脈不足等 |
| created_at | 欠如 |
| valid_until | 欠如、または過去日 |
| stale | valid_until なし + created_at から 1 年超 |
| source / provider | 欠如 |
| adoption_hints | missing_fields / assumptions / warnings / conflicts / required_decisions 非空 |
