## 1. Overview

v3.4.0 は **Itinerary 単位の Planned vs Actual difference** を、v3.3.0 と同じ derived 方針で **export-md Itinerary セクション** に追加した機能リリース。release 前に PR #62 で **小数通貨の負数表示**（`format_amount_value`）も修正した。

### 到達点

| 項目 | 判定 |
|---|---|
| **Difference 定義** | `Actual − Planned`（Itinerary 内・通貨別） | ✅ |
| **符号** | negative = 予定より少なく使った / positive = オーバー / zero = 一致 | ✅ |
| **表示 gate** | Itinerary 内 `estimate_count > 0 && expense_count > 0` | ✅ |
| **通貨 union** | `estimate_totals ∪ expense_totals`、欠損側 0 | ✅ |
| **export-md Itinerary** | 予定費用表 / Expenses の後に Planned / Actual / Difference サマリー | ✅ |
| **trip stats** | **変更なし**（Trip 単位 v3.3.0 のまま） | ✅ |
| **DB schema** | 変更なし | ✅ |
| **export / import schema** | v6 維持、変更なし | ✅ |
| **負数 decimal 表示** | USD `-0.50` 等（PR #62） | ✅ |
| **`make check`** | PASS | ✅ |

**結論:** estimate-model の Itinerary カード想定を export-md で実現。Difference 計算は v3.3.0 helper 再利用。Release blocker となる問題はない。

## 3. Difference Semantics Review

### 式（v3.3.0 と同一）

```text
difference_totals[currency] =
  expense_totals.get(currency).unwrap_or(0)
  - estimate_totals.get(currency).unwrap_or(0)
```

### Itinerary gate

| 状況 | サマリー |
|---|---|
| Estimate + Expense 両方あり | **表示** |
| Estimate のみ | **省略** |
| Expense のみ | **省略** |
| どちらもなし | **省略** |

Trip 単位 gate（v3.3.0 Overview）とは **独立**。

## 6. Deferred Scope

v3.4.0 および本 Review **では実装しない** 範囲:

```text
- Day 単位 difference
- trip stats / trip stats --json への itinerary_differences
- doctor / advisor Estimate 活用（v3.5.0 候補）
- Budget 独立エンティティ
- FX conversion
- Balance / Settlement / advance payment / transfer
- trip export JSON への difference 永続化
- commands/ への段階移行
```

次フェーズ候補: **Estimate-aware doctor / advisor**、**Day 単位 difference**、**commands/ 段階移行**。
