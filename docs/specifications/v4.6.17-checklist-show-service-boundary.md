## Background

v4.6.16 follow-up review で次候補として `checklist show` を推奨。

目標形:

```text
CLI / Tauri
  → service (use case, structured result, no I/O)
  → repository / domain
```

`checklist show` は ID 指定で1件の `ChecklistItem` を取得する read-only command。write 系（add / check / auto-generate 等）には触れず、既存挙動を保ったまま service boundary のみ導入する。

---

## Target command

- `checklist show <id>`
- `--json` フラグあり（`ChecklistItem` をそのまま serialize）

---

## Current flow before split

```text
main.rs (ChecklistAction::Show)
  → checklist::get_checklist_item(conn, id)
  ├─ --json  → print_json(&item)
  └─ human   → print_checklist_detail(&item)
```

---

## Service boundary introduced

新規:

```text
src/services/checklist_show.rs
```

API:

```rust
pub struct ChecklistShowServiceResult {
    pub item: ChecklistItem,
}

pub fn show_checklist(conn: &Connection, id: i64) -> Result<ChecklistShowServiceResult>
```

service は `checklist::get_checklist_item` を呼び、**structured result のみ**返す。

human 表示（`print_checklist_detail`）は CLI 層に残す。

---

## CLI compatibility

- human: `print_checklist_detail(&result.item)` — `[x]` / `[ ]` マーク含む詳細表示
- JSON: `result.item` をそのまま serialize

---

## JSON compatibility

- `--json` 時は `ChecklistItem` を serialize（shape 不変、wrapper なし）

---

## Error compatibility

- Checklist item 不存在: `Checklist item not found: <id>`

---

## checked / unchecked compatibility

- `is_done` フィールドは `ChecklistItem` に含まれる
- human 表示の `状態 : [x]` / `状態 : [ ]` は `print_checklist_detail` 内（変更なし）

---

## Test coverage

- service test:
  - 既存 item 取得
  - checked / unchecked state 維持
  - not found message 維持
- 既存 `not_found_cli::cli_checklist_show_not_found` を回帰として維持

## What changed

- `src/services/checklist_show.rs` 追加
- `src/services/mod.rs` に `checklist_show` module 追加
- `main.rs` の `ChecklistAction::Show` が service 経由に

---

## What did not change

- `checklist list` / add / update / delete / check / uncheck（対象外）
- `checklist auto-generate`（対象外）
- note / reservation / expense service（対象外）
- write command / migration / FK（変更なし）

---

## Lessons learned

- `trip show` / `itinerary show` と同様、単一 domain model の show は薄い service で十分
- JSON に wrapper がない command は `result.item` を直接 serialize
- write command 後の detail 表示（update / check / uncheck）は `get_checklist_item` 直結のまま（対象外）
