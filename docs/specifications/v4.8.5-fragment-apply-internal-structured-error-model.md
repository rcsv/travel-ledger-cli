## 1. 内部モデル

実装: `src/proposal/apply_errors.rs`（`apply` module 配下）

| 型 | 役割 |
|---|---|
| `ApplyErrorCode` | stable machine-readable code（registry） |
| `ApplyErrorKind` | `blocking` / `warning` / `decision_required` |
| `ApplyErrorPhase` | `fragment_validate` / `gate` / `confirm_scope` / `simulate` / `confirm_transaction` |
| `StructuredApplyError` | code + kind + message + phase + optional metadata |

`FragmentApplyDryRunReport` に `structured_errors: Vec<StructuredApplyError>` を追加（`#[serde(skip)]`）。

`push_structured_error` は structured vec へ追加し、同じ `message` を legacy `errors[]` へ mirror（dedupe）。

## 2. Stable code registry

| Code | 用途（代表） |
|---|---|
| `APPLY_UNSUPPORTED_INTENT` | simulate / confirm scope で未対応 intent |
| `APPLY_UNSUPPORTED_TARGET` | intent が要求する target_type と不一致 |
| `APPLY_TARGET_UNRESOLVED` | `target_type=unresolved` |
| `APPLY_TARGET_AMBIGUOUS` | itinerary selector 曖昧、confirm scope ambiguous |
| `APPLY_REQUIRED_DECISION` | `required_decisions` 非空 |
| `APPLY_FIELD_INVALID` | target / candidate 必須フィールド欠落・不正 |
| `APPLY_MUTATION_FIELD_REJECTED` | delete_estimate 等での mutation field 拒否 |
| `APPLY_ENTITY_NOT_FOUND` | day / itinerary / estimate 不存在 |
| `APPLY_ENTITY_PARENT_MISMATCH` | estimate の itinerary 配下不一致 |
| `APPLY_BASELINE_MISMATCH` | `expected_*` baseline 不一致 |
| `APPLY_PREVIEW_MISMATCH` | （registry 定義のみ — v4.8.5 wiring 対象外） |
| `APPLY_TOCTOU_DRIFT` | （registry 定義のみ — v4.8.5 wiring 対象外） |
| `APPLY_SCOPED_WRITE_ZERO_ROWS` | （registry 定義のみ） |
| `APPLY_SCOPED_WRITE_MULTIPLE_ROWS` | （registry 定義のみ） |
| `APPLY_TRANSACTION_FAILED` | （registry 定義のみ） |

`ApplyErrorCode::as_str()` が JSON 露出時の正本文字列（SCREAMING_SNAKE_CASE）。

## 3. Limited wiring（v4.8.5）

### Gate / resolver（`ApplyErrorPhase::Gate`）

- `resolve_apply_target`: missing `target_type`、`unresolved`、unsupported `target_type`、day not found、itinerary ambiguous / not found、invalid `day_reference`
- gate 末尾: `required_decisions` 非空 → `APPLY_REQUIRED_DECISION`

### Confirm scope（`ApplyErrorPhase::ConfirmScope`）

- ambiguous target、`required_decisions` 非空
- 代表: `add_expense` / `delete_estimate` の unsupported target
- 未対応 intent/action 組み合わせ → `APPLY_UNSUPPORTED_INTENT`

### Simulate（`ApplyErrorPhase::Simulate`）

- 未対応 intent → `APPLY_UNSUPPORTED_INTENT`
- `add_expense` unsupported target → `APPLY_UNSUPPORTED_TARGET`
- `update_estimate` / `delete_estimate`: unsupported target、ambiguous target、entity not found、parent mismatch、baseline mismatch
- `delete_estimate` parse: mutation field rejection、invalid `estimate_id`

### 未 wiring（v4.8.6+ 候補）

- confirm transaction: preview mismatch、TOCTOU、scoped write rows、transaction failed
- parser 全 intent 横断
- fragment file validation 段階
