## Background

v4.6.14 / v4.6.21 で expense list/show を service 化したが、enriched context（payer name、beneficiaries、shared）は CLI helper `expense_to_json(&conn, ...)` が解決していた。

v4.6.25 で Option C を採用:

```text
service → Expense + enriched context parts
CLI     → enriched_expense_to_json → ExpenseJson（shape 不変）
```

v4.6.26 で Phase 2（service enrichment）と Phase 3（CLI mapper 接続）を同一実装で完了。

---

## Types introduced

```rust
// src/expense.rs — service/CLI 間の enriched context（wire DTO ではない）
pub(crate) struct ExpenseBeneficiaryPart {
    pub participant_id: i64,
    pub name: String,
    pub sort_order: i32,
}

pub(crate) struct ExpenseEnrichedPart {
    pub expense: Expense,
    pub paid_by_participant_name: Option<String>,
    pub beneficiaries: Vec<ExpenseBeneficiaryPart>,
    pub shared: bool,
}
```

`ExpenseJson` / `ExpenseBeneficiaryJson` は **CLI JSON wire format** として維持。

---

## Enrichment (`enrich_expense`)

service と adapter が共有する DB context 解決:

```text
list_beneficiaries_for_expense(conn, expense.id)
get_participant(conn, paid_by_participant_id)     // payer name
get_participant(conn, beneficiary.participant_id) // per beneficiary
expense_is_shared(beneficiary_count)              // count > 0 → shared
```

terminal I/O なし。

---

## CLI mapper (`enriched_expense_to_json`)

```rust
pub(crate) fn enriched_expense_to_json(part: &ExpenseEnrichedPart) -> ExpenseJson
```

- `&Connection` を受け取らない
- 既存 `ExpenseJson` フィールドを 1:1 マップ
- tests で `expense_to_json` 出力と `serde_json` 等価を確認

## CLI handler changes — `main.rs`

### `ExpenseAction::List`

```text
--json  → enriched_expense_to_json × N → ExpenseListJson → print_json
human   → print_expense_list_from_enriched(target, &expenses)
```

### `ExpenseAction::Show`

```text
--json  → enriched_expense_to_json → print_json
human   → print_expense_detail_from_enriched
```

## Adapters retained (write path / compatibility)

| Helper | 状態 | 用途 |
|---|---|---|
| `expense_to_json(&conn, ...)` | 残置 | `enrich_expense` + mapper の adapter；tests |
| `print_expense_detail(&conn, ...)` | 残置 | write path（add/update/delete 後表示） |
| `print_expense_list(&conn, ...)` | 残置 | adapter（read-only は `_from_enriched` 使用） |

`expense add` / `update` / `delete` は **未変更** — 引き続き `print_expense_detail(&conn, ...)`。
