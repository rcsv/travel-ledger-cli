## 1. IsoStrict 適用 surfaces

### 1.1 CLI write path（v4.8.13）

| Command | Validation | 備考 |
|---|---|---|
| `expense add` | `validate_cli_write_currency_code` → IsoStrict | `main.rs` entry |
| `expense update` | currency 明示時のみ IsoStrict | `--currency` 未指定なら既存値 untouched |
| `estimate add` | IsoStrict | `main.rs` entry |
| `estimate update` | currency 明示時のみ IsoStrict | amount+currency ペア contract 維持 |

API:

```rust
pub(crate) fn validate_cli_write_currency_code(code: &str) -> Result<String>
// → validate_currency_code_with_mode(code, IsoStrict)
```

### 1.2 format-only 維持 surfaces

| Surface | 理由 |
|---|---|
| `validate_currency_code()` wrapper | 既定 FormatOnly — domain / import 互換 |
| `expense::add_expense` / `update_expense` | domain 層 — Fragment apply 等が利用 |
| `estimate::add_estimate` / `update_estimate` | domain 層 — import / Fragment / internal tests |
| `trip import` / `validate-export` | v4.8.15 まで format-only |
| Proposal Fragment apply parse | v4.8.14 まで format-only |
| Receipt / inbox add/update | v4.8.14+ defer（workflow 複合のため） |
| read / list / show / export / export-md | legacy unknown currency 非破壊 |

## 2. Strict reject 方針

| Input | Result |
|---|---|
| `JPY`, `USD`, `EUR`, withdrawn（`HRK`, `LTL` 等） | OK（uppercase normalize） |
| `jpy`, `usd` | OK → normalize |
| `ZZZ`, `ABC`, `JPN` | reject — `currency は有効な ISO 4217 コードではありません` |
| `XXX`, `XTS`, `XAU`, `XAG` 等 denylist | reject — `currency は旅行費用として許可されていません: {code}` |
| `JP`, `JPYY`, 空文字 | reject — 既存 format message |

---

## 3. Update path contract

### expense update

- `--currency` **未指定** → 既存 DB currency を再検証しない（legacy `ZZZ` 等 OK）
- `--currency` **指定** → IsoStrict + `--amount` 必須（既存 contract）

### estimate update

- `--currency` **未指定** → 既存 currency untouched
- `--currency` **指定** → IsoStrict + `--amount` 必須（既存 contract）

---

## 4. Legacy unknown currency 互換

| Operation | Policy |
|---|---|
| DB に `ZZZ` 等 legacy row | read / list / show / export 可能 |
| CLI update without `--currency` | block しない |
| CLI add with `ZZZ` | **reject**（新規入力 strict） |
| CLI update with `--currency ZZZ` | **reject** |

Unit tests: `expense.rs`, `estimate.rs` — SQL insert legacy row + update without currency
