# Travel Support Design Memo — 旅行支援情報・注意喚起に関する知見

Caglla.Travel **Web 版**を運用していた頃に得た、旅行支援情報・注意喚起機能に関する設計知見です。将来の方向性を考える上での参考資料であり、**v1.x の実装対象ではありません**。

関連: [Checklist 設計メモ](checklist-design-memo.md)（自動生成・Provenance）/ [Travel Ledger Responsibilities](travel-ledger-responsibilities.md) / [Export Schema](export-schema.md)

**v1.x の最優先事項（変更なし）:**

- データモデルの安定化
- Export / Import
- データ完全性

---

## 1. ユーザーが本当に求めていたもの

当初は、

> Checklist = 持ち物リスト

だと考えていました。

しかし利用者との会話や実運用を通じて、

> 持ち物よりも事前準備や注意喚起の方が価値を感じてもらえる

ケースが多くありました。

### 持ち物

```text
パスポート
運転免許証
水着
```

よりも、

### 事前準備

```text
インド e-Visa を申請する
海外旅行保険へ加入する
予防接種を受ける
```

の方が旅行失敗防止への寄与が大きい。

さらに、

### 注意喚起

```text
パスポート残存期間6ヶ月以上必要
雨季です
黄熱病ワクチン推奨
```

のような情報の方が価値を感じてもらえる場合がありました。

---

## 2. Checklist は将来的に旅行準備支援へ発展する可能性がある

将来の Checklist は単なる Packing List ではなく、次のような複数の役割を持つ可能性があります。

| 役割 | 例 |
|---|---|
| **Packing** | 持ち物の確認 |
| **Task** | 事前に完了すべき手続き |
| **Warning** | 旅行失敗につながる条件の注意 |
| **Advisory** | 推奨・勧告・参考情報 |

例:

```text
[ ] パスポートを持参する
[ ] e-Visaを申請する

! パスポート残存期間6ヶ月必要
! 黄熱病ワクチン推奨
```

自動生成の設計原則（Provenance、最小限・高信頼など）: [checklist-design-memo.md](checklist-design-memo.md)。

**v1.x:** 上記の役割拡張は予定なし。あくまで将来の方向性メモ。

## 3. Destination 情報の価値

Web 版では旅行データに主要な渡航先を保存し、

- 外務省情報
- 天気
- 地域情報

などを表示していました。

当時は補助機能のつもりでしたが、振り返ると旅行支援として非常に価値が高い機能でした。

旅行者が知りたい情報の多くは、

```text
Trip
 ↓
Destination
```

から派生します。

例: `Japan → India` なら、

- ビザ
- パスポート条件
- 予防接種
- 安全情報
- 気候

などが導出可能になります。

将来的に Caglla.Travel が発展する場合、Trip から **Country** / **Region** / **Destination** などの概念へ接続できる余地を残しておくことが望ましい。

## 4. 外務省オープンデータの活用可能性

将来的に調査したところ、外務省は海外安全情報をオープンデータとして公開しています。

特徴:

- XML 形式
- 国別・地域別情報
- 危険情報
- 在外公館情報
- 安全情報

などを取得可能。

ただし、

- REST API ではない
- XML ダウンロード型
- 情報は文章主体

であるため、

そのまま

```text
visa_required = true
```

のような構造化データにはなりません。

利用する場合は、

- XML 解析
- 独自ルール化
- キャッシュ

などが必要になると思われます。

とはいえ、旅行支援情報の **信頼できる一次ソース候補** としては非常に有力です。

## 5. 将来提供すると価値がありそうな情報

Web 版の経験と設計議論から、旅行者が価値を感じそうな情報候補を列挙します。

### 渡航条件

- ビザ要否
- パスポート残存期間
- 入国条件

### 安全情報

- 外務省危険情報
- 感染症情報
- 渡航注意情報

### 気候

- 月別平均気温
- 月別降水量
- 雨季 / 乾季

※ 短期天気予報よりも **長期統計情報** の方が価値が高い可能性あり

### 通貨

- 現地通貨
- 為替レート目安
- キャッシュ文化 / カード文化

### 緊急時情報

- 日本大使館
- 日本領事館
- 緊急連絡先

### 現地コミュニケーション

- 簡単な現地フレーズ
- 緊急時によく使う表現

## 6. 今回の学び

Web 版で得た最も大きな学びは、

> **人が本当に価値を感じるのはデータ入力機能そのものではなく、そのデータから得られる支援情報かもしれない**

ということです。

旅行計画データは目的ではなく、**旅行準備や意思決定を支援するための基盤**として機能する可能性があります。

ただし現時点では、まずは Export / Import とデータモデル整備を優先し、本メモの内容は将来の設計参考として保管するのが適切と考えます。

**v1.x:** DB / CLI / export schema 変更なし。Destination エンティティや外務省データ連携は本メモのスコープ外。
