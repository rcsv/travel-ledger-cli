## 1. Overview

v3.2.0 は **DB 参照先の可視化** を追加した。CWD 相対 `caglla.db` の解決ルールは変更せず、**表示・確認系** コマンドでサンプル DB・本番 DB・テスト DB の混同を防ぐ。

### 到達点

| 項目 | 判定 |
|---|---|
| **`db path`** | 絶対パス 1 行表示。DB open なし | ✅ |
| **`db status`** | 存在確認 +（存在時）size / table counts | ✅ |
| **`db status --json`** | pretty JSON のみ、CLI envelope `schema_version: 1` | ✅ |
| **未存在 DB** | open しない・ファイルを作成しない | ✅ |
| **`db reset`** | 従来どおり `open_db()` → `reset_db()` | ✅ |
| **DB パス切替** | 未実装（v3.2.0 non-goal） | ✅ 意図どおり |

**結論:** Implementation Plan で定義した「open 前 dispatch」「未存在 DB 非作成」「trip export schema version の明示」が実装・テスト・ドキュメントで一貫している。

## 3. Behavior Review

| 観点 | レビュー | 判定 |
|---|---|---|
| **`db path` — no open** | `resolve_db_path()` + `println` のみ。`Connection::open` なし | ✅ |
| **`db path` — no create** | `main.rs` で `open_db()` 前に return | ✅ |
| **`db status` — missing DB** | `path_buf.exists()` が false なら `collect_db_status` が early return | ✅ |
| **`db status` — existing DB** | `open_db_at(&path)` → migration 後 `collect_table_counts` | ✅ |
| **JSON envelope** | `schema_version: 1` = CLI JSON フォーマット版 | ✅ |
| **trip export schema** | `trip_export_schema_version` = `TRIP_EXPORT_SCHEMA_VERSION`（**6**）。SQLite migration version ではない | ✅ |
| **`exists: false` JSON** | `file_size_bytes` / `table_counts` は **省略**（`null` ではない） | ✅ |
| **`db reset` 回帰** | `DbAction::Reset` のみ共通 `open_db()` フロー。他コマンド unchanged | ✅ |

## 5. Known Non-blocking Notes

| 項目 | 備考 |
|---|---|
| 極古 / 破損ローカル DB | 存在する DB に対する `db status` は `open_db_at` 経由のため、migration 不能なファイルではエラーになり得る — 他コマンドと同型 |
| macOS `/private` プレフィックス | 絶対パス表示は OS 依存。機能上問題なし |
| `db path --json` | v3.2.0 では意図的に未提供 |

## 6. Deferred Scope

```text
- --db <path>

- doctor / advisor command hint 修正
```

## 7. Conclusion

Database status foundation は、**参照 DB の明示** と **未存在 DB への副作用なし** という v3.2.0 の目的を満たしている。Release blocker となる問題はない。
