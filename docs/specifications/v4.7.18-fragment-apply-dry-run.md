## 1. apply --dry-run とは

```text
fragment apply --dry-run:
  reads an existing Trip from the local DB
  simulates applying a Proposal Fragment in memory
  writes preview output only when --output is specified
  does not modify Trip / Day / Itinerary data
  does not perform fragment apply --confirm
```

**apply preview / apply simulation** — Fragment を適用した場合に何が起きるかを計算・表示する。

`proposal materialize --dry-run`（v4.7.16）とは異なり **file-only ではない**。既存 Trip の解決のため **read-only DB access** が必要。

**原則（v4.7.17 から継承）:**

```text
dry-run means no side effects.
```

ここでの意味: **Trip domain data を変更しない**（read-only DB access は許容）。

---

## 2. Command

```bash
caglla fragment apply <file> --dry-run --trip <id> [--output preview.json] [--json]
```

| フラグ | 用途 |
|---|---|
| `--dry-run` | **必須** — apply preview のみ |
| `--trip` | 対象 Trip ID（**必須**） |
| `--output` | apply preview（schema v8 Trip JSON）の出力先 |
| `--json` | gate report JSON（preview そのものではない） |

`--dry-run` なしは **エラー**（`--confirm` 未実装）。

---

## 3. 処理フロー

1. Fragment JSON 読み込み
2. `fragment validate` 相当
3. `--trip` で Trip 存在確認
4. target 解決（trip / day / itinerary / unresolved）
5. intent に基づく **in-memory apply simulation**
6. preview Trip JSON 生成 + `trip validate-export` 自己検証
7. gate report 出力

---

## 4. Target 解決

| target_type | apply dry-run |
|---|---|
| `trip` | Trip 全体 — resolved |
| `day` | `day_reference` 必須、Trip 内に存在すること |
| `itinerary` | `day_reference` + `itinerary_reference`（sort_order または title） |
| `unresolved` | **blocking** — required decisions |

`--trip` は CLI 上の正本。`trip_reference` 不一致は **warning**。

---

## 5. Gate report

`FragmentApplyDryRunReport`（schema version 1）:

| フィールド | 意味 |
|---|---|
| `fragment_valid` | `fragment validate` 相当 |
| `trip_export_valid` | preview が `trip validate-export` 合格 |
| `resolved_target` | 解決結果 |
| `preview` | intent / action / itinerary 件数変化 |
| `errors` / `warnings` / `required_decisions` | gate 三層 |

### 5.1 intent 別 preview（v4.7.18 最小）

| intent | preview action |
|---|---|
| `add` | 対象 Day に itinerary 追加シミュレーション |
| `enrich` | Day summary 更新シミュレーション（summary がある場合） |
| `warning` | 構造変更なし |
| `replace_candidate` / `reorder_hint` | warning のみ（構造変更シミュレーションなし） |

---

## 6. 出力ストリーム

| モード | stdout | apply preview JSON |
|---|---|---|
| human, `--output` なし | gate report + preview JSON | stdout に続く（人間向け） |
| `--output <file>` | gate report | **ファイル** — `trip diff` 等向け |
| `--json` | gate report JSON のみ | `--output` 指定時のみファイル |

```text
apply preview を後続コマンドで扱う場合は --output を使う。
--json は gate report JSON を出すものであり、preview Trip JSON そのものではない。
```
