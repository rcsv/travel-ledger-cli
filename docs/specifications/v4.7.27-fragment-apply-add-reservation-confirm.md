## 1. 処理フロー

```text
fragment validate
  → target itinerary 解決
  → add_reservation preview（v4.7.26 同型 gate）
  → confirm scope 検証
  → reservations INSERT 1件
  → reservation list / reservation show で確認
```

---

## 2. supported scope

```text
intent:   add_reservation
target:   itinerary のみ
DB:       reservations INSERT 1件のみ
```

Trip / Day 直下への Reservation 追加は **unsupported**。

### field mapping（v4.7.26 維持）

| ソース | DB / export フィールド |
|---|---|
| `reservation_type` / `type` | `reservation_type` |
| `provider` / `provider_name` | `provider_name` |
| `confirmation` / `confirmation_code` | `confirmation_code` |
| `site_url` / `reservation_site_url` | `reservation_site_url` |
| `remark` / `note` / `memo` | `remark` |
| `start_at` / `end_at` | `start_at` / `end_at` |

`note` / `memo` は Reservation `remark` のみ。Note entity は作成しない。

---

## 3. gate report

| フィールド | 意味 |
|---|---|
| `inserted_reservation_id` | confirm 成功時の新規 Reservation ID |
| `reservation_preview` | dry-run 同型 preview サマリ |

---

## 4. 安全条件

以下を満たさない場合は DB 更新しません:

- Fragment validation valid
- Trip 存在
- target itinerary 一意解決（unresolved / ambiguous は blocking）
- intent `add_reservation`
- `reservation_type` / `provider_name` valid
- preview 生成 + trip validate-export 合格
- blocking errors なし
- required decisions 空
- `--confirm` 明示

---

- Reservation update / delete
- 複数 reservation 一括追加
- Trip / Day 直下 Reservation
- Note entity 作成
- add_expense / add_note 拡張
- schema v8 変更 / DB migration
- Proposal storage / import / materialize / AI / GUI
