| Phase | P-4 前の足場固め（storage strategy 本体は未着手） |

## 1. 目的

```text
公開 examples の正しい validate command が常に PASS
誤った command への投入は blocking error / invalid
将来の docs / example 変更で境界が壊れたら CI が落ちる
```

新 CLI 機能は追加しない。guard のみ。

## 3. Guard tests

`tests/public_examples_validation_guard.rs`

| Test | 内容 |
|---|---|
| `guard_schema_v8_public_examples_pass_trip_validate_export` | 3 本 PASS |
| `guard_envelope_non_normative_examples_pass_proposal_validate` | envelope PASS |
| `guard_fragment_non_normative_examples_pass_fragment_validate` | fragment PASS |
| `guard_schema_v8_trip_rejects_proposal_validate` | Trip → proposal FAIL |
| `guard_schema_v8_trip_rejects_fragment_validate` | Trip → fragment FAIL |
| `guard_envelope_rejects_fragment_validate` | Envelope → fragment FAIL |
| `guard_fragment_rejects_proposal_validate` | Fragment → proposal FAIL |
| `guard_envelope_rejects_trip_validate_export` | Envelope → validate-export FAIL |
| `guard_fragment_rejects_trip_validate_export` | Fragment → validate-export FAIL |

実行:

```bash
make check
# または
cargo test --test public_examples_validation_guard
```

Rust CI（`.github/workflows/rust.yml`）は `cargo test` を実行するため、guard は push / PR で自動実行される。v4.7.14+ では guard テストは isolated temp workdir を使用し、`trip validate-export` は DB open 前に dispatch される。
