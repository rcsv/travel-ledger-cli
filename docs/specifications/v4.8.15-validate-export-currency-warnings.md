# v4.8.15 — validate-export currency warnings (implementation)

| Release | v4.8.15（released — 本書） |

既存 DB / import-export データを壊さず、`trip validate-export` で currency 品質問題を **warning** として surface する implementation release です。

v4.8.13（CLI write path）と v4.8.14（Fragment apply）で新規入力は IsoStrict 化済み。本 release は **validate-export の data quality signal** に限定します。

---

## 0. Status（結論サマリ）

| 観点 | v4.8.15 判定 |
|---|---|
| 目的 | validate-export で unknown / denylist currency を warning 化 |
| 適用面 | `trip validate-export` — nested `expenses[]`（schema ≥ v3）、`estimates[]`（schema ≥ v6） |
| 未適用 | trip import strict reject、read/export、Receipt/inbox CLI、domain layer |

| `EXPORT_VALIDATION_REPORT_SCHEMA_VERSION` | **変更なし**（`1` 維持） |
| structured error code | **追加なし** — plain `warnings[]` string |

---

## 1. Warning / error 方針

| Condition | Severity |
|---|---|
| empty / non-alpha-3 format | **error**（現行維持 — `validate_trip_export_v3` / `collect_export_estimate_validation_errors`） |
| alpha-3 format OK, registry unknown（例 `ZZZ`, `ABC`） | **warning** |
| alpha-3 format OK, denylist（例 `XXX`, `XTS`, `XAU`） | **warning** |
| valid ISO active / withdrawn（例 `JPY`, `HRK`） | pass（warning なし） |
| lowercase input（例 `jpy`） | normalize 後に valid なら pass |

### Warning messages

```text
{field_path}: currency code is not a known ISO 4217 code: {CODE}
{field_path}: currency code is not allowed for travel expenses: {CODE}
```

`field_path` 例: `days[0].itineraries[0].expenses[0].currency`

---

## 2. Implementation

### Helper（`src/money/mod.rs`）

```rust
pub(crate) fn validate_export_currency_quality_warning(
    code: &str,
    field_path: &str,
) -> Option<String>
```

- format invalid → `None`（既存 error path に委譲）
- denylist → travel-expense warning
- unknown ISO → registry warning
- valid ISO → `None`

CLI write / Fragment apply の `validate_cli_write_currency_code` / `validate_fragment_candidate_currency` とは **責務分離**。

### Collector（`src/trip.rs`）

`collect_export_currency_quality_warnings(export, effective_schema)` — `analyze_trip_export_json` から `report.warnings` に extend。
