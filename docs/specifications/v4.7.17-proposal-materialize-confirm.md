## 1. 目的

```text
proposal validate
  → proposal materialize --dry-run（任意・推奨）
  → proposal materialize --confirm
  → 新規 Trip として DB 保存
  → trip list / trip show で確認
```

`--confirm` なしでは DB に書き込まない。

---

## 2. Command

```bash
caglla proposal materialize <file> --confirm [--start YYYY-MM-DD] [--end YYYY-MM-DD] [--json]
caglla proposal materialize <file> --dry-run [--output trip.json] [--json]
```

| フラグ | 用途 |
|---|---|
| `--confirm` | gate 通過後に **新規 Trip** として DB 保存 |
| `--dry-run` | Trip JSON 候補を file 出力（**DB open なし・副作用なし**） |
| `--dry-run` + `--confirm` | **併用不可（エラー）** — `dry-run means no side effects` |
| いずれもなし | **エラー** |

`--confirm` 経路のみ DB open する。`--dry-run` のみは v4.7.16 同様 file-only dispatch。

---

## 3. 処理フロー

1. Envelope validation（`proposal validate` 同型）
2. materialize gate（日付解決・採用内容）
3. schema v8 Trip JSON 候補生成
4. `analyze_trip_export_json` 自己検証
5. **`--confirm` 時:** `import_trip_from_json_with_summary` で新規 Trip insert
6. gate report + import summary 出力
