**`required_decisions` に関する注記（誤解防止）:**

Authoritative field-level blocking conditions are re-parsed and revalidated inside the confirm transaction (`parse_delete_estimate_fields`, ownership, baseline, preview agreement, TOCTOU).

The remaining P2 concerns rechecking the opaque `adoption_hints.required_decisions` report container itself from the Fragment JSON inside the transaction; it does not represent a known data-safety or rollback defect.

日本語: confirm transaction 内では **field 単位の blocking 条件**（mutation field、invalid `estimate_id`、wrong parent、stale baseline、preview mismatch、TOCTOU drift 等）は `parse_delete_estimate_fields` 等により **再 parse・再検証済み**（v4.8.2）。残る P2 は、gate report の opaque な `required_decisions` **コンテナ**を Fragment から transaction 内で再読しない点に限られる。**既知の data-safety 欠陥や rollback 不備ではない**。

- **v4.8.3**: 実装しない（v4.8+ API readiness / cross-cutting planning）

## 3. Confirm transaction safety

### 3.1 処理順（実装 vs 契約）

| Step | 契約 | 実装 | 判定 |
|---|---|---|---|
| 1 | gate preview 取得 | `report.preview` / `estimate_delete_preview` | ✓ |
| 2 | Fragment 再 parse | transaction 内 `serde_json::from_str` + intent 検証 | ✓ |
| 3 | `required_decisions` | gate report 非空チェック | ✓（P2-6 注記） |
| 4 | delete fields 再 parse | `parse_delete_estimate_fields(fragment_body, None)` | ✓ |
| 5 | target 再 resolve | `resolve_apply_target` | ✓ |
| 6 | row 再取得 + ownership | `get_estimate` + `itinerary_id` 一致 | ✓ |
| 7 | preview 再構築 | `build_estimate_delete_preview` | ✓ |
| 8 | gate preview agreement | `verify_delete_estimate_gate_preview_matches` | ✓ |
| 9 | `expected_*` + TOCTOU | `revalidate_delete_estimate_before_write` | ✓ |
| 10 | scoped DELETE | `delete_estimate_row_scoped` | ✓ |
| 11 | exactly-one | affected rows match 0/1/2+ | ✓ |
| 12 | post-delete 確認 | `verify_estimate_deleted` | ✓ |
| 13 | success 時 commit | `with_transaction` | ✓ |
| 14 | `deleted_estimate_id` | `ConfirmInsertResult::DeletedEstimate` | ✓ |

### 3.2 Preview agreement / informative-only title

`estimate_delete_snapshot_agreement_fields_match` は次を比較:

- `target_itinerary_id`, `target_estimate_id`
- `title`, `amount`, `currency`, `note`, `sort_order`, `updated_at`

**`target_itinerary_title` は agreement 対象外**（informative-only）。Itinerary タイトル変更後も confirm 成功する integration test（`itinerary_title_change_confirm_still_succeeds`）と一致。

### 3.3 Rollback vs user-facing undo

| 概念 | 実装 | 判定 |
|---|---|---|
| transaction rollback | `with_transaction` 失敗時 DB 不変 | ✓ |
| user-facing undo / restore | **未実装** | ✓（planning / policy どおり） |
| 利用者向け文言 | warning: undo 非保証; error: DB 更新しません | rollback と undo の混同なし |
