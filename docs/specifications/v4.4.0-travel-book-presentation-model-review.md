# v4.4.0 — Travel Book presentation model review

Caglla.Travel CLI **v4.3.x** 時点の `trip export-md`（Travel Book Generator v0）を、**renderer 非依存の表示モデル**として再整理するための設計レビューです。

## 1. Summary

`trip export-md` は **機能検証用の Travel Book 出力**であり、macOS native app や caglla.travel v2 に進む前の **読み物としての旅程表示**を試す場です。

現状、章組み立て・表示可否・冗長抑制・ラベル選択の多くが `markdown.rs` に同居している。Markdown は **renderer の一つ** であり、正本ではない。

**結論:**

1. **Domain model は維持** — Trip / Day / Itinerary / Reservation / Estimate / Note の責務は変えない
2. **Presentation model（view model）を段階的に導入** — renderer が共有する「何を・どの順で・どのラベルで見せるか」を表現
3. **大規模抽象化は v4.4.0 では行わない** — 本書は分類と最小スケッチのみ。実装は承認後の小さな抽出から
4. **Markdown 専用の業務ルールを増やさない** — これ以上の polish は presentation 層の設計を先に固めてから

---

## 2. Architectural principle

```text
Domain data  →  Travel Book presentation model  →  Renderer (Markdown / GUI / Web)
     ↑                        ↑                           ↑
  DB / export            章・項目・cue・ラベル            見出し級・表構文・余白
```

| 層 | 正本 | 例 |
|---|---|---|
| **Domain** | SQLite + trip export schema | `ItineraryItem.category`, `Reservation.confirmation_code` |
| **Presentation** | Travel Book 専用（renderer 非依存） | 「この itinerary に reservation cue あり」「provider 見出しは冗長」 |
| **Renderer** | 出力形式ごと | `#### 06:00 タイトル`、`\| Item \| Amount \|` |

Markdown 出力は **presentation model の検証用レンダラ**として扱う。Okinawa golden は **期待される presentation の具体例**であり、Markdown 構文そのものが設計の正本ではない。

## 4. What should a Travel Book itinerary item contain?

Daily schedule の **1 itinerary 行**を、renderer 非依存の **TravelBookItineraryItem**（仮称）として整理する。

### 4.1 Core fields（domain 由来・そのまま露出可）

| フィールド | ソース | Presentation 上の意味 |
|---|---|---|
| `day_number` | Itinerary.day | 日付コンテキストのキー |
| `sort_order` | Itinerary | 並び順（sequence-first） |
| `itinerary_id` | Itinerary.id | 内部参照・cue 紐づけ用（GUI では非表示が既定） |
| `title` | Itinerary.title | 主ラベル |
| `start_time` | Itinerary.start_time | **時刻ラベル**（任意。未設定なら時刻なし表示） |
| `category` | Itinerary.category | カテゴリ enum（domain） |
| `location` | Itinerary.location | **場所ラベル** |
| `duration_minutes` | Itinerary | 滞在時間（Travel Book では通常非表示 — v4.3.x 沖縄も Daily に出さない） |
| `travel_minutes` | Itinerary | 移動時間（同上） |
| `inline_memo` | Itinerary.note | **インライン memo**（Note entity ではない） |

### 4.2 Derived / linked cues（presentation 層で付与）

フルデータを Daily schedule に重複展開せず、**cue（存在・要約）** として持つ。

| Cue | ソース | 現状（沖縄） | 推奨 |
|---|---|---|---|
| `has_reservation` | Reservation が itinerary に紐づく | Reservations 章にのみ（Daily には cue なし） | **Defer** Daily 内ハイライト（v4.3.1 V11） |
| `reservation_cue` | 確認番号・期間の有無など | 未実装 | 将来: `予約あり` / `確認番号あり` 等の短い cue |
| `has_planned_cost` | Estimate が itinerary に紐づく | Planned cost 章にのみ | 同上 — 章分離を維持 |
| `planned_cost_cue` | 通貨・件数 | 未実装 | 将来: `予定費用 1 件` 等 |

**原則:** cue は **索引・注意喚起**。詳細は Reservations / Planned cost 章（または GUI の別パネル）に置く。

### 4.3 Future: schedule role（fixed / flexible / optional）

| ロール | 意味 | v4.4.0 |
|---|---|---|
| fixed | 時刻・予約が固定（便・チェックイン） | **Defer** — domain フィールドなし |
| flexible | だいたいの順序は決まっているが時刻は目安 | **Defer** |
| optional | 余裕があれば | **Defer** |

自動判定（doctor / advisor 連携）は v4.0.0 Highlights と同様 **Defer**。Presentation model には **拡張ポイントとして型だけ予約**してよいが、v4.4.0 では実装しない。

### 4.4 Okinawa fixture 観測（Daily schedule 1 件）

```markdown
#### 12:30 レンタカー屋へのシャトルバス乗車

- Category: transport
- 場所: 那覇国際空港
- メモ: スタッフに電話 : +81 80 6489 6688。…
```

| 観測 | 分類 |
|---|---|
| 時刻 + タイトルを 1 行に結合 | **Presentation**（primary label 組み立て） |
| `Category: transport`（英語 enum 生値） | **要改善 → Presentation**（`ItineraryCategory::definition().display_name` 未使用） |
| `場所:` / `メモ:` 日本語ラベル | **Presentation**（field label） |
| `####` / `-` 箇条書き | **Markdown renderer only** |

## 5. Renderer-independent labels

### 5.1 Field label keys（意味論）

ラベル文字列は locale 依存。Presentation 層では **key** を持ち、renderer が文字列化する。

| Key（仮） | 現行 Markdown（沖縄） | 推奨 display（ja） |
|---|---|---|
| `field.category` | `Category:` | `カテゴリ:` |
| `field.place` | `場所:` | `場所:`（維持） |
| `field.inline_memo` | `メモ:` | `メモ:`（維持） |
| `field.duration` | `所要時間:` | 将来 Travel Book で使う場合のみ |
| `field.travel_time` | `移動時間:` | 同上 |
| `reservation.confirmation` | `確認番号:` | 維持（v4.3.0） |
| `reservation.period` | `期間:` | 維持 |
| `reservation.remark` | `備考:` | 維持 |

章見出し（`## Trip overview` 等）の i18n は **Defer**（v4.3.1 V12）。

### 5.2 Category display names

Domain に既に **renderer 非依存の表示名**がある。

```text
ItineraryCategory::definition().display_name  →  「移動」「食事」等（日本語）
```

現行 Markdown は `category.as_str()`（`transport`）をそのまま出力しており、**presentation 層のバグに近い状態**。修正は presentation ルールとして分類し、Markdown 専用ハックにしない。

### 5.3 Summary labels

| 種別 | ラベル方針 | 分類 |
|---|---|---|
| Trip summary | 無ラベル散文 | **Presentation**（Keep） |
| Day summary | 無ラベル散文 | **Presentation**（Keep） |
| Days overview 各項目 | `Day N — date` + 本文 | **Presentation** |
| Note entity 見出し | `### Trip — …` / `### Day N — …` | **Presentation**（Markdown は `###` でレンダリング） |

## 6. Renderer-specific concerns

以下は **Markdown renderer のみ** に留める。

| 関心事 | 例 |
|---|---|
| 見出しレベル | `#` Cover、`##` 章、`###` Day、`####` itinerary |
| 表構文 | Planned cost の `\| Item \| Amount \|` |
| 強調 | Days overview の `**Day 1 — date**` |
| 余白 | 章間・ブロック間の空行 |
| チェックリスト | `- [ ]` / `- [x]` |
| Colophon 固定文言 | `Generated by Caglla.Travel CLI` |
| プレースホルダ | `_No itineraries scheduled._` |

GUI では card / section / list style に置き換わる。業務ルール（「空なら章ごと省略」）は presentation 層が持つ。

| `sort_export_notes_for_travel_book` | Notes 並び |

### 7.2 → Markdown renderer only（残す）

| 現行 | 内容 |
|---|---|
| `format_itinerary_item_markdown` の `####` / `-` 合成 | 見出し・箇条書き構文 |
| `format_planned_cost_chapter` の pipe table 行 | 表レンダリング |

| `format_trip_date_range` の `〜` 結合 | 日付レンジ **表示**（locale フォーマットは presentation へ移せるが優先度低） |

### 7.3 → Domain（変更しない）

| データ | 備考 |
|---|---|
| Trip / Day / Itinerary / Reservation / Estimate / Note | DB・export schema 正本 |
| `list_reservations_for_trip` の SQL 順 | データ取得 |
| `compute_trip_stats` | 会計・集計（Travel Book とは別責務だが overview で参照） |

## 8. Proposed presentation model sketch（最小）

v4.4.0 では **コード化しない**。承認後の v4.4.x で段階導入する最小スケッチ。

```text
TravelBookDocument
  chapters: Vec<TravelBookChapter>   // 省略可能章は Vec に含めない

TravelBookChapter
  kind: Cover | TripOverview | DailySchedule | Reservations | ...
  blocks: Vec<TravelBookBlock>

TravelBookBlock
  // 例: DaySection, ItineraryItem, ReservationEntry, PlannedCostTable, ...

TravelBookItineraryItem
  day_number, sort_order, title
  time_label: Option<TimeLabel>       // 生の start_time + 表示用結合規則
  category: Option<CategoryPresentation>  // enum + display_name
  place_label: Option<String>
  inline_memo: Option<String>
  cues: ItineraryCues                  // has_reservation, has_planned_cost, ...

CategoryPresentation
  domain: ItineraryCategory
  display_name: String                 // from definition() or i18n

LabeledField
  key: FieldLabelKey
  value: String
```

**パイプライン（目標）:**

```text
generate_trip_markdown(conn, trip_id)
  → load domain aggregates
  → build_travel_book_document(...)   // 新規（将来）
  → render_markdown(document)         // markdown.rs の renderer 部分
```
