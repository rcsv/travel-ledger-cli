## 1. Background

v3.4.0 では **Itinerary-level Planned vs Actual difference** を `trip export-md` に導入した。確定済み Estimate と Expense を前提とした derived 集計が Itinerary 単位で見えるようになった。

v3.5.0 では **Receipt Inbox concept design** を documentation-only release として正式化した。未整理の支払い証拠を Expense やダミー Itinerary に押し込まず、Estimate / Receipt / Expense / Difference の責務を分離した。

その後、**Receipt image handling** は明確に **deferred** とした。画像を扱い始めると、保存場所・パス・export/import・欠損・CUI 表示・Attachment / Photo 設計など、CLI / CUI の現在の責務を超える。

次の検討対象は、**画像を扱わない metadata-only Receipt Inbox 実装** である。

```text
Estimate   = planned amount
Receipt    = unreviewed payment evidence
Expense    = confirmed actual spending
Difference = planned-vs-actual delta based on confirmed Expenses
```

---

## 2. Goal

Itinerary に紐づかない **未整理の支払い証拠** を、**画像なし** で一時保存できるようにする。

Receipt は Expense ではない。Receipt は Actual spending として **確定していない**。

| 目的 | 内容 |
|---|---|
| 金額のみ | 金額だけ分かる支払いを残す |
| 日付のみ | 日付だけ分かる支払いを残す |
| メモ | 「これなんだっけ？」というメモを残す |
| 紐づけ | 後から Day / Itinerary に紐づける |
| 除外 | 不要・重複・対象外を `ignored` にできる |
| 昇格余地 | 将来的に Expense へ変換する余地を残す |

Receipt Inbox の価値は、**分からないものを分からないまま保存できること**（concept design 継続）。

---

## 3. Non-goals

v3.6.0 実装計画および初期実装候補で **含めない** 範囲:

```text
- Receipt image support
- image_path field
- image binary storage
- image export / import
- ZIP / archive handling
- OCR
- automatic receipt parsing
- generic Attachment / Photo model
- Balance / Settlement
- Participant-based payment sharing
- Planned vs Actual への即時反映
- Day-level Planned vs Actual difference
- trip stats への反映
- trip stats --json への反映
- during-trip budget control
```

Receipt image handling、Attachment / Photo 汎用化、Day-level difference、Balance / Settlement は **引き続き defer**。

## 4. Data model proposal

### `receipts` table（候補）

```sql
CREATE TABLE receipts (
  id INTEGER PRIMARY KEY,
  trip_id INTEGER NOT NULL,
  day_id INTEGER NULL,
  itinerary_id INTEGER NULL,
  linked_expense_id INTEGER NULL,
  amount INTEGER NULL,
  currency TEXT NULL,
  occurred_date TEXT NULL,
  memo TEXT NULL,
  status TEXT NOT NULL,
  created_at TEXT NOT NULL,
  updated_at TEXT NOT NULL
);
```

| カラム | 方針 |
|---|---|
| `trip_id` | **必須** — Receipt は Trip スコープ |
| `day_id` | 任意 — 同一 Trip 内 Day のみ |
| `itinerary_id` | 任意 — 同一 Trip 内 Itinerary のみ |
| `linked_expense_id` | 任意 — 同一 Trip 内 Expense のみ |
| `amount` / `currency` | 任意 — 片方のみは **拒否**（§9） |
| `occurred_date` | 任意 — ISO 日付文字列候補 |
| `memo` | 任意だが実用上 **重要** |
| `status` | **必須** — §5 |
| `image_path` | **持たない** |

FK は Expense / Estimate と同型で **DB FK なし + アプリ側 cascade** を想定（Entity Design で確定）。

### 最低限の登録パターン例

```text
trip_id + memo
trip_id + amount + currency
trip_id + day_id + memo
trip_id + occurred_date + amount + currency + memo
```

### Trip 階層での位置づけ

```text
Trip
  ├─ Day
  │    └─ Itinerary
  │         ├─ Expense
  │         └─ Estimate
  └─ Receipt   ← Trip-level collection（Itinerary 配下には置かない）
```

## 5. Receipt status lifecycle

| status | Meaning |
|---|---|
| `unreviewed` | 未確認・未整理 |
| `linked` | Day / Itinerary に紐づけ済み。**Expense 化はしていない** |
| `converted` | Receipt を元に Expense を作成済み |
| `ignored` | 旅行費用ではない、重複、対象外 |

初期実装では **`unreviewed` を主役** とする。

遷移候補（実装時に確定）:

```text
unreviewed → linked（receipt link）
unreviewed → ignored（receipt ignore）
unreviewed → converted（receipt convert — deferred 候補）
linked → converted
linked → ignored
```

## 6. CLI command proposal

metadata-only。`--image` フラグは **存在しない**。

### 追加

```bash
caglla receipt add --trip okinawa --day 1 --amount 1700 --currency JPY --memo "これなんだっけ？"
```

`--trip` は trip id または識別子（既存 CLI 慣習に合わせる — Entity Design で確定）。

### 一覧

```bash
caglla receipt list --trip okinawa
caglla receipt list --trip okinawa --unreviewed
caglla receipt list --trip okinawa --status ignored
```

### 詳細

```bash
caglla receipt show 3
```

### 更新

```bash
caglla receipt update 3 --memo "おかんのお土産っぽい"
caglla receipt update 3 --amount 1700 --currency JPY
```

### 紐づけ

```bash
caglla receipt link 3 --day 1
caglla receipt link 3 --itinerary 12
```

`link` 成功時は `status` を `linked` に遷移させる想定。

### 対象外

```bash
caglla receipt ignore 3 --memo "旅行費用ではない"
```

### 削除

```bash
caglla receipt delete 3
```

### 将来候補 — `receipt convert`（v3.6.0 では deferred 可）

```bash
caglla receipt convert 3 --itinerary 12 --title "お土産"
```

Expense 作成ロジックと接続するため、**v3.6.0 初期実装に含めるか要検討**。含めない場合は `linked` + 手動 `expense add` で代替可能。

## 7. Export / import schema impact

Receipt を実装する場合、旅行データの一部となるため **export/import に含める** 方針が自然。

### 基本方針案

| 項目 | 方針 |
|---|---|
| 配置 | **Trip-level collection** — Itinerary 配下には置かない |
| schema | **v7 候補** — v6 から additive |
| 画像 | なし — JSON export のみで表現可能 |
| internal id | export しない（既存 entity と同型） |

### 構造案

```json
{
  "schema_version": 7,
  "trip": { },
  "days": [],
  "participants": [],
  "receipts": [
    {
      "day_ref": { "day_number": 1 },
      "itinerary_ref": { "day_number": 1, "sort_order": 2 },
      "linked_expense_ref": null,
      "amount": 1700,
      "currency": "JPY",
      "occurred_date": "2026-04-26",
      "memo": "これなんだっけ？",
      "status": "unreviewed"
    }
  ]
}
```

`day_ref` / `itinerary_ref` / `linked_expense_ref` の具体形式は、既存 export の **day_number + sort_order / participant name ref** 方式に合わせる（Entity Design + export-schema.md 更新で確定）。

### import / validate-export / diff への影響

| コマンド | 影響 |
|---|---|
| `trip export` | v7 出力 — `receipts[]` 追加 |
| `trip import` | v7 受け入れ — v6 互換維持 |
| `trip validate-export` | v7 receipt フィールド検証追加 |
| `trip diff` | receipt 追加・変更・削除の検出 |

v6 export の import は **引き続き可能**（`receipts` 省略 = 空配列）。

### 注意点

* 内部 DB ID を export/import 参照に **使わない**
* `day_id` / `itinerary_id` / `linked_expense_id` は export 上は **ref オブジェクト** で表現
* Receipt は Difference / stats に影響しない — diff はデータ変更検出のみ

## 8. Planned vs Actual relationship

Receipt は Planned vs Actual difference に **直接反映しない**。

```text
Estimate → Planned
Expense  → Actual
Receipt  → Unreviewed evidence
```

Receipt は Actual ではない。v3.3.0 / v3.4.0 の Difference calculation（`Actual − Planned`）には **含めない**。

`trip stats` / `trip stats --json` / `trip export-md` の Planned / Actual / Difference 出力は **変更しない**（回帰テスト対象）。

### Future work — post-trip review 補助表示

将来的な post-trip review では補助情報として表示できる余地を残す（**本 release 非対象**）:

```text
Confirmed Actual: JPY 18,300
Unreviewed Receipts: JPY 1,700
Potential Actual: JPY 20,000
```

`Potential Actual` は参考表示であり、Difference の正本ではない。

## 9. Validation rules

実装時の候補ルール:

| # | ルール |
|---|---|
| 1 | `status` は `unreviewed` / `linked` / `converted` / `ignored` のみ |
| 2 | `currency` は既存 Expense と **同一通貨ルール**（`money.rs` 再利用） |
| 3 | `amount` がある場合、`currency` **必須** |
| 4 | `currency` がある場合、`amount` **必須** |
| 5 | `day_id` は同一 Trip に属する Day のみ |
| 6 | `itinerary_id` は同一 Trip に属する Itinerary のみ |
| 7 | `linked_expense_id` は同一 Trip に属する Expense のみ |
| 8 | `linked_expense_id` がある場合、`status` は `converted` または `linked` に限定するか検討 |
| 9 | `ignored` でも `amount` / `memo` は保持可能 |
| 10 | `receipt add` は `trip_id` + 少なくとも `memo` または (`amount` + `currency`) を要求するか検討 |
| 11 | delete は **物理削除** で開始。soft delete は将来検討 |
| 12 | export JSON に `image_path` キーがあっては **ならない** |
