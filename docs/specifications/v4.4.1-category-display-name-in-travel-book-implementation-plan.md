## Goal

Daily schedule category lines use `ItineraryCategory::definition().display_name` instead of raw enum strings (`transport`, `hotel`, вҖҰ).

This is a **presentation-oriented** change вҖ” not a Markdown-only mapping table.

## Rule

- When `ItineraryItem.category` is set, emit a detail line with:
  - Field label: `зЁ®еҲҘ:` (consistent with other Japanese Travel Book field labels: `е ҙжүҖ:`, `гғЎгғў:`)
  - Value: `category.definition().display_name` from domain (`з§»еӢ•`, `гғӣгғҶгғ«`, вҖҰ)
- When category is unset, omit the line (unchanged).
- Domain enum values (`as_str()`) are unchanged.
- JSON export and SQLite schema unchanged.

## Code

- `src/io/markdown.rs`
  - `format_travel_book_category_detail_line` вҖ” reusable category detail line (presentation)
  - `format_itinerary_item_markdown` вҖ” calls helper instead of `category.as_str()`

## Tests

- `format_travel_book_category_detail_line` вҖ” Transport вҶ’ `зЁ®еҲҘ: з§»еӢ•`; all categories use definition, not `as_str()`
- `test_export_md_includes_category` вҖ” Hotel вҶ’ `зЁ®еҲҘ: гғӣгғҶгғ«`
