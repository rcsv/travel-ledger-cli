## 1. read-only path の現状確認（v4.6.26 達成点）

## 3. write path との関係

### `main.rs` write handlers（v4.6.27 確認）

| Action | 成功後表示 | adapter 使用 |
|---|---|---|
| **Add** | `get_expense` → `print_expense_detail(&conn, &expense)` | ✓ |
| **Update** | `get_expense` → `print_expense_detail(&conn, &expense)` | ✓ |
| **Delete** | `println!`（amount のみ） | ✗ |

write path は **service 化されていない**。mutation 後に `Expense` 単体を再取得し、`print_expense_detail` が内部で `enrich_expense` を呼ぶ構造。

### write path に adapter が必要な理由

```text
- write handler は ExpenseEnrichedPart を持たない
- write service 化（Tier 3+）は v4.6.x の defer 対象
- add/update 後表示を変えずに enriched へ寄せるには handler 変更が必要（今回対象外）
- print_expense_detail は既に enrich + _from_enriched へ委譲しており、表示内容は read-only と一致
```

**結論:** write command に踏み込まない間、**adapter 残置は妥当**。✓
