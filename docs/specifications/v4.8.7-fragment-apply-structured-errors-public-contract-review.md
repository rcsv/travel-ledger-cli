## 2. Public contract — `structured_errors[]` 要素

### 2.1 Required fields（確定）

| Field | JSON 型 | 安定性 | Client 指針 |
|---|---|---|---|
| `code` | string | **stable public identifier**（registry 済み code） | **分岐の正本** |
| `kind` | string | enum 値 stable（拡張は forward-compatible） | `blocking` / `warning` / `decision_required` — unknown は generic 扱い |
| `message` | string | **unstable**（human-readable） | 表示・ログ用。分岐に使わない |
| `phase` | string | enum 値 stable（拡張は forward-compatible） | UI 段階表示用 — unknown は generic 扱い |

serde 出力例（実装どおり）:

```json
{
  "code": "APPLY_ENTITY_NOT_FOUND",
  "kind": "blocking",
  "message": "Estimate not found: ...",
  "phase": "simulate",
  "intent": "delete_estimate",
  "field_path": "candidate_content.estimate_id"
}
```

### 2.2 Optional fields（確定 — 存在時のみ）

| Field | v4.8.7 露出 | 備考 |
|---|---|---|
| `field_path` | ✓ wired path | dot-separated **hint** — grammar 固定なし（§2.4） |
| `intent` | ✓ simulate / confirm scope 代表 path | fragment intent 文字列 |
| `target_type` | ✓ unsupported target path | `trip` / `day` / `itinerary` 等 |
| その他（`entity_type`, `entity_id`, `expected`, `actual`, …） | ✗ | registry / planning 候補 — **未露出** |

**Client rule:** optional field の欠落は正常。必須扱いしない。

### 2.3 `errors[]` mirror contract（確定）

- wired path では `push_structured_error` が `structured_errors[]` に追加し、同一 `message` を `errors[]` に mirror（`push_unique` で dedupe）
- unwired path では `errors[]` のみ — **`structured_errors[]` に対応 entry なしが正常**
- Client rule: `errors[]` を sole source とする legacy client は引き続き動作。structured-aware client は **`code` 優先、message は参考**

### 2.4 `field_path` grammar（provisional）

v4.8.7 時点で保証するのは **「存在する場合、JSON string である」** のみ。

観測される hint パターン（非 exhaustive）:

```text
target.target_type
target.day_reference
target.itinerary_reference
candidate_content.estimate_id
```

**保証しない:** 全 error path での網羅、命名規則の凍結、locale、array index 表記。将来 `entity_type` / `entity_id` 等が追加されうる。

### 2.5 `required_decisions` と `kind`（provisional / deferred）

| 概念 | 現行 JSON | 公開 contract（v4.8.7） |
|---|---|---|
| `required_decisions[]` container | opaque string hints — **維持** | 人間判断待ちリスト — structured とは別 channel |
| structured `APPLY_REQUIRED_DECISION` | `kind: "blocking"`（`decision_required` ではない） | **code で decision workflow を識別** — `required_decisions[]` 非空と併用 |
| v4.8.4 planning 意図 | `kind=decision_required` 分離 | **deferred** — 実装変更なし。将来 release で kind 整合を検討 |

Client rule:

1. `required_decisions.length > 0` → decision workflow 候補
2. `structured_errors` に `code === "APPLY_REQUIRED_DECISION"` → 同上の機械判定
3. field-level validation blocking（例: `APPLY_FIELD_INVALID`）とは **code** で区別

### 2.6 Unknown / forward-compatible handling（確定）

| Unknown | Client 推奨 |
|---|---|
| unknown `code` | generic blocking 扱い + `message` / `phase` をログ。クラッシュしない |
| unknown `kind` | `blocking` 相当で扱う（安全側） |
| unknown `phase` | 表示ラベルのみ fallback |
| unknown top-level field | 無視（additive 前提） |
| 新 `code` 追加 | **non-breaking**（旧 client は unknown 扱い） |
| 新 optional field | **non-breaking** |

### 2.7 Breaking change 定義（確定）

**breaking（`schema_version` bump 候補）:**

- `errors[]` 削除・改名
- `structured_errors[]` 必須化（success でも空配列出力を強制等）
- required field の削除・改名・型変更
- 既存 `code` の意味変更

**non-breaking:**

- 新 `code`（registry 追加）
- 新 optional metadata field
- 未 wiring path への structured 追加（同一 message mirror 維持前提）
- `message` 文言変更

### 4.2 Unwired paths — `errors[]` only（limited exposure contract）

以下は **現行 contract として正常**。v4.8.7 では実装しない。

| Category | 例 | 備考 |
|---|---|---|
| Fragment file validation 失敗 | envelope / fragment shape | gate 入口 |
| JSON parse / envelope 欠落 | top-level object 不正 | gate |
| Trip / export build 失敗 | DB / export errors | gate |
| Preview export validation 失敗 | `trip_export_valid=false` | gate |
| confirm scope 多数 branch | add / add_note / update_itinerary 等 unsupported target | 代表 2 intent のみ wired |
| simulate — 他 intent unsupported target | **`add_estimate`** 等 plain `Err` | errors[] mirror via gate のみ |
| Parser field errors（非 delete/update estimate wired path） | add_expense parse 等 | 多くは unwired |
| confirm transaction 系 | preview mismatch, TOCTOU, scoped write 0/N rows, post-verify | `anyhow` → `errors[]` string |
| Registry-only codes | §3 表参照 | JSON に出ない |

**Client rule:** `errors.length > 0 && structured_errors.length === 0` — legacy-only blocking。`message` substring 分岐は引き続き可能だが **非推奨**（structured 化後は `code` へ移行）。

## 6. Downstream client guidance（確定）

### 6.1 推奨分岐アルゴリズム

```text
1. Parse JSON report (schema_version >= 2).
2. If structured_errors present:
     branch on structured_errors[].code (primary)
     use phase / kind for UI grouping only
     display message to user (do not match programmatically)
3. Else if errors present:
     legacy fallback — substring or manual mapping (deprecated path)
4. If required_decisions non-empty:
     treat as decision workflow regardless of structured_errors
5. Unknown code → generic error UI + log raw payload
```

### 6.2 API / GUI readiness（v4.8.7 時点）

| 能力 | v4.8.7 可否 | 備考 |
|---|---|---|
| Programmatic branching | **partial** | wired codes のみ — §4 |
| Field highlight | **partial** | `field_path` hint — grammar 未固定 |
| Decision vs validation UX | **partial** | `APPLY_REQUIRED_DECISION` + `required_decisions[]` |
| Retry / conflict | **deferred** | TOCTOU / preview codes 未露出 |
| Full coverage | **no** | limited exposure — by design |
