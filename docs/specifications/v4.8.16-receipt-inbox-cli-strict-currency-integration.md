# v4.8.16 — Receipt / inbox CLI strict currency integration (implementation)

| Release | v4.8.16（released — 本書） |

Receipt / inbox CLI の currency write path を、v4.8.13 CLI write path と同じ `IsoStrict` validation に揃える implementation release です。

「inbox」は独立 CLI ではなく、`receipt` コマンド群（metadata-only Receipt workflow）として実装されています。

---

## 0. Status（結論サマリ）

| 観点 | v4.8.16 判定 |
|---|---|
| 目的 | Receipt CLI の新規 currency 入力を IsoStrict 化 |
| 適用面 | `receipt add` / `receipt update`（currency 明示時）/ `receipt assign`（CLI currency 明示時） |
| 未適用 | `import_receipt_v7`, validate-export, read/list/show/export |
| DB schema | **変更なし** |
| structured error | **追加なし** — 既存 CLI `anyhow` error message |

---

## 1. Currency write surfaces

| Command | Field | Validation |
|---|---|---|
| `receipt add` | `--currency`（`--amount` とセット） | 常に IsoStrict |
| `receipt update` | `--currency` 明示時のみ | IsoStrict + `--amount` 必須（既存 contract） |
| `receipt assign` | CLI `--currency` 明示時のみ | IsoStrict + `--amount` 必須 |
| `receipt assign` | Receipt 既存 currency 使用時 | **再 strict 検証しない** |

### Validator

```rust
crate::money::validate_cli_write_currency_code(code)
// → validate_currency_code_with_mode(code, IsoStrict)
```

CLI entry（`main.rs`）で検証し、正規化済み currency を `receipt.rs` に渡す。domain 層の `validate_currency_code()`（FormatOnly）は import / internal path 用に維持。

---

## 2. Strict reject 方針

| Input | Result |
|---|---|
| `JPY`, `USD`, withdrawn codes | OK（uppercase normalize） |
| `jpy` | OK → `JPY` |
| `ZZZ`, `ABC`, `JPN` | reject |
| `XXX`, `XTS`, `XAU` 等 denylist | reject |
| `JP`, `JPYY` | reject（format invalid） |

---

| Surface | 状態 |
