---

## 1. 目的

```text
fragment validate
  → fragment apply --dry-run --trip <id>（任意・推奨）
  → fragment apply --confirm --trip <id>
  → 既存 Day に itinerary を1件追加
  → itinerary list / day show で確認
```

`--confirm` なしでは Trip / Day / Itinerary を変更しない。

---

## 2. Command

```bash
caglla fragment apply <file> --confirm --trip <id> [--json]
caglla fragment apply <file> --dry-run --trip <id> [--output preview.json] [--json]
```

| フラグ | 用途 |
|---|---|
| `--confirm` | gate 通過後に **既存 Day へ itinerary insert** |
| `--dry-run` | apply preview / simulation（**Trip domain data 非更新**） |
| `--dry-run` + `--confirm` | **併用不可（エラー）** — `dry-run means no Trip domain data side effects` |
| いずれもなし | **エラー** |

---

## 3. 処理フロー

1. Fragment validation（`fragment validate` 同型）
2. `--trip` で Trip 読み取り（read-only DB access）
3. target 解決（Day のみ confirm 対象）
4. in-memory apply simulation + preview Trip JSON
5. `trip validate-export` 自己検証
6. confirm scope 検証（intent / target / required decisions）
7. **`--confirm` 時:** `add_itinerary_item` で1件 insert
8. gate report + insert 結果出力

---

## 4. confirm 最小 scope

```text
Fragment intent:  add → add_itinerary のみ
Target:           Day target のみ
DB operation:     新規 itinerary insert のみ
```

| 条件 | confirm |
|---|---|
| `target_type: day` + `intent: add` | **許可** |
| `target_type: trip` / `itinerary` | **blocking** |
| `intent: enrich` / `warning` / 他 | **blocking** |
| `target_type: unresolved` | **blocking**（dry-run 同型） |
| `required_decisions` 非空 | **blocking** — DB 更新なし |
| ambiguous target | **blocking** — DB 更新なし |

## 6. dry-run / confirm 原則

```text
fragment apply --dry-run:
  read-only DB access
  in-memory simulation
  preview output（--output 指定時）
  Trip / Day / Itinerary は変更しない

fragment apply --confirm:
  read-only DB access + itinerary insert
  supported Fragment のみ既存 Day に反映

fragment apply --dry-run --confirm:
  error
```
