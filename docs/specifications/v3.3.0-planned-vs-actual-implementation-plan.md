## Purpose

```text
v3.1.0 Phase 3 で導入した Planned total（Estimate）と
Actual total（Expense）の並列表示に、通貨別 Difference を追加する。
```

Difference は **DB に永続化しない**。`estimate_totals` と `expense_totals` から **表示レイヤーで導出** する小さな機能リリース。

v3.3.0 では **Trip 単位のみ**。Itinerary / Day 単位の差分、doctor / advisor 連動、為替換算は持ち込まない。

## Definitions

| 概念 | 定義 | 正本 |
|---|---|---|
| **Planned** | Trip 配下 Estimate の通貨別合計 | `estimate_totals`（v3.1.0 実装済み） |
| **Actual** | Trip 配下 Expense の通貨別合計 | `expense_totals`（既存） |
| **Difference** | `Actual − Planned`（通貨別、derived） | **v3.3.0 で新規** |

### Difference の式

```text
difference_totals[currency] =
  expense_totals.get(currency).unwrap_or(0)
  - estimate_totals.get(currency).unwrap_or(0)
```

**keys** = `estimate_totals` の key ∪ `expense_totals` の key（`BTreeMap` で通貨コード昇順）。

| 符号（Actual − Planned） | 意味 |
|---|---|
| **negative** | 予定より少なく使った（アンダー） |
| **positive** | 予定より多く使った（オーバー） |
| **zero** | 一致 |

human / JSON / export-md で **同じ式** を使う。

### 通貨ルール

- 通貨別に差分を出す
- **為替換算しない**
- 片方にしかない通貨は、欠損側を **0** として扱う

例:

```text
Planned:  JPY 180,000
Actual:   JPY 172,500
Difference: JPY -7,500
```

```text
Planned:  JPY 10,000 / USD 50
Actual:   JPY  9,500
Difference: JPY -500 / USD -50
```

### 表示 gate

Difference は **以下を満たす場合のみ** 表示する。

```text
estimate_count > 0 && expense_count > 0
```

| 状況 | Planned | Actual | Difference |
|---|---|---|---|
| Estimate のみ | 表示 | 件数 0、Actual total 省略 | **省略** |
| Expense のみ | 省略 | 表示 | **省略** |
| 両方あり | 表示 | 表示 | **表示** |

Estimate 0 件のとき Planned total を省略する既存ルールは維持する。

## Phase 1 — `trip stats`

### Core helper（`src/analysis/statistics.rs`）

v3.3.0 では **`analysis/difference.rs` は作らない**。pure function を `statistics.rs` に置く。

```rust
/// Planned と Actual の両方があるときのみ Some。
pub(crate) fn compute_difference_totals(
    estimate_count: usize,
    expense_count: usize,
    estimate_totals: &BTreeMap<String, i64>,
    expense_totals: &BTreeMap<String, i64>,
) -> Option<BTreeMap<String, i64>>
```

- gate 不通過 → `None`
- gate 通過 → 全通貨 key の union で `actual - planned` を計算

`TripStats` への追加方針（いずれか、実装時に選択）:

| 方針 | 内容 |
|---|---|
| **A（推奨）** | `TripStats` に `#[serde(skip_serializing_if = "Option::is_none")] difference_totals: Option<BTreeMap<String, i64>>` を追加。`compute_trip_stats` 内で helper を呼ぶ |
| **B** | `TripStats` は変更せず、表示時のみ helper を呼ぶ（JSON に載せるには別途フィールド合成が必要 — **非推奨**） |

**推奨: 方針 A** — human / JSON / markdown が同じ `TripStats` を参照できる。

### `caglla trip stats`（human）

`Planned total` / `Actual total` ブロックの **後** に、gate 通過時のみ:

```text
Difference:
  JPY -7,500
```

- ラベル: **`Difference:`**（英語）
- 金額フォーマット: 既存どおり `crate::money::format_amount_value`
- 通貨順: `BTreeMap` 昇順

### `caglla trip stats --json`

additive field のみ（breaking 変更なし）:

```json
{
  "trip_name": "Example",
  "estimate_count": 2,
  "estimate_totals": { "JPY": 180000 },
  "expense_count": 3,
  "expense_totals": { "JPY": 172500 },
  "difference_totals": { "JPY": -7500 }
}
```

| フィールド | 型 | 条件 |
|---|---|---|
| `difference_totals` | `{ "CUR": i64, ... }` | gate 通過時のみ。不通過時は **省略**（`null` ではない） |

既存フィールド（`estimate_totals` / `expense_totals` 等）は **変更しない**。
