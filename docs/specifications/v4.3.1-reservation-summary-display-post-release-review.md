## 1. Summary

v4.3.0 は v4.2.x の章立てを維持したまま、**Reservations 章の読みやすさ**と **Trip overview での日別俯瞰**を改善した。沖縄 sample を Travel Book として読むと、次が確認できる。

1. **Days overview** — Daily schedule（58 Itinerary）の前に 4 日分の要約が一覧でき、旅行全体の流れを先に把握できる
2. **Reservations 旅程順フラット** — `### Flight` の重複が消え、往路→レンタカー→ホテル→復路の時系列が自然
3. **日本語ラベル・人間可読 Period** — `確認番号:` / `期間:` / `備考:` とホテル期間の整形がしおり向き
4. **便予約の見出しのみブロック** — 確認番号等が無い Flight は 1 行見出しのみ。意図は明確だが、やや「空」に見える
5. **Planned cost の Note 列** — v4.2.2 で fixture 注記除去後、全 8 表で Note が空欄

**結論:** v4.3.0 の表示方針と責務境界は **Keep**。実装バグや責務逸脱はない。改善は **小さな polish** に限定し、**v4.3.2 以降** に分離するのが適切。v4.3.1 は本 Review と分類表の確定のみ（documentation-only）。

## 3. Findings by review point

### 3.1 Days overview

**観測（沖縄）:**

```markdown
### Days overview

- **Day 1 — 2026-04-26**: 中部国際空港から那覇へ移動し、…
- **Day 2 — 2026-04-27**: 朝から美ら海水族館を楽しみ、…
…
```

| 観点 | 評価 |
|---|---|
| Trip overview 内の位置（metrics 直後） | Daily schedule 前の **俯瞰として有用** |
| 4 日 Trip でのリスト可読性 | 1 行 1 日でスキャンしやすい。**Keep** |
| 見出し `### Days overview` | `## Trip overview` の子見出しとして自然。英語だが章見出し体系と整合（v4.2.1 R12 は引き続き Defer） |
| 長期 Trip（7 日以上） | リストが長くなる可能性 — 現時点では **Keep**、折りたたみ・truncate は **Defer** |
| Daily schedule との重複感 | 同一 Day summary 文が 2 回出るが、役割が異なる（下記 §3.2） |

**分類:** **Keep** — Days overview の追加は v4.3.0 の意図どおり機能している。

---

### 3.2 Day summaries（二重表示）

**観測:** 各 Day の summary 本文が **Days overview**（箇条書き）と **Daily schedule 日見出し直後**（段落）の両方に同一内容で出る。

| 役割 | 表示場所 | 読者行動 |
|---|---|---|
| 俯瞰 | Days overview | 旅行全体を先に読む |
| 文脈 | Daily schedule 各日冒頭 | その日の Itinerary 58 件を読む直前のリマインダー |

| 選択肢 | 評価 |
|---|---|
| 両方維持（現状） | overview＝一覧、日別章＝詳細前の文脈。**旅行しおりとして妥当** |
| Days overview のみ | Daily schedule 冒頭が唐突になる |
| Daily schedule のみ | Trip overview で日別流れが掴みにくい |

**分類:** **Keep** — 二重表示は **意図的な重複**（overview vs detail context）。v4.3.0 設計の前提を **確認**。

---

### 3.3 Reservations chapter（v4.3.0 フラット形式）

**観測（沖縄 5 件）:**

```markdown
**Day 1 / P1 G Parking**
確認番号: EXAMPLE-PARKING-001
備考: 出発日の駐車場予約

**Day 1 / NU045 NGO ⇒ OKA (11:00着)**

**Day 1 / Toyota Alphard 又は同等車種** — Ks Rent A Car
備考: ETCカード必須

**Day 1 / チェックイン** — ヒルトン瀬底
期間: 2026-04-26 16:40 〜 2026-04-29 10:00

**Day 4 / NU046 OKA ⇒ NGO (22:05着)**
```

| 観点 | 評価 |
|---|---|
| 種別 `###` 廃止 | 往路・復路で Flight 見出しが分裂しなくなった。**Keep** |
| 旅程順 | Day 1 塊 → Day 4 復路。**Keep** |
| `確認番号:` / `期間:` / `備考:` | 日本語本文と整合。**Keep** |
| Provider subtitle | ホテル・レンタカーで有用。便名は itinerary 見出しに集約。**Keep** |
| 見出しのみの Flight ブロック | 確認番号・備考・期間が無いため **1 行のみ** |

**見出しのみ Flight の扱い:**

| 選択肢 | 分類 | 理由 |
|---|---|---|
| 現状維持（見出しのみ出す） | **Keep** | Reservation entity が存在することを Reservations 章に示す。Itinerary 見出しと対応づく索引として機能 |
| 将来 seed / データで enrichment | **Defer** | 確認番号等が DB に入れば自然に行が増える。export 側の特別ロジック不要 |
| 詳細フィールドが無いブロックを非表示 | **Do not do** | 予約の存在自体が消え、Reservations 章と Daily schedule の対応が分かりにくくなる |
| 見出しのみブロックを 1 行に圧縮（例: インライン化） | **Polish candidate** | 優先度低。v4.3.2 で検討可 |

**分類:** Reservations 章の v4.3.0 形式は **Keep**。空に見える Flight は **Keep（現状）** + enrichment は **Defer**。

### 3.4 Planned cost — Note 列

**観測:** 8 件すべての Estimate 表で `Note` 列が空（v4.2.2 で fixture 注記を seed から除去した結果）。

```markdown
| Item | Amount | Note |
|---|---:|---|
| 駐車場（出発） | JPY 13,000 |  |
```

| 観点 | 評価 |
|---|---|
| 空 Note 列の視覚ノイズ | 沖縄 sample では全表で空 — 列が意味を持たない |
| ユーザーが note を入れた Trip | 列は有用 — **透過表示は Keep** |
| 章全体で Note がすべて空 | 列ごと省略するとすっきりする |

**分類:** **Polish candidate → v4.3.2**（Trip 全体または Itinerary グループ内で Note がすべて空のとき列省略）。v4.3.1 の主目的ではない。

---

### 3.5 Responsibility boundaries

| 境界 | 観測（v4.3.0 後） | 分類 |
|---|---|---|
| Summary = 要約 | Trip summary、Days overview、Daily schedule 日冒頭 — 予約番号・金額なし | **Keep** |
| Reservation = 予約確認 | `## Reservations` に集約。確認番号・期間・備考のみ | **Keep** |
| Itinerary = 行動 | Daily schedule の `####` ブロック | **Keep** |
| Note = 補足メモ | `## Notes` + Itinerary `note` 行 | **Keep** |
| Estimate = 予定費用 | `## Planned cost` のみ | **Keep** |
| Expense / Receipt | Travel Book 非含有（golden に言及なし） | **Keep** |

**分類:** 責務境界は v4.0.0 / v4.1.0 / v4.3.0 設計どおり。**変更なし**。
