## 1. DB schema

| Column | Type | Notes |
|---|---|---|
| `main_destination` | TEXT NULL | 代表目的地ラベル |
| `main_destination_country_code` | TEXT NULL | ISO 3166-1 alpha-2 |
| `default_currency` | TEXT NULL | ISO 4217 |

Migration: `migrate_trips_metadata()` — 既存 `add_column_if_not_exists` 方式。既存 Trip は NULL のまま valid。

---

## 2. Validation

### `main_destination`

- trim; 空 / whitespace-only → NULL
- max length: **200 characters**（`TRIP_MAIN_DESTINATION_MAX_LEN`）
- Venue FK なし

### `main_destination_country_code`

- lowercase → uppercase normalize
- **IsoStrict country registry**（v4.9.1 で確定）
- 現在正式に割り当てられた ISO 3166-1 alpha-2 のみ受理
- reserved / historical / unknown は reject（例: `ZZ`, `EU`, `UK`, `AN`）
- Registry source: static sorted slice in `src/geo/country_registry.rs`（`currency_registry` と同型の保守方針）
- Reference: [ISO 3166-1 Maintenance Agency](https://www.iso.org/iso-3166-country-codes.html) — **Officially assigned alpha-2** code list
- Snapshot: **2026-07**（v4.9.1 release 時点の conservative static snapshot）
- Maintenance: reserved / withdrawn / user-assigned codes are excluded; updates are explicit registry bumps（automatic sync なし）
- Test: `registry_is_sorted` uses strict adjacent `<` comparison（unsorted **and duplicate** detection）

### `default_currency`

- `validate_cli_write_currency_code()` 再利用（IsoStrict）
- metadata 未変更の update では既存値を再検証しない
- clear 操作は validation 対象外

---

## 3. CLI

### Flags

```text
--main-destination
--main-destination-country-code
--default-currency
--clear-main-destination
--clear-main-destination-country-code
--clear-default-currency
```

- set / clear 同時指定は CLI error（clap `conflicts_with` + domain guard）
- `trip list` plain text: **変更なし**
- `trip show`: metadata 設定時のみ表示
- `trip show --json` / `trip list --json`: `TripCliJson` で additive 反映（NULL は JSON null）

---

## 4. JSON contract

- CLI Trip JSON: metadata fields additive（`TripCliJson`）
- export/import JSON: metadata **omit**（`Trip` metadata fields use `#[serde(skip)]`）
- validate-export / Proposal / Markdown export: 変更なし
