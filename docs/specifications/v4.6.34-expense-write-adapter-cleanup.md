## 2. adapter 棚卸し

### 削除（v4.6.34）

| Helper | 理由 |
|---|---|
| `print_expense_detail(&conn, &expense)` | write から外れ、tests も未使用。`_from_enriched` が正式経路 |
| `print_expense_list(&conn, target, &expenses)` | `main.rs` / tests とも未使用。`_from_enriched` が正式経路 |
| `expense_to_json(&conn, &expense)` | `enrich_expense` + `enriched_expense_to_json` への薄い委譲のみ。tests を enriched path へ寄せて削除 |

### 残置（`#[allow(dead_code)]` 継続）

| Helper | 残置理由 |
|---|---|
| `format_expense_markdown_line` | Markdown export（Travel Book）用。現行 export は別 helper 経路だが、shared expense spec 上の予定 API |
| `duplicate_expense_beneficiaries` | trip duplicate 時の beneficiary ID remap。export/import ref resolution 経路で将来使用 |

### 正式経路（削除対象外）

| Helper | 用途 |
|---|---|
| `enrich_expense` | read/write services の enrichment |
| `enriched_expense_to_json` | CLI wire DTO 組み立て（list/show JSON、detail human 内部） |
| `print_expense_detail_from_enriched` | add/update/show human |
| `print_expense_list_from_enriched` | list human |
| `add_expense` / `update_expense` / `delete_expense` | write services から呼び出し（mutation 本体） |

## 4. service result と CLI output の境界

```text
service layer:
  ExpenseAddServiceResult    { id, expense: ExpenseEnrichedPart }
  ExpenseUpdateServiceResult { expense: ExpenseEnrichedPart }
  ExpenseDeleteServiceResult { id, amount, currency }

CLI layer:
  success println!
  enriched_expense_to_json     → JSON wire（read）
  print_*_from_enriched        → human（read + write add/update）
```

adapter 削除は **CLI 内部の委譲短縮** のみ。service API・human 表示・JSON shape は不変。
