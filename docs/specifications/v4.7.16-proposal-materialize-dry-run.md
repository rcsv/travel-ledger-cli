## 1. 目的

```text
proposal validate
  → proposal materialize --dry-run
  → schema v8 Trip JSON 候補
  → trip validate-export
  → optional trip diff
```

DB に書き込まず、採用 gate の安全パターンをコードで固定する。

---

## 2. Command

```bash
caglla proposal materialize <file> --dry-run [--output <trip.json>] [--start YYYY-MM-DD] [--end YYYY-MM-DD] [--json]
```

| フラグ | 用途 |
|---|---|
| `--dry-run` | **必須**（v4.7.16 は dry-run のみ） |
| `--output` | schema v8 Trip JSON **候補**の出力先。**後続の `trip validate-export` 等で扱う場合は `--output` を使う** |
| `--start` / `--end` | Envelope 内で日付未確定のときに旅行期間を指定 |
| `--json` | **materialize gate report** を JSON 出力。**Trip JSON 候補そのものではない** |

`--dry-run` なしは **エラー**（将来の `--confirm` 誤実行防止）。

### 2.1 出力ストリーム（stdout の扱い）

| モード | stdout | Trip JSON 候補 |
|---|---|---|
| human（`--json` なし、`--output` なし） | gate report（human）のあと Trip JSON 候補が続く | stdout に混在 — **人間向けプレビュー用** |
| `--output <file>` | gate report（human）のみ（成功時は書き込み先パスを表示） | **ファイル** — **後続コマンド向け** |
| `--json` | gate report JSON のみ | `--output` 指定時のみファイル；未指定時は stdout に出さない |

```text
Trip JSON candidate を後続コマンドで扱う場合は --output を使う。
--json は gate report JSON を出すものであり、Trip JSON candidate そのものではない。
```

---

## 3. 処理フロー

1. Envelope JSON を読み込み
2. 既存 `analyze_proposal_envelope_json` 相当（`proposal validate` と同型）を実行
3. materialize gate — 日付解決、採用 Day / Itinerary 非空、day_number 範囲
4. `candidate_days` / `candidate_itineraries` を schema v8 `days[]` / `itineraries[]` にマッピング
5. `TripExportV3` を組み立て、`trip validate-export` 相当で自己検証
6. gate report 出力 + Trip JSON 候補を `--output` または stdout へ

---

## 4. 日付解決（優先順）

1. CLI `--start` / `--end`（両方必須）
2. `proposal.start_date` / `proposal.end_date`
3. `candidate_date_ranges[]` の `confirmed: true` な最初の range

未解決は **blocking error**。

---

## 5. Gate report

`ProposalMaterializeDryRunReport`（schema version 1）:

| フィールド | 意味 |
|---|---|
| `envelope_valid` | `proposal validate` 相当の結果 |
| `trip_export_valid` | 生成 JSON が `trip validate-export` に合格したか |
| `errors` | blocking（採用停止） |
| `warnings` | Envelope warnings + gate non-blocking |
| `required_decisions` | `materialize_hints.required_decisions` + 日付未確定ヒント |
| `output` | 成功時の trip 概要（name / dates / counts） |

## 6. 責務境界

```text
Trip Proposal Envelope  → proposal validate / show / inspect / materialize --dry-run
schema v8 Trip          → trip validate-export / trip import
Proposal Fragment       → fragment validate
DB update               → 今回対象外
```

materialize 出力は schema v8 Trip JSON **候補**。DB に保存された Trip ではない。
