| target | **itinerary のみ** |
| Estimate identity | **`candidate_content.estimate_id`**（DB PK）+ 親 Itinerary 検証 |
| missing / already deleted | **blocking** — idempotent success 不可 |
| optional baseline | `expected_*` + `expected_updated_at` — 不一致は blocking |
| mutation fields | `amount` / `title` 等の更新 field は **拒否** |

## 1. Semantic boundary

```text
Estimate = Planned Money（削除対象）
Expense  = Actual Money（不変）
```

- `delete_estimate` は **既存 Estimate 1 件の削除 preview のみ**
- Expense / Reservation / Note / Itinerary は変更しない
- sibling Estimate の `sort_order` は変更しない

---

## 2. supported scope

```text
intent:
  delete_estimate（dry-run のみ）

target:
  itinerary のみ（trip / day は unsupported）

candidate_content:
  estimate_id（必須、正の整数）
  expected_amount / expected_currency / expected_title / expected_note /
  expected_sort_order / expected_updated_at（任意 baseline）

DB:
  read-only
  write なし
```

---

## 3. gate report / preview

`FragmentApplyPreviewSummary` 追加フィールド:

| フィールド | 意味 |
|---|---|
| `action` | `delete_estimate` |
| `estimates_before` / `estimates_after` | Trip export 内 Estimate 件数（after = before − 1） |
| `estimate_delete_preview` | 削除対象の完全 snapshot + target identity |

`estimate_delete_preview` DTO:

| フィールド | 意味 |
|---|---|
| `target_itinerary_id` | 解決済み Itinerary DB ID（identity 正本） |
| `target_itinerary_title` | 表示用（agreement には使わない） |
| `target_estimate_id` | 削除対象 Estimate DB ID（identity 正本） |
| `title` / `amount` / `currency` / `note` / `sort_order` / `updated_at` | confirm TOCTOU baseline 材料 |

`FragmentApplyDryRunReport.deleted_estimate_id` は型上存在するが **dry-run / failure 時は unset**。v4.8.2 confirm で設定。

destructive warning: hard delete かつ undo 非保証。

### invalid report

- blocking errors → `valid=false`、DB 不変、削除後 export preview 抑制
- `required_decisions` 非空 → `valid=false`、DB 不変
- stale `expected_*` baseline mismatch → blocking
- mutation field 指定 → blocking

---

## 4. dry-run 原則

```text
fragment apply --dry-run:
  read-only DB access
  in-memory export から対象 Estimate 1 件除去
  estimates テーブルは変更しない
```

推奨内部責務分離（confirm 再利用向け）:

```text
parse_delete_estimate_fields
resolve parent Itinerary
load current estimate
validate parent ownership
validate optional expected_*
build estimate_delete_preview snapshot
remove target from in-memory export
build dry-run report
```
