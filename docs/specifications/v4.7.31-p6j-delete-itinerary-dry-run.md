## 1. Venue / Place handling（前提補足）

v4.7.30 policy §2.2.1 に加え、v4.7.31 実装前に以下を正本として扱う。

```text
Venue / Place data is treated as provider-derived cache, not as user-authored travel evidence.

Therefore, Venue links or itinerary-owned Venue snapshots must not block delete_itinerary.
Deleting an itinerary may remove its Venue relationship or itinerary-owned provider cache.

Shared provider cache may be retained for API cost reduction or garbage-collected when unreferenced.
In either case, Venue cache is not part of the user-facing rollback guarantee for delete_itinerary.
```

### 1.1 分類

| 種別 | delete blocker | rollback 保証 | 備考 |
|---|---|---|---|
| Expense | **Yes** | 将来 v4.8+ | user-authored / 実績 |
| Estimate | **Yes** | 将来 v4.8+ | planned money |
| Reservation | **Yes** | 将来 v4.8+ | 予約証跡 |
| Note（entity） | **Yes** | 将来 v4.8+ | owner_type = itinerary |
| Venue link / snapshot / provider cache | **No** | **対象外** | 再取得可能な provider 由来 |

Venue model は [venue-model-introduction-policy.md](venue-model-introduction-policy.md) で v4.8+ 候補。**v4.7.31 実装時点では DB 上に Venue テーブルが無い** ため、blocking 判定は既存 entity のみ。

### 1.2 Itinerary delete 時の Venue（将来）

confirm 実装（v4.7.32）時の想定:

- 削除可: itinerary–Venue 関連、itinerary-owned snapshot、itinerary 専用 place pointer
- 残置可: 共有 provider cache（API 節約）
- cleanup: 未参照 cache の GC は **delete_itinerary の責務外**

## 2. Dry-run scope

### 2.1 処理フロー

```text
fragment validate
  → target itinerary 解決
  → blocking child counts（Expense / Estimate / Reservation / Note）
  → blocking > 0 なら errors で終了（DB 不変）
  → in-memory export から itinerary 行を除去した preview
  → gate report（delete_itinerary preview）
```

### 2.2 supported scope

```text
intent:   delete_itinerary
target:   itinerary のみ
DB:       read-only（DELETE なし）
```

Trip / Day / unresolved / ambiguous → **blocking**（v4.7.30 同型）

### 2.3 Blocking child 判定（現行 DB）

| entity | カウント方法 | blocking |
|---|---|---|
| Expense | `list_expenses_for_itinerary` 相当 | count > 0 |
| Estimate | itinerary 配下 estimates | count > 0 |
| Reservation | itinerary 配下 reservations | count > 0 |
| Note | `owner_type = itinerary` | count > 0 |
| Itinerary 列 `note` | **含めない** | — |

**Venue:** 現行 DB に無い → 判定・表示とも **スキップ**（方針のみ本書・v4.7.30 policy に記載）

### 2.4 Dry-run preview（gate report）

| 項目 | v4.7.31 MVP | 将来（Venue 実装後） |
|---|---|---|
| `preview.action` | `delete_itinerary` | 同左 |
| `preview.deleted_itinerary` | title, day, sort_order 要約 | 同左 |
| `preview.blocking_children` | `{ expenses, estimates, reservations, notes }` 件数 | 同左 |
| `preview.non_blocking_relations` | **省略可** | `{ venue_links, venue_snapshots, provider_cache_refs }` 情報表示 |
| export preview | 当該 itinerary 除去後の in-memory Trip JSON | 同左 |
| warnings | 永久削除・undo 非保証（confirm 時の文言先取り可） | Venue cache は rollback 対象外 |

schema v8 / JSON golden 変更は不要。gate report schema v2 の preview 拡張で足りる想定。

### 2.5 Safety（dry-run）

- DB read-only
- blocking child > 0 → **blocking**、preview は生成しないか errors のみ
- childless → preview 生成、export から itinerary 除去をシミュレート
- `--dry-run --confirm` → error
