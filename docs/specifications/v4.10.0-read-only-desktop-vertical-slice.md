## 1. Goal

実際の Travel Ledger SQLite DB を native file dialog で選択し、GUI 上で次の read-only フローを辿れること。

```text
アプリ起動 → DB選択 → Trip一覧 → Trip詳細 + Day一覧 → Day選択 → Itinerary timeline
```

CLI command string / stdout 解析は使用しない。`travel_ledger_cli` library facade のみ。

**Developer preview / source-only** — bundle 配布・code signing・App Store 対応は scope 外。

## 4. Path dependency boundary

`desktop/src-tauri/Cargo.toml`:

```toml
travel-ledger-cli = { path = "../.." }
```

read command は **必ず** root 公開 API のみ使用:

- `open_db`
- `list_trip_summaries`
- `get_trip_detail`
- `get_day_timeline`
- DTO / `ServiceError`

desktop 側で SQL 再実装・独自 Trip query layer は禁止。

---

## 5. Tauri commands

| Command | Role |
|---|---|
| `select_database(path)` | path 検証 → `open_db` → facade probe → state 保存 |
| `list_trip_summaries()` | 選択 DB から Trip 一覧 |
| `get_trip_detail(trip_id)` | Trip 詳細 + Day 一覧 |
| `get_day_timeline(trip_id, day_number)` | Day timeline + Itinerary 一覧 |

Rust / TypeScript で command 名を統一（snake_case）。

## 6. Managed state

```rust
struct DesktopState {
    selected_db_path: Mutex<Option<PathBuf>>,
}
```

- 長寿命 `Connection` / pool / global singleton なし
- 各 command: path clone → lock 解放 → `open_db` → facade → DTO 返却
- DB 処理中に Mutex を保持し続けない

## 7. DB selection と validation

### Frontend

- `@tauri-apps/plugin-dialog` で file open
- filters: `*.db` / `*.sqlite` / `*.sqlite3`（UX 補助のみ）
- キャンセル → error 表示なし

### Backend (`select_database`)

1. path 存在
2. regular file
3. `open_db(path)` 成功
4. `list_trip_summaries` で読み取り可能であること確認
5. **成功時のみ** state へ path 保存
6. 存在しない path からの新規 DB 意図せず作成を避ける（`open_db` は既存ファイル前提の検証フロー）

## 8. Desktop error contract

```json
{ "code": "TRIP_NOT_FOUND", "message": "..." }
```

| Code | Source |
|---|---|
| `DATABASE_NOT_SELECTED` | desktop |
| `DATABASE_PATH_INVALID` | desktop |
| `DATABASE_OPEN_FAILED` | desktop（`open_db` 失敗） |
| `TRIP_NOT_FOUND` | `ServiceError` mapping |
| `DAY_NOT_FOUND` | `ServiceError` mapping |
| `STORAGE_FAILURE` | `ServiceError` mapping |
| `DATA_MAPPING_FAILURE` | `ServiceError` mapping |

raw SQL / stack trace / 内部 path を frontend へ露出しない。

## 9. UI states

| State | Behavior |
|---|---|
| DB 未選択 | empty state + Open ボタン |
| DB 選択中 | loading |
| Trip 0 件 | empty state |
| Trip 選択済み | detail + day tabs + timeline |
| Day 0 件 | day empty |
| Itinerary 0 件 | timeline empty |
| structured error | banner（code + message） |
| DB 選択キャンセル | 変更なし |

Trip 切替時は前 Trip の Day/timeline をクリア。DB 選択後は先頭 Trip / 先頭 Day を自動選択可。

## 11. Read-only 境界

- add / update / delete command なし
- frontend 編集フォームなし
- `open_db()` による既存 migration lifecycle は core 契約に従う（read-only 操作としての schema 初期化は許容）

### Itinerary completion state

**Itinerary に完了状態（`is_done` / `done` / `undone`）は存在しない。** これは抜けではなく意図的な責務分離である。

| Entity | Completion | Notes |
|---|---|---|
| **Itinerary** | **なし** | 旅行行動の流れ。タスク消化の行ではない |
| **Checklist** | `checklist_items.is_done` + `checklist check` / `uncheck` | 出発前の確認タスク |

desktop timeline は title / start_time / location / duration / note 等の既存 Itinerary フィールドのみ表示する。`itinerary_items.is_done` column・migration・DTO フィールドは **導入しない**（v4.10.0 で一度試行し撤回）。

参照: [planning-design-principles.md §2](planning-design-principles.md#2-itinerary-is-not-a-task-row) / [itinerary-model.md](itinerary-model.md)

## 13. Manual smoke test

```sh
# 1. canonical sample DB（repo root）— clean workdir で seed（is_done 直接 SQL は使わない）
cargo build
SMOKE_DIR=$(mktemp -d) && export CAGLLA_SAMPLE_WORKDIR="$SMOKE_DIR"
bash samples/okinawa_sesoko_2026/seed.sh
SMOKE_DB=/tmp/travel-ledger-desktop-smoke.db
cp "$SMOKE_DIR/caglla.db" "$SMOKE_DB"

# schema 確認: itinerary_items に is_done 列が無いこと
sqlite3 "$SMOKE_DB" "PRAGMA table_info(itinerary_items);" | grep -i is_done && exit 1 || true

# 2. desktop dev または release binary
cd desktop && npm install && npm run tauri dev
# open src-tauri/target/release/travel-ledger-desktop

# 3. Open DB → $SMOKE_DB
#    - Trip 一覧 / Day tabs / Itinerary timeline（58 件規模）
#    - start time / location なし行でも崩れないこと
#    - 狭い window で sidebar + detail が操作可能なこと

# 4. backend smoke（GUI なし）
SMOKE_DB="$SMOKE_DB" cargo test --manifest-path desktop/src-tauri/Cargo.toml smoke_populated_sample_database -- --ignored
```
