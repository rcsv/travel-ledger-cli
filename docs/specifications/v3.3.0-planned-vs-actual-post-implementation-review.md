## 1. Overview

v3.3.0 は **Trip 単位の Planned vs Actual difference** を、表示レイヤーの **derived 集計** として追加した小さな機能リリース。

### 到達点

| 項目 | 判定 |
|---|---|
| **Difference 定義** | `Actual − Planned`（通貨別）— human / JSON / export-md で一貫 | ✅ |
| **符号** | negative = 予定より少なく使った / positive = オーバー / zero = 一致 | ✅ |
| **表示 gate** | `estimate_count > 0 && expense_count > 0` のみ | ✅ |
| **通貨 union** | `estimate_totals ∪ expense_totals`、欠損側 0 | ✅ |
| **trip stats human** | `Difference:` ブロック追加 | ✅ |
| **trip stats --json** | `difference_totals` additive field（条件外は省略） | ✅ |
| **trip export-md** | Overview のみ `Difference:` | ✅ |
| **DB schema** | 変更なし | ✅ |
| **export / import schema** | v6 維持、変更なし | ✅ |
| **`make check`** | PASS | ✅ |

**結論:** v3.1.0 で defer していた Difference 表示を、DB 永続化なしで Trip 単位に成立させた。Release blocker となる問題はない。

## 3. Difference Semantics Review

### 式

```text
difference_totals[currency] =
  expense_totals.get(currency).unwrap_or(0)
  - estimate_totals.get(currency).unwrap_or(0)
```

| 符号（Actual − Planned） | 意味 |
|---|---|
| **negative** | 予定より少なく使った |
| **positive** | 予定より多く使った |
| **zero** | 一致 |

### 表示 gate

| 状況 | Difference |
|---|---|
| Estimate + Expense 両方あり | **表示** |
| Estimate のみ | **省略** |
| Expense のみ | **省略** |
| どちらもなし | **省略** |

### JSON

- `difference_totals` は **additive field**
- gate 不通過時は **省略**（`null` ではない）
- 既存 `estimate_totals` / `expense_totals` は変更なし

## 6. Deferred Scope

v3.3.0 および本 Review **では実装しない** 範囲:

```text
- Day 単位 difference
- Itinerary 単位 difference
- Budget 独立エンティティ
- doctor / advisor での Estimate 活用
- FX conversion
- trip export JSON への difference 永続化
- Shared Expense / Settlement 連動
- --db <path> / CAGLLA_DB / db use
- GUI / Web 版 Planned vs Actual カード表示
```

次フェーズ候補: **Itinerary 単位 difference**、**doctor / advisor Estimate 活用**、**commands/ への段階移行**。
